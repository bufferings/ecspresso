1.2 インストール方法

バイナリからのインストール

ecspressoは、Go言語で書かれたシングルバイナリアプリケーションです。GitHubリリースページから、お使いのプラットフォーム向けの最新バイナリをダウンロードして使用できます。

Linux / macOS

$ curl -sL https://github.com/kayac/ecspresso/releases/latest/download/ecspresso-{プラットフォーム}-{アーキテクチャ} -o ecspresso
$ chmod +x ecspresso
$ sudo mv ecspresso /usr/local/bin/

Homebrew（macOS）

$ brew install kayac/tap/ecspresso

ソースからのビルド

Go言語の開発環境がある場合は、ソースからecspressoをビルドすることもできます。

$ go install github.com/kayac/ecspresso/v2/cmd/ecspresso@latest

Dockerでの実行

Dockerがインストールされている環境では、ecspressoのDockerイメージを使用することも可能です。

$ docker run -it --rm \
  -v ~/.aws:/home/ecspresso/.aws \
  -v $(pwd):/work \
  -w /work \
  kayac/ecspresso:latest [コマンド]

AWS認証の設定

ecspressoはAWS SDKを使用してAWSリソースにアクセスします。以下のいずれかの方法でAWS認証情報を設定してください。

AWS CLIの認証情報を使用

すでにAWS CLIが設定されている場合は、その認証情報がecspressoでも使用されます。

$ aws configure
AWS Access Key ID [None]: [アクセスキー]
AWS Secret Access Key [None]: [シークレットキー]
Default region name [None]: ap-northeast-1
Default output format [None]: json

環境変数の使用

export AWS_ACCESS_KEY_ID=[アクセスキー]
export AWS_SECRET_ACCESS_KEY=[シークレットキー]
export AWS_REGION=ap-northeast-1

IAMロールの使用（EC2、ECSタスク、Lambda内）

AWS環境内で実行する場合は、インスタンスプロファイルやタスクロールを使用できます。この場合、明示的な認証情報設定は不要です。

バージョンの確認

インストールが正常に完了したことを確認するには、バージョン情報を表示します。

$ ecspresso version
