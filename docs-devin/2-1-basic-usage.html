2.1 基本的な使い方

設定ファイルの作成

ecspressoを使用するには、まず設定ファイル（ecspresso.yml）を作成する必要があります。既存のECSサービスから設定ファイルを生成するには、以下のコマンドを使用します。

$ ecspresso init --region ap-northeast-1 --cluster your-cluster --service your-service

これにより、現在のディレクトリに以下のファイルが生成されます：

  • ecspresso.yml - ecspressoの設定ファイル
  • ecs-service-def.json - ECSサービス定義
  • ecs-task-def.json - ECSタスク定義

基本的なワークフロー

graph TD
    A[設定ファイル作成] --> B[タスク定義の編集]
    B --> C[デプロイ前の検証]
    C --> D[サービスのデプロイ]
    D --> E[ステータス確認]

サービスのデプロイ

サービスをデプロイするには、以下のコマンドを使用します。

$ ecspresso deploy

デプロイ前に変更内容を確認するには、--dry-runオプションを使用します。

$ ecspresso deploy --dry-run

サービスのステータス確認

デプロイしたサービスのステータスを確認するには、以下のコマンドを使用します。

$ ecspresso status

タスク定義とサービス定義の差分確認

ローカルの設定ファイルと実際にデプロイされているリソースの差分を確認するには、以下のコマンドを使用します。

$ ecspresso diff

タスクの実行

一時的なタスクを実行するには、以下のコマンドを使用します。

$ ecspresso run

特定のコマンドをタスク内で実行するには、タスク定義のオーバーライドを使用します。

$ ecspresso run --overrides='{"containerOverrides":[{"name":"app","command":["echo","hello"]}]}'

設定ファイルの例

以下は、基本的なecspresso.ymlの例です。

region: ap-northeast-1
cluster: your-cluster
service: your-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m

より詳細な設定オプションについては、コマンドリファレンスを参照してください。
