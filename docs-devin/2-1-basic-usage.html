2.1 基本的な使い方

プロジェクトの初期化

ecspressoを使用するには、まず設定ファイルを作成する必要があります。initコマンドを使用して、既存のECSサービスから設定を生成できます。

$ ecspresso init --region ap-northeast-1 --cluster your-cluster --service your-service

このコマンドは、以下のファイルを生成します：

  • ecspresso.yml - ecspressoの設定ファイル
  • ecs-service-def.json - ECSサービス定義
  • ecs-task-def.json - ECSタスク定義

設定ファイルの構造

ecspresso.ymlは、ecspressoの動作を制御する主要な設定ファイルです。基本的な構造は以下の通りです：

region: ap-northeast-1
cluster: your-cluster
service: your-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
plugins:
  - name: tfstate
    config:
      path: terraform.tfstate

主な設定項目：

  • region: AWSリージョン
  • cluster: ECSクラスター名
  • service: ECSサービス名
  • service_definition: サービス定義ファイルのパス
  • task_definition: タスク定義ファイルのパス
  • timeout: 操作のタイムアウト時間
  • plugins: 使用するプラグインの設定

基本的なワークフロー

flowchart LR Init[初期化\ninit] --> Edit[設定編集] Edit --> Verify[検証\nverify] Verify --> Diff[差分確認\ndiff] Diff --> Deploy[デプロイ\ndeploy] Deploy --> Status[状態確認\nstatus]

1. サービスの状態確認

現在のサービスの状態を確認します。

$ ecspresso status

2. 設定の検証

デプロイ前に設定が正しいか検証します。

$ ecspresso verify

3. 変更内容の確認

現在のデプロイ済みサービスと、ローカルの設定ファイルの差分を確認します。

$ ecspresso diff

4. サービスのデプロイ

サービスをデプロイします。

$ ecspresso deploy

5. デプロイ状態の確認

デプロイ後のサービスの状態を確認します。

$ ecspresso status

ドライランモード

実際に変更を適用せずに、どのような変更が行われるかを確認するには、--dry-runオプションを使用します。

$ ecspresso deploy --dry-run

タスク定義の更新

タスク定義を更新する場合は、ecs-task-def.jsonファイルを編集し、再度デプロイします。

$ vi ecs-task-def.json
$ ecspresso deploy

サービス定義の更新

サービス定義を更新する場合は、ecs-service-def.jsonファイルを編集し、再度デプロイします。

$ vi ecs-service-def.json
$ ecspresso deploy
