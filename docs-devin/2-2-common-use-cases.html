ブルー/グリーンデプロイの実施

AWS CodeDeployを使用したブルー/グリーンデプロイを行うには、まず設定ファイルでデプロイメントコントローラーを指定します。

# ecs-service-def.json
{
  "deploymentController": {
    "type": "CODE_DEPLOY"
  },
  ...
}

次に、AppSpecの設定をecspresso.ymlに追加します。

# ecspresso.yml
appspec:
  hooks:
    BeforeInstall:
      - location: scripts/before_install.sh
        timeout: 300
    AfterInstall:
      - location: scripts/after_install.sh
        timeout: 300

デプロイを実行すると、ecspressoはCodeDeployを使用してブルー/グリーンデプロイを行います。

$ ecspresso deploy

サービスのスケーリング

サービスのタスク数を変更するには、以下のコマンドを使用します。

$ ecspresso scale --tasks 10

これは、タスク定義を更新せずにサービスのタスク数だけを変更します。

サービスのリフレッシュ

タスク定義を変更せずに、すべてのタスクを再起動するには、以下のコマンドを使用します。

$ ecspresso refresh

これは、--force-new-deploymentオプションを使用してサービスを更新します。

タスク内でのコマンド実行

実行中のタスク内でコマンドを実行するには、以下のコマンドを使用します。

$ ecspresso exec --command /bin/bash

特定のタスクを指定する場合は、--taskオプションを使用します。

$ ecspresso exec --task task-id --command /bin/bash

サービスのロールバック

以前のタスク定義にロールバックするには、以下のコマンドを使用します。

$ ecspresso rollback

特定のリビジョンにロールバックする場合は、--revisionオプションを使用します。

$ ecspresso rollback --revision 123

環境変数の使用

環境変数ファイルを使用して、タスク定義の環境変数を設定できます。

# .env
DATABASE_URL=mysql://user:pass@hostname/dbname
API_KEY=secret_key

環境変数ファイルを指定してデプロイします。

$ ecspresso deploy --envfile .env

複数環境での設定管理

環境ごとに異なる設定ファイルを使用することができます。

$ ecspresso deploy --config ecspresso.prod.yml

より詳細な複数環境での運用方法については、4.2 複数環境での運用を参照してください。
