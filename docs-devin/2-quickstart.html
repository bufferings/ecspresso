前提条件

ecspressoを使用するには、以下が必要です：

  • AWS認証情報（アクセスキーとシークレットキー）
  • ECSクラスターへのアクセス権限
  • ECSサービスとタスク定義を操作する権限

AWS認証情報は、以下のいずれかの方法で設定できます：

  • 環境変数（AWS_ACCESS_KEY_IDとAWS_SECRET_ACCESS_KEY）
  • 共有認証情報ファイル（~/.aws/credentials）
  • IAMロール（EC2インスタンスまたはECSタスク）

初期化

既存のECSサービスからecspresso設定を初期化するには：

$ ecspresso init --region ap-northeast-1 --cluster your-cluster --service your-service

これにより、以下のファイルが生成されます：

  • ecspresso.yml: ecspressoの設定ファイル
  • ecs-service-def.json: ECSサービス定義
  • ecs-task-def.json: ECSタスク定義

デプロイ

サービスをデプロイするには：

$ ecspresso deploy --config ecspresso.yml

デプロイ前に差分を確認するには：

$ ecspresso diff

デプロイ前にリソースを検証するには：

$ ecspresso verify

基本ワークフロー

flowchart TD A[設定ファイル編集] --> B[差分確認] B --> C{差分あり?} C -->|はい| D[リソース検証] C -->|いいえ| H[完了] D --> E{検証成功?} E -->|はい| F[デプロイ] E -->|いいえ| A F --> G[デプロイ待機] G --> H[完了]

基本的なワークフローは以下の通りです：

  1. 設定ファイル（タスク定義やサービス定義）を編集します。
  2. ecspresso diffで変更内容を確認します。
  3. ecspresso verifyでリソースを検証します。
  4. ecspresso deployでサービスをデプロイします。
  5. サービスが安定するまで待機します。

設定ファイル

ecspressoの設定ファイル（ecspresso.yml）は、ECSサービスとタスク定義の管理に必要な情報を定義します。基本的な設定ファイルの例は以下の通りです：

region: ap-northeast-1
cluster: my-cluster
service: my-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
plugins:
  - name: tfstate
    config:
      path: terraform.tfstate

主な設定項目：

  • region: AWSリージョン
  • cluster: ECSクラスター名
  • service: ECSサービス名
  • service_definition: サービス定義ファイルのパス
  • task_definition: タスク定義ファイルのパス
  • timeout: 操作のタイムアウト時間
  • plugins: 使用するプラグインの設定

タスク定義

タスク定義ファイル（ecs-task-def.json）は、ECSタスクの実行方法を定義します。基本的なタスク定義の例は以下の通りです：

{
  "family": "my-app",
  "executionRoleArn": "arn:aws:iam::123456789012:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::123456789012:role/ecsTaskRole",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "app",
      "image": "123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:latest",
      "essential": true,
      "portMappings": [
        {
          "containerPort": 8080,
          "hostPort": 8080,
          "protocol": "tcp"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/my-app",
          "awslogs-region": "ap-northeast-1",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ],
  "requiresCompatibilities": [
    "FARGATE"
  ],
  "cpu": "256",
  "memory": "512"
}

タスク定義では、以下の項目を設定できます：

  • コンテナの定義（イメージ、ポート、環境変数など）
  • タスクのリソース要件（CPU、メモリ）
  • ネットワークモード
  • IAMロール
  • ログ設定

サービス定義

サービス定義ファイル（ecs-service-def.json）は、ECSサービスの実行方法を定義します。基本的なサービス定義の例は以下の通りです：

{
  "cluster": "my-cluster",
  "serviceName": "my-service",
  "desiredCount": 2,
  "launchType": "FARGATE",
  "networkConfiguration": {
    "awsvpcConfiguration": {
      "subnets": [
        "subnet-12345678",
        "subnet-87654321"
      ],
      "securityGroups": [
        "sg-12345678"
      ],
      "assignPublicIp": "ENABLED"
    }
  },
  "loadBalancers": [
    {
      "targetGroupArn": "arn:aws:elasticloadbalancing:ap-northeast-1:123456789012:targetgroup/my-tg/1234567890123456",
      "containerName": "app",
      "containerPort": 8080
    }
  ]
}

サービス定義では、以下の項目を設定できます：

  • デプロイ設定（タスク数、デプロイ方法）
  • ネットワーク設定（サブネット、セキュリティグループ）
  • ロードバランサー設定
  • Auto Scaling設定

基本コマンド

ecspressoの基本的なコマンドは以下の通りです：

設定の初期化

$ ecspresso init --region ap-northeast-1 --cluster my-cluster --service my-service

サービスのデプロイ

$ ecspresso deploy

サービスの状態確認

$ ecspresso status

設定の差分確認

$ ecspresso diff

リソースの検証

$ ecspresso verify

タスクの実行

$ ecspresso run

サービスのスケーリング

$ ecspresso scale --tasks 5

注意: コマンドの詳細なオプションについては、コマンドリファレンスを参照してください。

新しいサービスのデプロイ

新しいECSサービスをデプロイする手順は以下の通りです：

  1. 設定ファイルの初期化：

    $ ecspresso init --region ap-northeast-1 --cluster my-cluster --service my-service

  2. 生成されたタスク定義とサービス定義を編集します。

  3. リソースを検証します：

    $ ecspresso verify

  4. サービスをデプロイします：

    $ ecspresso deploy

ヒント: 初めてのデプロイ前に --dry-run オプションを使用して、実際の変更を確認することをお勧めします。

既存サービスの更新

既存のECSサービスを更新する手順は以下の通りです：

  1. タスク定義またはサービス定義を編集します。

  2. 変更内容を確認します：

    $ ecspresso diff

  3. リソースを検証します：

    $ ecspresso verify

  4. サービスを更新します：

    $ ecspresso deploy

注意: タスク定義のみを更新する場合は、--skip-service-update オプションを使用できます。

デプロイのロールバック

問題が発生した場合に、以前のバージョンにロールバックする手順は以下の通りです：

  1. 現在のサービスの状態を確認します：

    $ ecspresso status

  2. ロールバックを実行します：

    $ ecspresso rollback

  3. サービスが安定するまで待機します：

    $ ecspresso wait

ヒント: --deregister オプションを使用すると、ロールバック後に現在のタスク定義を登録解除できます。

一時的なタスクの実行

データベースマイグレーションやバッチ処理などの一時的なタスクを実行する手順は以下の通りです：

  1. タスクを実行します：

    $ ecspresso run

  2. 特定のコマンドを実行する場合は、オーバーライドを使用します：

    $ ecspresso run --overrides '{"containerOverrides":[{"name":"app","command":["rails","db:migrate"]}]}'

  3. タスクの実行状況を確認します：

    $ ecspresso tasks

ヒント: --watch オプションを使用すると、タスクの完了を待機し、ログを表示できます。

コンテナ内でのコマンド実行

実行中のコンテナ内でコマンドを実行する手順は以下の通りです：

  1. 実行中のタスクを確認します：

    $ ecspresso tasks

  2. コンテナ内でコマンドを実行します：

    $ ecspresso exec --id TASK_ID --container app -- /bin/bash

注意: exec コマンドを使用するには、タスクロールに ssmmessages:CreateControlChannel などの権限が必要です。

インタラクティブシェルを使用する例：

$ ecspresso exec --interactive -- /bin/bash

ポートフォワーディングを使用する例：

$ ecspresso exec --port-forward 8080:80 -- /bin/bash
