3.1 デプロイ関連コマンド

deploy コマンド

ECSサービスをデプロイします。新しいタスク定義を登録し、サービスを更新します。

基本的な使い方

$ ecspresso deploy [オプション]

主なオプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--tasks タスク数を指定 -1（変更なし）
--skip-task-definition 新しいタスク定義の登録をスキップ false
--revision --skip-task-definitionと共に使用する特定のリビジョン 0
--force-new-deployment 強制的に新しいデプロイメントを作成 false
--no-wait サービスの安定を待たない false
--wait-until 待機条件（stable/deployed） stable
--suspend-auto-scaling Auto Scalingを一時停止 -
--resume-auto-scaling Auto Scalingを再開 -
--rollback-events 指定イベント発生時にロールバック（CodeDeployのみ） ""
--no-update-service サービス属性の更新をスキップ false
--latest-task-definition 新しいタスク定義を登録せず最新のものを使用 false

デプロイフロー

flowchart TD Start[開始] --> CheckService{サービスが存在?} CheckService -->|Yes| LoadTD[タスク定義読み込み] CheckService -->|No| CreateService[サービス作成] LoadTD --> SkipTD{タスク定義スキップ?} SkipTD -->|Yes| UseExisting[既存のタスク定義使用] SkipTD -->|No| RegisterTD[新しいタスク定義登録] UseExisting --> UpdateService[サービス更新] RegisterTD --> UpdateService UpdateService --> Wait{待機する?} Wait -->|Yes| WaitStable[安定状態まで待機] Wait -->|No| End[終了] WaitStable --> End

使用例

$ ecspresso deploy --tasks 10 --force-new-deployment

タスク数を10に設定し、強制的に新しいデプロイメントを作成します。

status コマンド

ECSサービスの現在の状態を表示します。

基本的な使い方

$ ecspresso status [オプション]

主なオプション

オプション 説明 デフォルト値
--events 表示するイベント数 10

使用例

$ ecspresso status --events 20

サービスの状態と直近20件のイベントを表示します。

rollback コマンド

以前のタスク定義にロールバックします。

基本的な使い方

$ ecspresso rollback [オプション]

主なオプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--deregister-task-definition ロールバック後に現在のタスク定義を登録解除 false
--no-wait サービスの安定を待たない false

使用例

$ ecspresso rollback --deregister-task-definition

以前のタスク定義にロールバックし、現在のタスク定義を登録解除します。
