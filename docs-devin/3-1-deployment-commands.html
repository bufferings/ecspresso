deploy

ECSサービスをデプロイします。新しいタスク定義を登録し、サービスを更新します。

オプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--tasks タスクの希望数 -1 (現在の値を維持)
--skip-task-definition 新しいタスク定義の登録をスキップ false
--revision --skip-task-definitionを使用する場合のタスク定義のリビジョン 0
--force-new-deployment サービスの強制的な新しいデプロイメントを行う false
--wait/--no-wait サービスが安定するまで待機するかどうか true
--wait-until 待機する状態（stable/deployed） stable
--suspend-auto-scaling ECSサービスに関連付けられたアプリケーションのオートスケーリングを一時停止 -
--resume-auto-scaling ECSサービスに関連付けられたアプリケーションのオートスケーリングを再開 -
--auto-scaling-min アプリケーションのオートスケーリングの最小容量を設定 -
--auto-scaling-max アプリケーションのオートスケーリングの最大容量を設定 -
--rollback-events 指定されたイベントが発生した場合にロールバックする（CodeDeployのみ） -
--update-service/--no-update-service サービス定義によってサービス属性を更新するかどうか true
--latest-task-definition 新しいタスク定義を登録せずに最新のタスク定義でデプロイ false

使用例

$ ecspresso deploy
$ ecspresso deploy --dry-run
$ ecspresso deploy --tasks 10
$ ecspresso deploy --skip-task-definition --force-new-deployment
$ ecspresso deploy --rollback-events DEPLOYMENT_FAILURE

rollback

ECSサービスを以前のタスク定義にロールバックします。

オプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--deregister ロールバック後に現在のタスク定義を登録解除 false
--revision ロールバックするタスク定義のリビジョン 0 (1つ前のリビジョン)
--wait/--no-wait サービスが安定するまで待機するかどうか true

使用例

$ ecspresso rollback
$ ecspresso rollback --revision 123
$ ecspresso rollback --deregister

refresh

ECSサービスをリフレッシュします。タスク定義を更新せずに、すべてのタスクを再起動します。

オプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--wait/--no-wait サービスが安定するまで待機するかどうか true

使用例

$ ecspresso refresh
$ ecspresso refresh --dry-run
$ ecspresso refresh --no-wait

scale

ECSサービスのタスク数を変更します。

オプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--tasks タスクの希望数 -1 (現在の値を維持)
--wait/--no-wait サービスが安定するまで待機するかどうか true

使用例

$ ecspresso scale --tasks 10
$ ecspresso scale --tasks 0 --dry-run

delete

ECSサービスを削除します。

オプション

オプション 説明 デフォルト値
--dry-run 実際の変更を行わずに実行内容を表示 false
--force 確認なしで削除を実行 false

使用例

$ ecspresso delete --force
$ ecspresso delete --dry-run
