3.3 設定・検証コマンド

init コマンド

ecspressoプロジェクトを初期化し、設定ファイルとサービス/タスク定義ファイルを生成します。

基本的な使い方

$ ecspresso init [オプション]

主なオプション

オプション 説明 デフォルト値
--region AWSリージョン 環境変数またはAWS設定
--cluster ECSクラスター名 -
--service ECSサービス名 -
--task-definition タスク定義名 -
--config 設定ファイル名 ecspresso.yml
--service-definition サービス定義ファイル名 ecs-service-def.json
--task-definition-path タスク定義ファイル名 ecs-task-def.json
--skip-task-definition タスク定義の取得をスキップ false
--update 既存の設定ファイルを更新 false
--format 設定ファイルのフォーマット（yaml/json） yaml

使用例

$ ecspresso init --region ap-northeast-1 --cluster my-cluster --service my-service

指定したクラスターとサービスの設定ファイルを生成します。

verify コマンド

設定ファイルとリソースの検証を行います。デプロイ前に問題がないか確認するのに便利です。

基本的な使い方

$ ecspresso verify [オプション]

主なオプション

オプション 説明 デフォルト値
--task-definition 検証するタスク定義ファイル 設定ファイルの値
--secrets シークレットの存在を検証 true
--cache 検証結果をキャッシュ true
--save-stopped 停止したタスクの情報を保存 false

使用例

$ ecspresso verify --task-definition=custom-task.json

カスタムタスク定義ファイルの検証を行います。

diff コマンド

ローカルの設定と実際にデプロイされているリソースの差分を表示します。

基本的な使い方

$ ecspresso diff [オプション]

主なオプション

オプション 説明 デフォルト値
--config-only 設定ファイルのみを比較 false
--unified unified形式で差分を表示 false
--color 差分の色付け表示 auto
--task-definition 比較するタスク定義ファイル 設定ファイルの値

使用例

$ ecspresso diff --unified --color=always

unified形式で色付きの差分を表示します。

render コマンド

タスク定義やサービス定義をレンダリングして表示します。プラグインやテンプレート変数が適用された結果を確認できます。

基本的な使い方

$ ecspresso render [オプション]

主なオプション

オプション 説明 デフォルト値
--task-definition レンダリングするタスク定義ファイル 設定ファイルの値
--service-definition レンダリングするサービス定義ファイル 設定ファイルの値
--output 出力形式（json/yaml） json

使用例

$ ecspresso render --output=yaml

タスク定義とサービス定義をYAML形式でレンダリングして表示します。

wait コマンド

サービスが安定状態になるまで待機します。

基本的な使い方

$ ecspresso wait [オプション]

主なオプション

オプション 説明 デフォルト値
--timeout タイムアウト時間 設定ファイルの値
--until 待機条件（stable/deployed） stable

使用例

$ ecspresso wait --timeout=5m

サービスが安定状態になるまで最大5分間待機します。
