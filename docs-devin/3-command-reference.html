コマンド構造

ecspressoのコマンドは以下の構造に従います：

$ ecspresso [グローバルオプション] コマンド [コマンドオプション]

例えば、以下のようにコマンドを実行します：

$ ecspresso --config ecspresso.yml deploy --tasks 2

この例では：

  • --config ecspresso.yml: グローバルオプション
  • deploy: コマンド
  • --tasks 2: コマンドオプション

グローバルオプション

すべてのコマンドで使用できるグローバルオプションは以下の通りです：

オプション 説明 デフォルト値
--config FILE 設定ファイルのパス ecspresso.yml
--debug デバッグログを有効にする false
--envfile FILE 環境変数ファイルのパス -
--help, -h ヘルプを表示する -
--profile NAME AWS認証情報のプロファイル名 -
--region REGION AWSリージョン 設定ファイルの値
--version, -v バージョン情報を表示する -

コマンドカテゴリ

ecspressoのコマンドは以下のカテゴリに分類されます：

ecspressoの主要機能
  • デプロイ管理
  • タスク実行
  • 設定検証

デプロイ関連コマンド

  • deploy: サービスをデプロイします
  • rollback: 以前のバージョンに戻します
  • scale: サービスのタスク数を変更します
  • wait: サービスが安定するまで待機します

タスク関連コマンド

  • run: 一時的なタスクを実行します
  • register: タスク定義を登録します
  • tasks: タスクの一覧を表示します
  • exec: タスク内でコマンドを実行します

設定・検証コマンド

  • init: 設定ファイルを初期化します
  • verify: リソースを検証します
  • diff: 設定の差分を表示します
  • render: テンプレートをレンダリングします
  • status: サービスの状態を表示します
  • appspec: AppSpecを生成します

デプロイ関連コマンド

ECSサービスのデプロイ、ロールバック、スケーリングを管理するコマンドです。

タスク関連コマンド

ECSタスクの実行、コマンド実行、タスク管理を行うコマンドです。

設定・検証コマンド

設定の初期化、検証、差分確認などを行うコマンドです。

デプロイ関連コマンド

デプロイ関連コマンドは、ECSサービスのデプロイ、ロールバック、スケーリング、および待機を管理するためのコマンドです。これらのコマンドを使用して、アプリケーションの安全かつ効率的なデプロイメントを実現できます。

deployコマンド

deployコマンドは、ECSサービスをデプロイするために使用します。タスク定義の登録とサービスの更新を行います。

基本的な使い方

ecspresso deploy [オプション]

主なオプション

オプション 説明 デフォルト値
--dry-run 実際にデプロイせずに、実行されるアクションを表示する false
--force-new-deployment 新しいデプロイメントを強制する false
--no-wait-until-stable サービスが安定するまで待機しない false
--wait-until 待機条件（deployed または stable） stable
--suspend-auto-scaling デプロイ中にApplication Auto Scalingを一時停止する false
--update-service サービスの更新のみを行う（タスク定義は更新しない） false
--skip-task-definition タスク定義の登録をスキップする false
--revision 使用するタスク定義のリビジョン -
--latest-task-definition 最新のタスク定義を使用する false
--rollback-events 自動ロールバックをトリガーするイベント -
--tasks デプロイするタスク数 -

処理フロー

deployコマンドの基本的な処理フロー：

デプロイプロセスの流れ
  1. 設定ファイルを読み込む
  2. タスク定義ファイルを読み込む
  3. サービス定義ファイルを読み込む
  4. タスク定義をAWSに登録する
  5. サービスを更新する
  6. サービスの状態を確認する
  7. サービスが安定するまで待機する

使用例

# 基本的なデプロイ
ecspresso deploy

# ドライランモード（実際にデプロイしない）
ecspresso deploy --dry-run

# 新しいデプロイメントを強制する
ecspresso deploy --force-new-deployment

# サービスが安定するまで待機しない
ecspresso deploy --no-wait-until-stable

# デプロイ中にApplication Auto Scalingを一時停止する
ecspresso deploy --suspend-auto-scaling

# 特定のリビジョンのタスク定義を使用する
ecspresso deploy --revision 10

# 最新のタスク定義を使用する
ecspresso deploy --latest-task-definition

# タスク数を指定してデプロイする
ecspresso deploy --tasks 5

rollbackコマンド

rollbackコマンドは、ECSサービスを以前のバージョンにロールバックするために使用します。

基本的な使い方

ecspresso rollback [オプション]

主なオプション

オプション 説明 デフォルト値
--dry-run 実際にロールバックせずに、実行されるアクションを表示する false
--deregister ロールバック後に現在のタスク定義を登録解除する false
--no-wait-until-stable サービスが安定するまで待機しない false
--revision ロールバックするタスク定義のリビジョン -

ロールバックプロセス

rollbackコマンドの内部処理フロー：

ロールバックプロセスの流れ
  1. 設定ファイルを読み込む
  2. 現在のサービス情報を取得する
  3. 前回のタスク定義を取得する
  4. 前回のタスク定義でサービスを更新する
  5. サービスの状態を確認する
  6. サービスが安定するまで待機する
  7. （--deregisterオプション指定時）現在のタスク定義を登録解除する

使用例

# 基本的なロールバック（前回のタスク定義に戻す）
ecspresso rollback

# ドライランモード（実際にロールバックしない）
ecspresso rollback --dry-run

# 特定のリビジョンにロールバックする
ecspresso rollback --revision 15

# ロールバック後に現在のタスク定義を登録解除する
ecspresso rollback --deregister

# サービスが安定するまで待機しない
ecspresso rollback --no-wait-until-stable

scaleコマンド

scaleコマンドは、ECSサービスのタスク数を変更するために使用します。

基本的な使い方

ecspresso scale --tasks N [オプション]

主なオプション

オプション 説明 デフォルト値
--tasks 設定するタスク数（必須） -
--dry-run 実際にスケールせずに、実行されるアクションを表示する false
--no-wait-until-stable サービスが安定するまで待機しない false

スケーリングプロセス

scaleコマンドの内部処理フロー：

スケーリングプロセスの流れ
  1. 設定ファイルを読み込む
  2. 現在のサービス情報を取得する
  3. タスク数を変更してサービスを更新する
  4. サービスの状態を確認する
  5. サービスが安定するまで待機する

使用例

# タスク数を5に設定
ecspresso scale --tasks 5

# ドライランモード（実際にスケールしない）
ecspresso scale --tasks 10 --dry-run

# タスク数を0に設定（サービスを停止）
ecspresso scale --tasks 0

# サービスが安定するまで待機しない
ecspresso scale --tasks 3 --no-wait-until-stable

waitコマンド

waitコマンドは、ECSサービスが安定するまで待機するために使用します。

基本的な使い方

ecspresso wait [オプション]

主なオプション

オプション 説明 デフォルト値
--timeout タイムアウト時間（秒） 600
--wait-until 待機条件（deployed または stable） stable

待機プロセス

waitコマンドの内部処理フロー：

待機プロセスの流れ
  1. 設定ファイルを読み込む
  2. サービスの状態を取得する
  3. タイムアウトまで以下を繰り返す：
    • サービスが安定している場合：待機終了
    • サービスが安定していない場合：一定時間待機して状態を再取得
  4. タイムアウト発生時：エラー終了

使用例

# サービスが安定するまで待機
ecspresso wait

# デプロイメントが完了するまで待機
ecspresso wait --wait-until deployed

# タイムアウト時間を300秒に設定
ecspresso wait --timeout 300

デプロイ戦略

ecspressoは以下の2つのデプロイ戦略をサポートしています：

デプロイ戦略の比較
ECSローリングデプロイ
  • タスクを徐々に置き換える
  • deploymentConfiguration設定で制御
  • サーキットブレーカー機能あり
CodeDeployブルー/グリーン
  • 新環境を作成
  • トラフィックを徐々に移行
  • appspec.yaml設定で制御
  • デプロイフック機能あり

ECSローリングデプロイ

ECSローリングデプロイは、タスクを徐々に置き換えていく戦略です。以下のパラメータで設定できます：

# ecspresso.yml
service_definition: ecs-service-def.json

# ecs-service-def.json
{
  "deploymentConfiguration": {
    "maximumPercent": 200,
    "minimumHealthyPercent": 100,
    "deploymentCircuitBreaker": {
      "enable": true,
      "rollback": true
    }
  }
}

CodeDeployブルー/グリーンデプロイ

CodeDeployブルー/グリーンデプロイは、新しい環境（グリーン）を作成し、トラフィックを徐々に移行する戦略です。以下のパラメータで設定できます：

# ecspresso.yml
service_definition: ecs-service-def.json
appspec: appspec.yaml

# ecs-service-def.json
{
  "deploymentController": {
    "type": "CODE_DEPLOY"
  }
}

# appspec.yaml
version: 0.0
Resources:
  - TargetService:
      Type: AWS::ECS::Service
      Properties:
        TaskDefinition: "<TASK_DEFINITION>"
        LoadBalancerInfo:
          ContainerName: "web"
          ContainerPort: 80
        PlatformVersion: "1.4.0"
Hooks:
  - BeforeInstall: "LambdaFunctionToValidateBeforeInstall"
  - AfterInstall: "LambdaFunctionToValidateAfterInstall"
  - AfterAllowTestTraffic: "LambdaFunctionToValidateAfterTestTrafficStarts"
  - BeforeAllowTraffic: "LambdaFunctionToValidateBeforeAllowingProductionTraffic"
  - AfterAllowTraffic: "LambdaFunctionToValidateAfterAllowingProductionTraffic"
