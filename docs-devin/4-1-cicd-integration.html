4.1 CI/CDパイプラインとの統合

ecspressoは、CI/CDパイプラインと簡単に統合できるように設計されています。このセクションでは、一般的なCI/CDツールでecspressoを使用する方法を説明します。

CI/CDパイプラインの基本フロー

flowchart LR Code[コード変更] --> Build[ビルド] Build --> Test[テスト] Test --> Package[パッケージング] Package --> Verify[検証\necspresso verify] Verify --> Deploy[デプロイ\necspresso deploy] Deploy --> Monitor[モニタリング]

GitHub Actionsでの統合

GitHub Actionsを使用してecspressoをCI/CDパイプラインに統合する例を示します。

name: Deploy to ECS

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v1
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ap-northeast-1
      
      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v1
      
      - name: Build, tag, and push image to Amazon ECR
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          ECR_REPOSITORY: my-app
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
      
      - name: Update task definition
        run: |
          # タスク定義内のイメージタグを更新
          sed -i "s|{{IMAGE_TAG}}|${{ github.sha }}|g" ecs-task-def.json
      
      - name: Install ecspresso
        run: |
          curl -sL https://github.com/kayac/ecspresso/releases/latest/download/ecspresso-linux-amd64 -o ecspresso
          chmod +x ecspresso
          sudo mv ecspresso /usr/local/bin/
      
      - name: Verify configuration
        run: ecspresso verify
      
      - name: Deploy to ECS
        run: ecspresso deploy

CircleCIでの統合

CircleCIを使用してecspressoをCI/CDパイプラインに統合する例を示します。

version: 2.1
orbs:
  aws-ecr: circleci/aws-ecr@6.15.3
  aws-cli: circleci/aws-cli@2.0.3

jobs:
  deploy:
    docker:
      - image: cimg/base:2021.04
    steps:
      - checkout
      - setup_remote_docker:
          version: 20.10.7
      - aws-cli/setup:
          profile-name: default
      - aws-ecr/build-and-push-image:
          repo: my-app
          tag: ${CIRCLE_SHA1}
      - run:
          name: Update task definition
          command: |
            # タスク定義内のイメージタグを更新
            sed -i "s|{{IMAGE_TAG}}|${CIRCLE_SHA1}|g" ecs-task-def.json
      - run:
          name: Install ecspresso
          command: |
            curl -sL https://github.com/kayac/ecspresso/releases/latest/download/ecspresso-linux-amd64 -o ecspresso
            chmod +x ecspresso
            sudo mv ecspresso /usr/local/bin/
      - run:
          name: Verify configuration
          command: ecspresso verify
      - run:
          name: Deploy to ECS
          command: ecspresso deploy

workflows:
  version: 2
  build-and-deploy:
    jobs:
      - deploy:
          filters:
            branches:
              only: main

AWS CodePipelineでの統合

AWS CodePipelineを使用してecspressoをCI/CDパイプラインに統合する例を示します。

# buildspec.yml
version: 0.2

phases:
  install:
    runtime-versions:
      docker: 19
    commands:
      - curl -sL https://github.com/kayac/ecspresso/releases/latest/download/ecspresso-linux-amd64 -o ecspresso
      - chmod +x ecspresso
      - mv ecspresso /usr/local/bin/
  
  pre_build:
    commands:
      - echo Logging in to Amazon ECR...
      - aws ecr get-login-password --region $AWS_DEFAULT_REGION | docker login --username AWS --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com
  
  build:
    commands:
      - echo Building the Docker image...
      - docker build -t $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com/$ECR_REPOSITORY:$CODEBUILD_RESOLVED_SOURCE_VERSION .
      - docker push $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com/$ECR_REPOSITORY:$CODEBUILD_RESOLVED_SOURCE_VERSION
      
      - echo Updating task definition...
      - sed -i "s|{{IMAGE_TAG}}|$CODEBUILD_RESOLVED_SOURCE_VERSION|g" ecs-task-def.json
  
  post_build:
    commands:
      - echo Verifying configuration...
      - ecspresso verify
      - echo Deploying to ECS...
      - ecspresso deploy

ベストプラクティス

1. 環境変数の活用

CI/CD環境では、環境変数を使用して設定を柔軟に変更できます。

$ ecspresso --config=config.$ENV.yml deploy

2. ドライランの実施

本番環境へのデプロイ前に、ドライランを実行して変更内容を確認します。

$ ecspresso deploy --dry-run

3. 検証ステップの追加

デプロイ前に必ず検証を行い、設定の問題を早期に発見します。

$ ecspresso verify || exit 1

4. デプロイ後の確認

デプロイ後にサービスの状態を確認し、問題があればすぐに対応できるようにします。

$ ecspresso deploy && ecspresso status

5. ロールバック戦略の準備

デプロイに問題があった場合のロールバック手順を準備しておきます。

if ! ecspresso deploy; then
  echo "Deployment failed, rolling back..."
  ecspresso rollback
  exit 1
fi

セキュリティ考慮事項

1. AWS認証情報の管理

CI/CD環境では、AWS認証情報をシークレットとして安全に管理します。環境変数として直接スクリプトに記述しないでください。

2. 最小権限の原則

CI/CDパイプラインで使用するIAMロールには、必要最小限の権限のみを付与します。

3. シークレット管理

タスク定義内のシークレットは、AWS Secrets ManagerやSSMパラメータストアを使用して管理します。
