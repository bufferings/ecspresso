CI/CDパイプラインの概要

ecspressoは、CI/CDパイプラインと簡単に統合できるように設計されています。一般的なCI/CDツール（GitHub Actions、CircleCI、AWS CodePipelineなど）と組み合わせて使用することで、ECSサービスの継続的なデプロイを自動化できます。

graph LR
    A[コード変更] --> B[ビルド]
    B --> C[テスト]
    C --> D[イメージ作成]
    D --> E[ecspresso verify]
    E --> F[ecspresso deploy]
    F --> G[デプロイ完了]

GitHub Actionsとの統合

GitHub Actionsを使用したCI/CDパイプラインの例を以下に示します。

# .github/workflows/deploy.yml
name: Deploy to ECS

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v1
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ap-northeast-1
      
      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v1
      
      - name: Build, tag, and push image to Amazon ECR
        id: build-image
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          ECR_REPOSITORY: my-app
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
          echo "image=$ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG" >> $GITHUB_OUTPUT
      
      - name: Install ecspresso
        run: |
          curl -sL https://github.com/kayac/ecspresso/releases/latest | \
          grep -o '/kayac/ecspresso/releases/download/v[0-9.]*/ecspresso_.*_linux_amd64.zip' | \
          xargs -I{} curl -sL https://github.com{} -o ecspresso.zip
          unzip ecspresso.zip
          sudo mv ecspresso /usr/local/bin/
      
      - name: Update task definition
        run: |
          # 環境変数を設定
          export IMAGE_URI=${{ steps.build-image.outputs.image }}
          
          # タスク定義を更新
          cat ecs-task-def.json | sed "s|{{IMAGE_URI}}|$IMAGE_URI|g" > ecs-task-def.updated.json
          mv ecs-task-def.updated.json ecs-task-def.json
      
      - name: Verify and deploy
        run: |
          ecspresso verify
          ecspresso deploy

CircleCIとの統合

CircleCIを使用したCI/CDパイプラインの例を以下に示します。

# .circleci/config.yml
version: 2.1
jobs:
  build-and-deploy:
    docker:
      - image: cimg/base:2021.04
    steps:
      - checkout
      - setup_remote_docker:
          version: 20.10.7
      - run:
          name: Install AWS CLI
          command: |
            curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
            unzip awscliv2.zip
            sudo ./aws/install
      - run:
          name: Install ecspresso
          command: |
            curl -sL https://github.com/kayac/ecspresso/releases/latest | \
            grep -o '/kayac/ecspresso/releases/download/v[0-9.]*/ecspresso_.*_linux_amd64.zip' | \
            xargs -I{} curl -sL https://github.com{} -o ecspresso.zip
            unzip ecspresso.zip
            sudo mv ecspresso /usr/local/bin/
      - run:
          name: Configure AWS credentials
          command: |
            aws configure set aws_access_key_id ${AWS_ACCESS_KEY_ID}
            aws configure set aws_secret_access_key ${AWS_SECRET_ACCESS_KEY}
            aws configure set default.region ap-northeast-1
      - run:
          name: Login to Amazon ECR
          command: |
            aws ecr get-login-password --region ap-northeast-1 | docker login --username AWS --password-stdin ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com
      - run:
          name: Build and push Docker image
          command: |
            docker build -t ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:${CIRCLE_SHA1} .
            docker push ${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:${CIRCLE_SHA1}
      - run:
          name: Update task definition
          command: |
            export IMAGE_URI="${AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:${CIRCLE_SHA1}"
            cat ecs-task-def.json | sed "s|{{IMAGE_URI}}|$IMAGE_URI|g" > ecs-task-def.updated.json
            mv ecs-task-def.updated.json ecs-task-def.json
      - run:
          name: Verify and deploy
          command: |
            ecspresso verify
            ecspresso deploy

workflows:
  version: 2
  build-and-deploy:
    jobs:
      - build-and-deploy:
          filters:
            branches:
              only: main

AWS CodePipelineとの統合

AWS CodePipelineを使用したCI/CDパイプラインの例を以下に示します。この例では、CodeBuildプロジェクトでecspressoを使用しています。

# buildspec.yml
version: 0.2

phases:
  install:
    runtime-versions:
      docker: 19
    commands:
      - curl -sL https://github.com/kayac/ecspresso/releases/latest | grep -o '/kayac/ecspresso/releases/download/v[0-9.]*/ecspresso_.*_linux_amd64.zip' | xargs -I{} curl -sL https://github.com{} -o ecspresso.zip
      - unzip ecspresso.zip
      - mv ecspresso /usr/local/bin/
  pre_build:
    commands:
      - echo Logging in to Amazon ECR...
      - aws ecr get-login-password --region $AWS_DEFAULT_REGION | docker login --username AWS --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com
      - REPOSITORY_URI=$AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com/my-app
      - COMMIT_HASH=$(echo $CODEBUILD_RESOLVED_SOURCE_VERSION | cut -c 1-7)
      - IMAGE_TAG=${COMMIT_HASH:=latest}
  build:
    commands:
      - echo Build started on `date`
      - echo Building the Docker image...
      - docker build -t $REPOSITORY_URI:$IMAGE_TAG .
      - docker tag $REPOSITORY_URI:$IMAGE_TAG $REPOSITORY_URI:latest
  post_build:
    commands:
      - echo Build completed on `date`
      - echo Pushing the Docker image...
      - docker push $REPOSITORY_URI:$IMAGE_TAG
      - docker push $REPOSITORY_URI:latest
      - echo Update task definition...
      - export IMAGE_URI="$REPOSITORY_URI:$IMAGE_TAG"
      - cat ecs-task-def.json | sed "s|{{IMAGE_URI}}|$IMAGE_URI|g" > ecs-task-def.updated.json
      - mv ecs-task-def.updated.json ecs-task-def.json
      - echo Verify and deploy...
      - ecspresso verify
      - ecspresso deploy

artifacts:
  files:
    - appspec.yml
    - taskdef.json

デプロイ前の検証

CI/CDパイプラインでは、デプロイ前に設定を検証することが重要です。ecspressoのverifyコマンドを使用して、タスク定義やサービス定義が正しいことを確認できます。

ecspresso verify

また、diffコマンドを使用して、現在のデプロイと新しい設定の差分を確認することもできます。

ecspresso diff --unified

ロールバック戦略

デプロイに失敗した場合のロールバック戦略も重要です。ecspressoのrollbackコマンドを使用して、以前のタスク定義にロールバックできます。

ecspresso rollback

CI/CDパイプラインでは、デプロイ後のヘルスチェックを実施し、失敗した場合に自動的にロールバックするように設定することをお勧めします。

# デプロイ後のヘルスチェック例
if ! curl -s http://your-service-endpoint/health | grep -q "ok"; then
  echo "Health check failed. Rolling back..."
  ecspresso rollback
  exit 1
fi

セキュリティのベストプラクティス

CI/CDパイプラインでecspressoを使用する際のセキュリティのベストプラクティスを以下に示します。

  • AWSの認証情報は、CI/CDツールのシークレット管理機能を使用して安全に保存する
  • IAMロールには最小限の権限を付与する
  • 環境変数ファイルには機密情報を直接記述せず、AWS Systems Manager Parameter StoreやSecrets Managerを使用する
  • デプロイ前にverifyコマンドを実行して設定を検証する
  • 本番環境へのデプロイには承認ステップを追加する
