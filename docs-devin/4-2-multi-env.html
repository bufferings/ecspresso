複数環境の管理

多くのプロジェクトでは、開発（dev）、ステージング（staging）、本番（production）など、複数の環境でECSサービスを運用する必要があります。ecspressoを使用して、これらの環境を効率的に管理する方法を説明します。

graph TD
    A[共通設定] --> B[開発環境]
    A --> C[ステージング環境]
    A --> D[本番環境]
    B --> E[開発用デプロイ]
    C --> F[ステージング用デプロイ]
    D --> G[本番用デプロイ]

環境ごとの設定ファイル

環境ごとに異なる設定ファイルを用意することで、各環境に適した設定でデプロイできます。

ディレクトリ構造の例

project/
├── ecspresso/
│   ├── common/
│   │   ├── ecs-task-def.json.template
│   │   └── ecs-service-def.json.template
│   ├── dev/
│   │   ├── ecspresso.yml
│   │   ├── ecs-task-def.json
│   │   └── ecs-service-def.json
│   ├── staging/
│   │   ├── ecspresso.yml
│   │   ├── ecs-task-def.json
│   │   └── ecs-service-def.json
│   └── production/
│       ├── ecspresso.yml
│       ├── ecs-task-def.json
│       └── ecs-service-def.json
└── ...

環境ごとの設定ファイル例

開発環境の設定ファイル（ecspresso.yml）:

# ecspresso/dev/ecspresso.yml
region: ap-northeast-1
cluster: my-app-dev
service: my-service-dev
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m

ステージング環境の設定ファイル（ecspresso.yml）:

# ecspresso/staging/ecspresso.yml
region: ap-northeast-1
cluster: my-app-staging
service: my-service-staging
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m

本番環境の設定ファイル（ecspresso.yml）:

# ecspresso/production/ecspresso.yml
region: ap-northeast-1
cluster: my-app-production
service: my-service-production
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 15m

環境変数ファイルの活用

環境ごとに異なる環境変数を設定するために、環境変数ファイルを活用できます。

開発環境の環境変数ファイル（.env.dev）:

# .env.dev
DATABASE_URL=mysql://user:pass@dev-db.example.com/myapp
API_KEY=dev_api_key
LOG_LEVEL=debug

ステージング環境の環境変数ファイル（.env.staging）:

# .env.staging
DATABASE_URL=mysql://user:pass@staging-db.example.com/myapp
API_KEY=staging_api_key
LOG_LEVEL=info

本番環境の環境変数ファイル（.env.production）:

# .env.production
DATABASE_URL=mysql://user:pass@production-db.example.com/myapp
API_KEY=production_api_key
LOG_LEVEL=warn

環境変数ファイルを指定してデプロイ:

$ cd ecspresso/dev
$ ecspresso deploy --envfile ../../.env.dev

$ cd ../staging
$ ecspresso deploy --envfile ../../.env.staging

$ cd ../production
$ ecspresso deploy --envfile ../../.env.production

テンプレートの活用

共通のテンプレートを使用して、環境ごとの設定ファイルを生成することもできます。

タスク定義のテンプレート（ecs-task-def.json.template）:

{
  "family": "{{SERVICE_NAME}}",
  "executionRoleArn": "arn:aws:iam::{{AWS_ACCOUNT_ID}}:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::{{AWS_ACCOUNT_ID}}:role/{{TASK_ROLE}}",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "app",
      "image": "{{ECR_REPOSITORY}}:{{IMAGE_TAG}}",
      "essential": true,
      "portMappings": [
        {
          "containerPort": 80,
          "hostPort": 80,
          "protocol": "tcp"
        }
      ],
      "environment": [
        {
          "name": "ENV",
          "value": "{{ENVIRONMENT}}"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/{{SERVICE_NAME}}",
          "awslogs-region": "{{AWS_REGION}}",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ],
  "requiresCompatibilities": [
    "FARGATE"
  ],
  "cpu": "{{CPU}}",
  "memory": "{{MEMORY}}"
}

テンプレートから環境ごとの設定ファイルを生成するスクリプト例:

#!/bin/bash

# 環境を指定
ENV=$1

# 環境ごとの変数を設定
case $ENV in
  dev)
    AWS_ACCOUNT_ID="123456789012"
    SERVICE_NAME="my-service-dev"
    TASK_ROLE="ecsTaskRole-dev"
    ECR_REPOSITORY="123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app"
    IMAGE_TAG="latest"
    ENVIRONMENT="development"
    AWS_REGION="ap-northeast-1"
    CPU="256"
    MEMORY="512"
    ;;
  staging)
    AWS_ACCOUNT_ID="123456789012"
    SERVICE_NAME="my-service-staging"
    TASK_ROLE="ecsTaskRole-staging"
    ECR_REPOSITORY="123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app"
    IMAGE_TAG="staging"
    ENVIRONMENT="staging"
    AWS_REGION="ap-northeast-1"
    CPU="512"
    MEMORY="1024"
    ;;
  production)
    AWS_ACCOUNT_ID="123456789012"
    SERVICE_NAME="my-service-production"
    TASK_ROLE="ecsTaskRole-production"
    ECR_REPOSITORY="123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app"
    IMAGE_TAG="production"
    ENVIRONMENT="production"
    AWS_REGION="ap-northeast-1"
    CPU="1024"
    MEMORY="2048"
    ;;
  *)
    echo "Unknown environment: $ENV"
    exit 1
    ;;
esac

# テンプレートから設定ファイルを生成
cat ecspresso/common/ecs-task-def.json.template | \
  sed "s/{{AWS_ACCOUNT_ID}}/$AWS_ACCOUNT_ID/g" | \
  sed "s/{{SERVICE_NAME}}/$SERVICE_NAME/g" | \
  sed "s/{{TASK_ROLE}}/$TASK_ROLE/g" | \
  sed "s/{{ECR_REPOSITORY}}/$ECR_REPOSITORY/g" | \
  sed "s/{{IMAGE_TAG}}/$IMAGE_TAG/g" | \
  sed "s/{{ENVIRONMENT}}/$ENVIRONMENT/g" | \
  sed "s/{{AWS_REGION}}/$AWS_REGION/g" | \
  sed "s/{{CPU}}/$CPU/g" | \
  sed "s/{{MEMORY}}/$MEMORY/g" > ecspresso/$ENV/ecs-task-def.json

echo "Generated task definition for $ENV environment"

AWS Systems Manager Parameter Storeの活用

機密情報や環境固有の設定値は、AWS Systems Manager Parameter Storeに保存し、タスク定義から参照することをお勧めします。

タスク定義での参照例:

{
  "containerDefinitions": [
    {
      "name": "app",
      "image": "...",
      "secrets": [
        {
          "name": "DATABASE_URL",
          "valueFrom": "arn:aws:ssm:ap-northeast-1:123456789012:parameter/myapp/{{ENVIRONMENT}}/database_url"
        },
        {
          "name": "API_KEY",
          "valueFrom": "arn:aws:ssm:ap-northeast-1:123456789012:parameter/myapp/{{ENVIRONMENT}}/api_key"
        }
      ]
    }
  ]
}

環境間のプロモーション

アプリケーションを開発環境からステージング環境、そして本番環境へとプロモーションする際のワークフローを以下に示します。

sequenceDiagram
    participant Dev as 開発環境
    participant Staging as ステージング環境
    participant Prod as 本番環境
    
    Dev->>Dev: 開発とテスト
    Dev->>Staging: イメージをプロモーション
    Staging->>Staging: 統合テスト
    Staging->>Prod: イメージをプロモーション
    Prod->>Prod: 本番デプロイ

イメージのプロモーションスクリプト例:

#!/bin/bash

# 引数からソース環境とターゲット環境を取得
SOURCE_ENV=$1
TARGET_ENV=$2

# ECRリポジトリ
ECR_REPOSITORY="123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app"

# ソース環境のイメージタグ
SOURCE_TAG="${SOURCE_ENV}"

# ターゲット環境のイメージタグ
TARGET_TAG="${TARGET_ENV}"

# イメージをプル
docker pull ${ECR_REPOSITORY}:${SOURCE_TAG}

# イメージにターゲット環境のタグを付ける
docker tag ${ECR_REPOSITORY}:${SOURCE_TAG} ${ECR_REPOSITORY}:${TARGET_TAG}

# イメージをプッシュ
docker push ${ECR_REPOSITORY}:${TARGET_TAG}

echo "Promoted image from ${SOURCE_ENV} to ${TARGET_ENV}"

# ターゲット環境にデプロイ
cd ecspresso/${TARGET_ENV}
ecspresso deploy --envfile ../../.env.${TARGET_ENV}

環境ごとのスケーリング設定

環境ごとに異なるスケーリング設定を適用することもできます。

開発環境のスケーリング設定例:

$ cd ecspresso/dev
$ ecspresso deploy --tasks 1

ステージング環境のスケーリング設定例:

$ cd ecspresso/staging
$ ecspresso deploy --tasks 2

本番環境のスケーリング設定例:

$ cd ecspresso/production
$ ecspresso deploy --tasks 5

Auto Scalingを設定する例:

# 本番環境のAuto Scaling設定
$ cd ecspresso/production
$ ecspresso deploy --auto-scaling-min 3 --auto-scaling-max 10
