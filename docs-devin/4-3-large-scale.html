4.3 大規模サービスの管理

大規模なサービスでは、多数のECSサービスやタスク定義を管理する必要があります。このセクションでは、ecspressoを使用して大規模サービスを効率的に管理する方法を説明します。

1. サービスごとの設定ファイル

マイクロサービスアーキテクチャでは、各サービスごとに別々の設定ファイルを作成します。

# ecspresso.service1.yml
region: ap-northeast-1
cluster: microservices-cluster
service: auth-service
service_definition: auth-service-def.json
task_definition: auth-service-task-def.json
timeout: 10m

# ecspresso.service2.yml
region: ap-northeast-1
cluster: microservices-cluster
service: user-service
service_definition: user-service-def.json
task_definition: user-service-task-def.json
timeout: 10m

2. デプロイスクリプトの作成

複数のサービスを一括でデプロイするスクリプトを作成します。

#!/bin/bash
# 全サービスをデプロイするスクリプト

SERVICES=("auth" "user" "payment" "notification")

for service in "${SERVICES[@]}"; do
  echo "Deploying $service service..."
  ecspresso --config=ecspresso.$service.yml deploy --no-wait
done

# すべてのサービスが安定するまで待機
for service in "${SERVICES[@]}"; do
  echo "Waiting for $service service to stabilize..."
  ecspresso --config=ecspresso.$service.yml wait
done

echo "All services deployed successfully!"

リソース管理の最適化

1. タスク定義の最適化

大規模サービスでは、タスク定義を最適化して、リソース使用率を向上させることが重要です。

  • コンテナのサイジング: 各コンテナに適切なCPUとメモリを割り当てます。
  • リソース予約: 適切なリソース予約を設定して、オーバーコミットを防ぎます。
  • ヘルスチェック: 適切なヘルスチェックを設定して、問題のあるタスクを早期に検出します。
{
  "containerDefinitions": [
    {
      "name": "web",
      "image": "web-service:latest",
      "cpu": 256,
      "memory": 512,
      "essential": true,
      "healthCheck": {
        "command": ["CMD-SHELL", "curl -f http://localhost:8080/health || exit 1"],
        "interval": 30,
        "timeout": 5,
        "retries": 3,
        "startPeriod": 60
      }
    }
  ]
}

2. Auto Scalingの設定

大規模サービスでは、Auto Scalingを設定して、トラフィックの変動に対応することが重要です。

# サービス定義内のAuto Scaling設定
{
  "deploymentConfiguration": {
    "maximumPercent": 200,
    "minimumHealthyPercent": 100
  },
  "desiredCount": 3,
  "enableECSManagedTags": true,
  "capacityProviderStrategy": [
    {
      "capacityProvider": "FARGATE_SPOT",
      "weight": 1
    }
  ]
}

# Blue/Greenデプロイの設定

大規模デプロイのベストプラクティス

1. 段階的デプロイ

大規模サービスでは、段階的にデプロイして、問題を早期に検出することが重要です。

#!/bin/bash
# 段階的デプロイスクリプト

# 最初に1つのサービスをデプロイしてテスト
echo "Deploying to canary service..."
ecspresso --config=ecspresso.canary.yml deploy
echo "Waiting for canary service to stabilize..."
ecspresso --config=ecspresso.canary.yml wait

# テストが成功したら、残りのサービスをデプロイ
echo "Canary deployment successful. Deploying to remaining services..."
for service in "${REMAINING_SERVICES[@]}"; do
  echo "Deploying $service service..."
  ecspresso --config=ecspresso.$service.yml deploy --no-wait
done

2. ロールバック計画

大規模サービスでは、問題が発生した場合に備えて、ロールバック計画を準備することが重要です。

#!/bin/bash
# ロールバックスクリプト

if [ "$1" == "all" ]; then
  # すべてのサービスをロールバック
  for service in "${ALL_SERVICES[@]}"; do
    echo "Rolling back $service service..."
    ecspresso --config=ecspresso.$service.yml rollback
  done
else
  # 特定のサービスをロールバック
  echo "Rolling back $1 service..."
  ecspresso --config=ecspresso.$1.yml rollback
fi

3. デプロイ履歴の管理

大規模サービスでは、デプロイ履歴を管理して、問題が発生した場合に原因を特定することが重要です。

#!/bin/bash
# デプロイ履歴を記録するスクリプト

DEPLOY_ID=$(date +%Y%m%d%H%M%S)
DEPLOY_LOG="deploy_history/$DEPLOY_ID.log"

mkdir -p deploy_history

echo "Starting deployment $DEPLOY_ID at $(date)" > $DEPLOY_LOG
echo "Git commit: $(git rev-parse HEAD)" >> $DEPLOY_LOG

# デプロイを実行
for service in "${ALL_SERVICES[@]}"; do
  echo "Deploying $service service..." | tee -a $DEPLOY_LOG
  ecspresso --config=ecspresso.$service.yml deploy --no-wait 2>&1 | tee -a $DEPLOY_LOG
done

echo "Deployment $DEPLOY_ID completed at $(date)" >> $DEPLOY_LOG
