5. トラブルシューティング

ecspressoを使用する際に発生する可能性のある一般的な問題とその解決方法を説明します。

一般的なエラーと解決策

1. AWS認証エラー

症状: 「Unable to locate credentials」や「Access denied」などのエラーが表示される。

解決策:

  • AWS CLIが正しく設定されていることを確認します。
  • AWS認証情報が正しく設定されていることを確認します。
  • IAMロールに必要な権限があることを確認します。
$ aws configure
$ export AWS_PROFILE=your-profile
$ aws sts get-caller-identity

2. サービス定義エラー

症状: 「Invalid service definition」などのエラーが表示される。

解決策:

  • サービス定義ファイルが有効なJSONまたはYAMLであることを確認します。
  • 必須フィールドがすべて含まれていることを確認します。
  • renderコマンドを使用して、レンダリングされたサービス定義を確認します。
$ ecspresso render --service-definition

3. タスク定義エラー

症状: 「Invalid task definition」などのエラーが表示される。

解決策:

  • タスク定義ファイルが有効なJSONまたはYAMLであることを確認します。
  • 必須フィールドがすべて含まれていることを確認します。
  • renderコマンドを使用して、レンダリングされたタスク定義を確認します。
$ ecspresso render --task-definition

4. デプロイタイムアウト

症状: デプロイが完了する前にタイムアウトする。

解決策:

  • 設定ファイルのタイムアウト値を増やします。
  • ヘルスチェックの設定を確認します。
  • --no-waitオプションを使用して、デプロイ後の待機をスキップします。
# ecspresso.yml
timeout: 20m

$ ecspresso deploy --no-wait

5. リソース検証エラー

症状: 「Resource validation failed」などのエラーが表示される。

解決策:

  • 必要なリソース（IAMロール、ECRイメージなど）が存在することを確認します。
  • --secrets=falseオプションを使用して、シークレットの検証をスキップします。
  • 詳細なログを確認するために--debugオプションを使用します。
$ ecspresso verify --secrets=false --debug

よくある質問（FAQ）

1. ecspressoが最新のタスク定義を使用しない

質問: デプロイ時に最新のタスク定義が使用されません。

回答: デフォルトでは、ecspressoはローカルのタスク定義ファイルから新しいタスク定義を登録します。最新の登録済みタスク定義を使用するには、--latest-task-definitionオプションを使用します。

$ ecspresso deploy --latest-task-definition

2. 特定のリビジョンにロールバックする方法

質問: 特定のタスク定義リビジョンにロールバックするにはどうすればよいですか？

回答: --skip-task-definitionと--revisionオプションを使用して、特定のリビジョンにロールバックできます。

$ ecspresso deploy --skip-task-definition --revision=10

3. デプロイ中にサービスが不安定になる

質問: デプロイ中にサービスが不安定になり、ヘルスチェックに失敗します。

回答: デプロイ設定を調整して、最小ヘルスパーセントと最大パーセントを変更します。また、ヘルスチェックの設定も確認してください。

# サービス定義内のデプロイ設定
{
  "deploymentConfiguration": {
    "maximumPercent": 200,
    "minimumHealthyPercent": 100,
    "deploymentCircuitBreaker": {
      "enable": true,
      "rollback": true
    }
  }
}

4. 環境変数の上書き方法

質問: タスク実行時に環境変数を上書きするにはどうすればよいですか？

回答: runコマンドの--envオプションを使用して、環境変数を上書きできます。

$ ecspresso run --env="KEY=VALUE" --env="ANOTHER_KEY=ANOTHER_VALUE"

5. プラグインの設定方法

質問: Terraformの状態ファイルから値を取得するプラグインを設定するにはどうすればよいですか？

回答: 設定ファイルにpluginsセクションを追加します。

# ecspresso.yml
plugins:
  - name: tfstate
    config:
      path: terraform.tfstate
      vars:
        vpc_id: $.outputs.vpc_id.value
        subnet_ids: $.outputs.private_subnet_ids.value

デバッグ技法

1. デバッグログの有効化

詳細なログを表示するには、--debugオプションを使用します。

$ ecspresso --debug deploy

2. ドライランの実行

実際の変更を行わずに、実行内容を確認するには、--dry-runオプションを使用します。

$ ecspresso deploy --dry-run

3. 差分の確認

ローカルの設定と実際にデプロイされているリソースの差分を確認するには、diffコマンドを使用します。

$ ecspresso diff --color

4. サービス状態の確認

サービスの現在の状態を確認するには、statusコマンドを使用します。

$ ecspresso status --events=20

5. コンテナ内でのデバッグ

実行中のコンテナ内でコマンドを実行してデバッグするには、execコマンドを使用します。

$ ecspresso exec --command="/bin/bash"

一般的なエラーコードと意味

エラーコード 説明 解決策
InvalidParameterException パラメータが無効 パラメータの値を確認し、正しい形式で指定します。
ClusterNotFoundException 指定されたクラスターが見つからない クラスター名が正しいことを確認します。
ServiceNotFoundException 指定されたサービスが見つからない サービス名が正しいことを確認します。
TaskDefinitionNotFoundException 指定されたタスク定義が見つからない タスク定義名とリビジョンが正しいことを確認します。
AccessDeniedException アクセス権限がない IAMロールに必要な権限があることを確認します。
ResourceInUseException リソースが使用中 リソースが他のプロセスで使用されていないことを確認します。
LimitExceededException リソース制限を超えている AWS Service Quotasを確認し、必要に応じて引き上げを申請します。
