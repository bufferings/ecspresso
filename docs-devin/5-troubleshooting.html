トラブルシューティング

このセクションでは、ecspressoを使用する際に発生する可能性のある一般的な問題と、その解決方法について説明します。

デプロイの問題

サービスが安定しない

症状: デプロイ後、サービスが安定状態に達しない。

考えられる原因:

  • タスク定義のヘルスチェックが失敗している
  • コンテナが起動時にクラッシュしている
  • 必要なリソースが不足している

解決策:

  1. サービスのステータスを確認する: 
    $ ecspresso status --events 20
  2. タスクのログを確認する: 
    $ ecspresso tasks
$ aws logs get-log-events --log-group-name /ecs/your-service --log-stream-name ecs/container-name/task-id
  3. ヘルスチェックの設定を見直す
  4. 必要に応じてタスク定義のリソース（CPU、メモリ）を増やす

デプロイがタイムアウトする

症状: デプロイコマンドがタイムアウトする。

考えられる原因:

  • タイムアウト設定が短すぎる
  • サービスの起動に時間がかかっている

解決策:

  1. タイムアウト設定を長くする: 
    # ecspresso.yml
timeout: 20m

# または、コマンドラインで指定
$ ecspresso deploy --timeout 20m
  2. デプロイの進行状況を別のターミナルで確認する: 
    $ ecspresso status --events 20

CodeDeployデプロイの失敗

症状: CodeDeployを使用したブルー/グリーンデプロイが失敗する。

考えられる原因:

  • AppSpecの設定が正しくない
  • ターゲットグループのヘルスチェックが失敗している
  • デプロイフックスクリプトが失敗している

解決策:

  1. CodeDeployのデプロイステータスを確認する（AWSコンソールまたはAWS CLI）
  2. AppSpecの設定を確認する: 
    $ ecspresso appspec
  3. ターゲットグループのヘルスチェック設定を確認する
  4. デプロイフックスクリプトのログを確認する

タスクの問題

タスクが起動しない

症状: タスクが起動せず、すぐに停止する。

考えられる原因:

  • コンテナのエントリポイントまたはコマンドが正しくない
  • 必要な環境変数が設定されていない
  • リソース制約（CPU、メモリ）
  • ネットワーク設定の問題

解決策:

  1. タスクのステータスを確認する: 
    $ ecspresso tasks --status STOPPED
  2. タスクのログを確認する（CloudWatch Logs）
  3. タスク定義を検証する: 
    $ ecspresso verify
  4. デバッグモードでecspressoを実行する: 
    $ ecspresso --debug run

execコマンドが失敗する

症状: ecspresso execコマンドが失敗する。

考えられる原因:

  • タスクでECS Execが有効になっていない
  • 必要なIAM権限がない
  • SSMセッションマネージャーの設定が正しくない

解決策:

  1. タスク定義でECS Execを有効にする: 
    # ecs-service-def.json
{
  "enableExecuteCommand": true,
  ...
}
  2. 必要なIAM権限を追加する: 
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ssmmessages:CreateControlChannel",
        "ssmmessages:CreateDataChannel",
        "ssmmessages:OpenControlChannel",
        "ssmmessages:OpenDataChannel"
      ],
      "Resource": "*"
    }
  ]
}
  3. デバッグモードでecspressoを実行する: 
    $ ecspresso --debug exec --command /bin/bash

設定の問題

設定ファイルの読み込みエラー

症状: ecspressoが設定ファイルを読み込めない。

考えられる原因:

  • YAMLまたはJSONの構文エラー
  • 設定ファイルが見つからない
  • 環境変数の展開に失敗している

解決策:

  1. 設定ファイルの構文を確認する: 
    $ yamllint ecspresso.yml
$ jsonlint ecs-task-def.json
  2. 設定ファイルをレンダリングして確認する: 
    $ ecspresso render --config
  3. デバッグモードでecspressoを実行する: 
    $ ecspresso --debug status

タスク定義の検証エラー

症状: ecspresso verifyコマンドがエラーを報告する。

考えられる原因:

  • 参照しているリソース（IAMロール、セキュリティグループなど）が存在しない
  • シークレットが存在しない
  • ログ設定が正しくない

解決策:

  1. 詳細なエラーメッセージを確認する: 
    $ ecspresso --debug verify
  2. 参照しているリソースが存在することを確認する
  3. シークレットの検証をスキップする（必要な場合）: 
    $ ecspresso verify --secrets=false

権限の問題

IAM権限エラー

症状: 「アクセスが拒否されました」などのIAM権限エラーが発生する。

考えられる原因:

  • 必要なIAM権限がない
  • ロールの引き受けに失敗している

解決策:

  1. 必要なIAM権限を確認する（インストール方法のIAMポリシー例を参照）
  2. AWS認証情報が正しく設定されていることを確認する: 
    $ aws sts get-caller-identity
  3. ロールを引き受ける場合は、正しいARNを指定する: 
    $ ecspresso --assume-role-arn arn:aws:iam::123456789012:role/EcsDeployRole status

タスク実行ロールの問題

症状: タスクが起動するが、すぐに停止する。CloudWatchログに「アクセスが拒否されました」などのエラーが表示される。

考えられる原因:

  • タスク実行ロールに必要な権限がない
  • タスクロールに必要な権限がない

解決策:

  1. タスク実行ロールに必要な権限を追加する: 
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ecr:GetAuthorizationToken",
        "ecr:BatchCheckLayerAvailability",
        "ecr:GetDownloadUrlForLayer",
        "ecr:BatchGetImage",
        "logs:CreateLogStream",
        "logs:PutLogEvents"
      ],
      "Resource": "*"
    }
  ]
}
  2. シークレットを使用している場合は、シークレットへのアクセス権を追加する: 
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "secretsmanager:GetSecretValue",
        "ssm:GetParameters"
      ],
      "Resource": [
        "arn:aws:secretsmanager:region:account:secret:secret-name",
        "arn:aws:ssm:region:account:parameter/parameter-name"
      ]
    }
  ]
}

その他の問題

デバッグ情報の取得

問題のトラブルシューティングに役立つデバッグ情報を取得するには、--debugオプションを使用します。

$ ecspresso --debug deploy

また、ログ形式をJSONに変更することで、より詳細な情報を取得できます。

$ ecspresso --log-format json deploy

一般的なエラーメッセージと解決策

エラーメッセージ 考えられる原因 解決策
service xxx is not found 指定されたサービスが存在しない サービス名とクラスター名を確認する
failed to register task definition: InvalidParameterException タスク定義のパラメータが無効 タスク定義を検証し、エラーメッセージの詳細を確認する
failed to update service: InvalidParameterException サービス更新のパラメータが無効 サービス定義を検証し、エラーメッセージの詳細を確認する
failed to run task: ResourceNotFoundException 指定されたリソースが存在しない クラスター名、サブネット、セキュリティグループなどを確認する
failed to verify: resource not found 検証中に参照されているリソースが見つからない リソースの存在を確認するか、--secrets=falseオプションを使用する

サポートの取得

問題が解決しない場合は、以下の方法でサポートを受けることができます。

  • GitHubのIssueを作成する
  • GitHubのDiscussionsで質問する

問題を報告する際は、以下の情報を含めると、より迅速に解決できます。

  • ecspressoのバージョン（ecspresso versionの出力）
  • 使用しているAWSリージョン
  • 実行したコマンドとオプション
  • エラーメッセージ
  • デバッグログ（--debugオプションを使用して取得）
