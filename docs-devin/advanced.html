高度な機能

バージョン制約

設定ファイル内のrequired_versionは、ecspressoのバージョンを固定するためのものです。

required_version: ">= 2.0.0, < 3"

これにより、バージョンが2.0.0以上かつ3未満の場合にのみecspressoが実行できるようになります。バージョンがこの範囲内にない場合、実行は失敗します。

Application Auto Scalingの管理

Application Auto Scalingを使用するECSサービスの場合、ecspresso scaleコマンドを使用して自動スケーリングの最小値と最大値の設定を調整することが簡単にできます。scale --auto-scaling-minまたはscale --auto-scaling-maxを指定するだけで設定を変更できます。

$ ecspresso scale --tasks 5 --auto-scaling-min 5 --auto-scaling-max 20

ecspresso deployとscaleはアプリケーションの自動スケーリングを一時停止および再開できます：

  • --suspend-auto-scalingは一時停止状態をtrueに設定します。
  • --resume-auto-scalingは一時停止状態をfalseに設定します。

JSONとYAMLの代わりにJsonnetを使用する

ecspressoはJsonnetファイル形式をサポートしています。

  • v1.7以降：サービスとタスク定義のJsonnetサポート
  • v2.0以降：設定ファイルのJsonnetサポート
  • v2.4以降：Jsonnet関数をサポート

ファイルの拡張子が.jsonnetの場合、ecspressoは以下の順序で処理します：

  1. Jsonnetとして処理
  2. JSONに変換
  3. テンプレート構文の評価でロード

Jsonnet関数

v2.4以降はJsonnetファイル内でJsonnetネイティブ関数をサポートしています。

.jsonnetファイル内で：

  1. local func = std.native('func');を定義
  2. func()を使用

Jsonnet関数はJsonnetファイルのレンダリング時に評価され、テンプレート構文との競合を回避するのに役立ちます。

env、must_env

local env = std.native('env');
local must_env = std.native('must_env');
{
  foo: env('FOO', 'default value'),
  bar: must_env('BAR'),
  bazNumber: std.parseInt(env('BAZ_NUMBER', '0')),
  booBool: std.parseJson(env('BOO_BOOL', 'false')),
}

Fargateへのデプロイ

Fargateにサービスをデプロイする場合、タスク定義とサービス定義の両方に特定の設定が必要です。

タスク定義の場合：

  • requiresCompatibilities（「FARGATE」が必要）
  • networkMode（「awsvpc」が必要）
  • cpu（必須）
  • memory（必須）
  • executionRoleArn（オプション）

サービス定義の場合：

  • launchType（「FARGATE」が必要）
  • networkConfiguration（「awsvpcConfiguration」が必要）

ECS Service Connectのサポート

ecspressoはECS Service Connectをサポートしています。

設定するには、サービス定義にserviceConnectConfigurationを、タスク定義にportMappingsを定義します。

EBSボリュームのサポート

ecspressoはAmazon EBSボリュームをサポートしています。

設定するには、サービス定義にvolumeConfigurationsを、タスク定義にmountPointsとvolumesを定義します。

VPC Latticeのサポート

ecspressoはVPC Lattice統合をサポートしています。

タスク定義にportMappingsを、サービス定義にvpcLatticeConfigurationsを定義します。
