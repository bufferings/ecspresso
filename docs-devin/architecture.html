アーキテクチャの全体像

ecspressoは、モジュール化されたアーキテクチャを採用しており、各コンポーネントが特定の責任を持っています。中心となるのはApp構造体で、これが他のすべてのコンポーネントを調整します。

flowchart TB User[ユーザー] --> CLI CLI[CLI] --> App App --> Config[設定管理] App --> Deploy[デプロイ操作] App --> Verify[リソース検証] App --> Diff[差分比較] App --> Tasks[タスク管理] App --> Plugins[プラグインシステム] Config --> Templates[テンプレート処理] Config --> Plugins Deploy --> ECS[AWS ECS API] Deploy --> CodeDeploy[AWS CodeDeploy API] Verify --> IAM[AWS IAM API] Verify --> ECR[AWS ECR API] Verify --> VPC[AWS VPC API] Tasks --> ECS Diff --> ECS

中核コンポーネント

ecspressoの中核コンポーネントは以下の通りです：

  • CLI（cli.go）：コマンドライン引数を解析し、適切なハンドラーにディスパッチします。kingpinライブラリを使用してコマンドとフラグを定義しています。
  • App（ecspresso.go）：中心的なオーケストレーターで、AWS SDKクライアントと設定への参照を含み、すべての主要な操作メソッドを実装しています。
  • 設定管理（config.go）：ecspresso.ymlからの設定値を管理します。YAML、JSON、Jsonnet形式をサポートし、テンプレート機能を通じて環境変数や外部ソースからの値の参照が可能です。
  • デプロイ操作（deploy.go）：ECSサービスのデプロイロジックを実装しています。ECSローリングデプロイとCodeDeployの両方をサポートします。
  • リソース検証（verify.go）：リソースが正しく構成されていることを確認します。ECSクラスター、IAMロール、ECRイメージ、コンテナ定義、ネットワーク設定など多岐にわたるリソースを検証します。
  • 差分比較（diff.go）：ローカル定義と実際のリソースの違いを表示します。サービス定義とタスク定義の比較を行い、統一差分形式で出力します。
  • タスク管理（tasks.go、run.go）：タスクの実行と管理を担当します。一時的なタスクの実行、実行中のタスクの一覧表示、停止などの機能を提供します。
  • プラグインシステム（plugin.go）：機能拡張のための柔軟なプラグインシステムを提供します。tfstate、cloudformation、ssm、secretsmanager、externalなどの組み込みプラグインをサポートしています。

App構造体の詳細

App構造体は、ecspressoの中心的なコンポーネントで、以下のフィールドを持っています：

type App struct {
	ecs               *ecs.Client
	ecsWaiter         *ecs.Waiter
	codedeploy        *codedeploy.Client
	iam               *iam.Client
	cwl               *cloudwatchlogs.Client
	config            *Config
	rootfs            afero.Fs
	loader            *configLoader
	appSpecRenderer   *appspec.Renderer
	outputJSONFunc    func(context.Context, interface{}) error
}

主要なメソッドには以下があります：

  • Start：アプリケーションを初期化し、AWS SDKクライアントを設定します。
  • DescribeService：現在のサービス状態を取得します。
  • DescribeTaskDefinition：タスク定義の詳細を取得します。
  • Deploy：ECSサービスをデプロイします。
  • Rollback：サービスをロールバックします。
  • Verify：リソースを検証します。
  • Diff：設定を比較します。
  • Run：一時的なタスクを実行します。
  • Wait：サービスの安定性を待ちます。

データフロー

ecspressoでのデータフローは以下のようになります：

sequenceDiagram participant User as ユーザー participant CLI participant App participant Config as 設定管理 participant AWS as AWS API User->>CLI: コマンド実行 CLI->>App: コマンドディスパッチ App->>Config: 設定読み込み Config-->>App: 設定データ App->>AWS: API呼び出し AWS-->>App: レスポンス App-->>CLI: 結果 CLI-->>User: 出力

例えば、デプロイコマンドを実行すると、以下のようなフローになります：

sequenceDiagram participant User as ユーザー participant CLI participant App participant Config as 設定管理 participant Verify as リソース検証 participant Deploy as デプロイ操作 participant AWS as AWS API User->>CLI: ecspresso deploy CLI->>App: Deployメソッド呼び出し App->>Config: 設定読み込み Config-->>App: 設定データ App->>Verify: リソース検証 Verify->>AWS: リソース確認 AWS-->>Verify: 検証結果 Verify-->>App: 検証結果 App->>Deploy: デプロイ実行 Deploy->>AWS: タスク定義登録 AWS-->>Deploy: タスク定義ARN Deploy->>AWS: サービス更新 AWS-->>Deploy: デプロイ状態 Deploy-->>App: デプロイ結果 App-->>CLI: 結果 CLI-->>User: 出力
