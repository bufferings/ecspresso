設定ファイル

ecspressoの設定ファイル（ecspresso.yml）は、ECSサービスとタスク定義の管理に必要な情報を定義します。基本的な設定ファイルの例は以下の通りです：

region: ap-northeast-1
cluster: my-cluster
service: my-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
plugins:
  - name: tfstate
    config:
      path: terraform.tfstate

主な設定項目：

  • region: AWSリージョン
  • cluster: ECSクラスター名
  • service: ECSサービス名
  • service_definition: サービス定義ファイルのパス
  • task_definition: タスク定義ファイルのパス
  • timeout: 操作のタイムアウト時間
  • plugins: 使用するプラグインの設定

タスク定義

タスク定義ファイル（ecs-task-def.json）は、ECSタスクの実行方法を定義します。基本的なタスク定義の例は以下の通りです：

{
  "family": "my-app",
  "executionRoleArn": "arn:aws:iam::123456789012:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::123456789012:role/ecsTaskRole",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "app",
      "image": "123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:latest",
      "essential": true,
      "portMappings": [
        {
          "containerPort": 8080,
          "hostPort": 8080,
          "protocol": "tcp"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/my-app",
          "awslogs-region": "ap-northeast-1",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ],
  "requiresCompatibilities": [
    "FARGATE"
  ],
  "cpu": "256",
  "memory": "512"
}

タスク定義では、以下の項目を設定できます：

  • コンテナの定義（イメージ、ポート、環境変数など）
  • タスクのリソース要件（CPU、メモリ）
  • ネットワークモード
  • IAMロール
  • ログ設定

サービス定義

サービス定義ファイル（ecs-service-def.json）は、ECSサービスの実行方法を定義します。基本的なサービス定義の例は以下の通りです：

{
  "cluster": "my-cluster",
  "serviceName": "my-service",
  "desiredCount": 2,
  "launchType": "FARGATE",
  "networkConfiguration": {
    "awsvpcConfiguration": {
      "subnets": [
        "subnet-12345678",
        "subnet-87654321"
      ],
      "securityGroups": [
        "sg-12345678"
      ],
      "assignPublicIp": "ENABLED"
    }
  },
  "loadBalancers": [
    {
      "targetGroupArn": "arn:aws:elasticloadbalancing:ap-northeast-1:123456789012:targetgroup/my-tg/1234567890123456",
      "containerName": "app",
      "containerPort": 8080
    }
  ]
}

サービス定義では、以下の項目を設定できます：

  • デプロイ設定（タスク数、デプロイ方法）
  • ネットワーク設定（サブネット、セキュリティグループ）
  • ロードバランサー設定
  • Auto Scaling設定

基本コマンド

ecspressoの基本的なコマンドは以下の通りです：

設定の初期化

$ ecspresso init --region ap-northeast-1 --cluster my-cluster --service my-service

サービスのデプロイ

$ ecspresso deploy

サービスの状態確認

$ ecspresso status

設定の差分確認

$ ecspresso diff

リソースの検証

$ ecspresso verify

タスクの実行

$ ecspresso run

サービスのスケーリング

$ ecspresso scale --tasks 5

注意: コマンドの詳細なオプションについては、コマンドリファレンスを参照してください。
