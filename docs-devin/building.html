ビルド概要

ecspressoは、Go言語で開発されており、標準的なGoのビルドツールとMakefileを使用してビルドされます。このセクションでは、ecspressoのビルドとテストの方法について説明します。

graph TD A[ビルドプロセス] --> B[依存関係の解決] A --> C[コンパイル] A --> D[テスト] A --> E[パッケージング] B --> B1[go mod download] C --> C1[go build] C --> C2[クロスコンパイル] D --> D1[ユニットテスト] D --> D2[統合テスト] D --> D3[リントチェック] E --> E1[バイナリ作成] E --> E2[リリースアーカイブ作成]

ecspressoのビルドには、以下のツールが使用されます：

  • Go 1.16以上
  • Make
  • goreleaser（リリース用）
  • golangci-lint（リントチェック用）

ビルドプロセス

ecspressoのビルドプロセスは、Makefileで定義されています。主要なビルドターゲットは以下の通りです：

依存関係の解決

ビルドの最初のステップは、依存関係の解決です。これは、go mod downloadコマンドで行われます：

go mod download

コンパイル

次に、ソースコードをコンパイルしてバイナリを作成します。これは、go buildコマンドで行われます：

go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso cmd/ecspresso/main.go

このコマンドは、cmd/ecspresso/main.goファイルをコンパイルして、bin/ecspressoバイナリを作成します。-ldflagsオプションは、バイナリにバージョン情報を埋め込むために使用されます。

クロスコンパイル

ecspressoは、複数のプラットフォーム向けにクロスコンパイルすることもできます。これは、GOOSとGOARCH環境変数を設定してgo buildコマンドを実行することで行われます：

GOOS=linux GOARCH=amd64 go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso_linux_amd64 cmd/ecspresso/main.go
GOOS=darwin GOARCH=amd64 go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso_darwin_amd64 cmd/ecspresso/main.go
GOOS=windows GOARCH=amd64 go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso_windows_amd64.exe cmd/ecspresso/main.go

パッケージング

リリース用のパッケージングは、goreleaserツールを使用して行われます。これは、.goreleaser.ymlファイルで設定されています：

goreleaser release --rm-dist

このコマンドは、複数のプラットフォーム向けにバイナリをビルドし、リリースアーカイブを作成します。

テストプロセス

ecspressoのテストプロセスは、Makefileで定義されています。主要なテストターゲットは以下の通りです：

ユニットテスト

ユニットテストは、go testコマンドで実行されます：

go test -v ./...

このコマンドは、すべてのパッケージのテストを実行します。

カバレッジテスト

カバレッジテストは、go testコマンドの-coverオプションで実行されます：

go test -v -cover ./...

このコマンドは、すべてのパッケージのテストを実行し、カバレッジ情報を表示します。

リントチェック

リントチェックは、golangci-lintツールで実行されます：

golangci-lint run

このコマンドは、コードの品質をチェックし、潜在的な問題を検出します。

CI/CD統合

ecspressoは、GitHub Actionsを使用してCI/CDパイプラインを実装しています。主要なワークフローは以下の通りです：

テストワークフロー

プルリクエストとmainブランチへのプッシュ時に実行されるテストワークフローです：

name: Test
on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - '*'
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-go@v2
        with:
          go-version: 1.16
      - name: lint
        run: |
          curl -sSfL https://raw.githubusercontent.com/golangci/golangci-lint/master/install.sh | sh -s -- -b $(go env GOPATH)/bin v1.39.0
          golangci-lint run
      - name: test
        run: |
          go test -v ./...

リリースワークフロー

タグがプッシュされたときに実行されるリリースワークフローです：

name: Release
on:
  push:
    tags:
      - 'v*'
jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-go@v2
        with:
          go-version: 1.16
      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v2
        with:
          version: latest
          args: release --rm-dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

ビルド例

以下は、ecspressoのビルドとテストの例です：

ローカルビルド

# リポジトリのクローン
git clone https://github.com/kayac/ecspresso.git
cd ecspresso

# 依存関係のインストール
go mod download

# ビルド
make build

# バイナリの実行
./bin/ecspresso --help

テスト実行

# ユニットテスト
make test

# カバレッジテスト
go test -v -cover ./...

# リントチェック
make lint

クロスコンパイル

# Linux用ビルド
GOOS=linux GOARCH=amd64 go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso_linux_amd64 cmd/ecspresso/main.go

# macOS用ビルド
GOOS=darwin GOARCH=amd64 go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso_darwin_amd64 cmd/ecspresso/main.go

# Windows用ビルド
GOOS=windows GOARCH=amd64 go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso_windows_amd64.exe cmd/ecspresso/main.go

リリース作成

# タグの作成
git tag v1.0.0
git push origin v1.0.0

# GoReleaserでリリース作成（ローカル）
goreleaser release --rm-dist
