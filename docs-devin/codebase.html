コードベース概要

ecspressoのコードベースは、Go言語で書かれており、モジュール化された構造を持っています。このセクションでは、ecspressoのコードベース構造について説明します。

graph TD A[ecspresso] --> B[コアモジュール] A --> C[コマンドモジュール] A --> D[プラグインモジュール] A --> E[ユーティリティモジュール] B --> B1[App構造体] B --> B2[Config構造体] B --> B3[AWS SDK統合] C --> C1[CLI] C --> C2[デプロイコマンド] C --> C3[タスクコマンド] C --> C4[ユーティリティコマンド] D --> D1[プラグインシステム] D --> D2[tfstateプラグイン] D --> D3[CloudFormationプラグイン] D --> D4[SSMプラグイン] D --> D5[Secrets Managerプラグイン] E --> E1[ロギング] E --> E2[テンプレート関数] E --> E3[ユーティリティ関数]

ecspressoのコードベースは、以下の主要なコンポーネントで構成されています：

  • コアモジュール: アプリケーションの中心的な機能を提供
  • コマンドモジュール: CLIコマンドを実装
  • プラグインモジュール: 拡張機能を提供
  • ユーティリティモジュール: 共通の機能を提供

ディレクトリ構造

ecspressoのディレクトリ構造は、以下のようになっています：

ecspresso/
├── cmd/
│   └── ecspresso/
│       ├── main.go       # エントリーポイント
│       └── signals_unix.go # UNIXシグナル処理
├── plugin/
│   ├── cloudformation/   # CloudFormationプラグイン
│   ├── external/         # 外部コマンドプラグイン
│   ├── secretsmanager/   # Secrets Managerプラグイン
│   ├── ssm/              # SSMプラグイン
│   └── tfstate/          # tfstateプラグイン
├── tests/
│   ├── e2e/              # E2Eテスト
│   └── fixtures/         # テスト用フィクスチャ
├── .github/
│   └── workflows/        # GitHub Actions
├── appspec.go            # CodeDeployアプリケーション仕様
├── cli.go                # CLIコマンド定義
├── config.go             # 設定管理
├── deploy.go             # デプロイ機能
├── diff.go               # 差分機能
├── ecspresso.go          # コアアプリケーション
├── exec.go               # コマンド実行機能
├── init.go               # 初期化機能
├── plugin.go             # プラグインシステム
├── register.go           # タスク定義登録
├── render.go             # テンプレートレンダリング
├── rollback.go           # ロールバック機能
├── run.go                # タスク実行機能
├── status.go             # ステータス表示
├── tasks.go              # タスク管理
├── util.go               # ユーティリティ関数
├── verify.go             # 検証機能
├── wait.go               # 待機機能
├── go.mod                # Go モジュール定義
├── go.sum                # Go モジュールチェックサム
├── Makefile              # ビルドスクリプト
└── README.md             # プロジェクト説明

主要なディレクトリの説明：

  • cmd/ecspresso/: CLIアプリケーションのエントリーポイント
  • plugin/: プラグインの実装
  • tests/: テストコードとフィクスチャ
  • .github/workflows/: GitHub Actionsの設定
