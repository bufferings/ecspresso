コマンド構造

ecspressoのコマンドは以下の構造に従います：

$ ecspresso [グローバルオプション] コマンド [コマンドオプション]

例えば、以下のようにコマンドを実行します：

$ ecspresso --config ecspresso.yml deploy --tasks 2

この例では：

  • --config ecspresso.yml: グローバルオプション
  • deploy: コマンド
  • --tasks 2: コマンドオプション

グローバルオプション

すべてのコマンドで使用できるグローバルオプションは以下の通りです：

オプション 説明 デフォルト値
--config FILE 設定ファイルのパス ecspresso.yml
--debug デバッグログを有効にする false
--envfile FILE 環境変数ファイルのパス -
--help, -h ヘルプを表示する -
--profile NAME AWS認証情報のプロファイル名 -
--region REGION AWSリージョン 設定ファイルの値
--version, -v バージョン情報を表示する -

コマンドカテゴリ

ecspressoのコマンドは以下のカテゴリに分類されます：

mindmap root((ecspresso)) デプロイ関連 deploy rollback scale wait タスク関連 run exec tasks 設定・検証 init verify diff status render appspec ユーティリティ version help

デプロイ関連コマンド

ECSサービスのデプロイ、ロールバック、スケーリングを管理するコマンドです。詳細はデプロイ関連コマンドを参照してください。

タスク関連コマンド

ECSタスクの実行、コマンド実行、タスク管理を行うコマンドです。詳細はタスク関連コマンドを参照してください。

設定・検証コマンド

設定の初期化、検証、差分確認などを行うコマンドです。詳細は設定・検証コマンドを参照してください。

コマンド一覧

ecspressoで利用可能なすべてのコマンドは以下の通りです：

コマンド 説明 カテゴリ
deploy ECSサービスをデプロイする デプロイ関連
rollback 以前のタスク定義にロールバックする デプロイ関連
scale サービスのタスク数を変更する デプロイ関連
wait サービスが安定するまで待機する デプロイ関連
run 一時的なタスクを実行する タスク関連
exec 実行中のタスク内でコマンドを実行する タスク関連
tasks タスクの一覧表示や管理を行う タスク関連
init 設定ファイルを初期化する 設定・検証
verify リソースの設定を検証する 設定・検証
diff ローカルとAWS上の設定の差分を表示する 設定・検証
status サービスの現在の状態を表示する 設定・検証
render タスク定義やサービス定義をレンダリングする 設定・検証
appspec CodeDeployのAppSpecを生成する 設定・検証
version バージョン情報を表示する ユーティリティ
help ヘルプを表示する ユーティリティ

ヒント: 各コマンドの詳細なオプションを確認するには、ecspresso help コマンド名を実行してください。
