コマンド構造

ecspressoのコマンドは以下の構造に従います：

ecspresso [グローバルオプション] コマンド [コマンドオプション]

コマンドはcli.goファイルのCLI関数で処理され、App構造体のメソッドにディスパッチされます。内部的には、urfave/cliライブラリを使用してコマンドラインインターフェースを構築しています。

flowchart TD CLI[CLI関数] --> Parse[コマンドライン解析] Parse --> App[App構造体作成] App --> Command[コマンド実行] Command --> Result[結果出力]

グローバルオプション

すべてのコマンドで使用できるグローバルオプション：

オプション 説明 デフォルト値
--config, -c 設定ファイルのパス ecspresso.yml
--debug, -d デバッグログを有効にする false
--envfile 環境変数ファイルのパス -
--help, -h ヘルプを表示する -
--ignore-no-changes 変更がない場合でもエラーを返さない false
--no-color カラー出力を無効にする false
--profile AWS認証情報のプロファイル名 -
--quiet, -q 出力を最小限にする false
--region AWS リージョン -
--timeout タイムアウト時間（秒） 300
--version, -v バージョンを表示する -

コマンド一覧

ecspressoで利用可能なコマンド一覧：

mindmap root((ecspresso)) デプロイ関連 deploy rollback scale wait タスク関連 run exec tasks register 検証関連 verify diff status 設定関連 init render appspec ユーティリティ version help
コマンド 説明 実装ファイル
deploy ECSサービスをデプロイする deploy.go
rollback ECSサービスをロールバックする rollback.go
scale ECSサービスのタスク数を変更する deploy.go
wait ECSサービスが安定するまで待機する wait.go
run ECSタスクを実行する run.go
exec 実行中のECSタスクでコマンドを実行する exec.go
tasks ECSタスクを一覧表示または停止する tasks.go
register タスク定義を登録する ecspresso.go
verify 設定を検証する verify.go
diff ローカル設定とAWS上の設定の差分を表示する diff.go
status ECSサービスの状態を表示する ecspresso.go
init 設定ファイルを初期化する ecspresso.go
render タスク定義とサービス定義をレンダリングする ecspresso.go
appspec CodeDeployのAppSpecを生成する appspec.go
version バージョンを表示する cli.go
help ヘルプを表示する cli.go

コマンド詳細

各コマンドの詳細な使用方法については、以下のページを参照してください：

  • 2.2 デプロイメントコマンド - deploy, rollback, scale, waitコマンドの詳細
  • 2.3 タスク管理コマンド - run, exec, tasks, registerコマンドの詳細

コマンドのヘルプを表示するには、以下のコマンドを実行します：

ecspresso help [コマンド名]

例えば、deployコマンドのヘルプを表示するには：

ecspresso help deploy
