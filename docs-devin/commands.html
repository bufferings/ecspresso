コマンド

Deploy

deployコマンドは設定ファイルに基づいてECSサービスをデプロイします。

ローリングデプロイメントの場合：

2017/11/09 23:20:13 myService/default Registering a new task definition...
2017/11/09 23:20:13 myService/default Task definition is registered myService:4

2017/11/09 23:23:23 myService/default  PRIMARY myService:4 desired:1 pending:0 running:1
2017/11/09 23:23:29 myService/default Service is stable now. Completed!

ブルー/グリーンデプロイメント（AWS CodeDeployを使用）の場合：

ecspresso deployはCODE_DEPLOYデプロイメントコントローラーを使用してサービスをデプロイできます。ecs-service-def.jsonを以下のように設定します：

{
  "deploymentController": {
    "type": "CODE_DEPLOY"
  },
  // ...
}

主なオプション：

  • --wait-until: デプロイメントを待機する条件を指定（stableまたはdeployed）
  • --revision: タスク定義の特定のリビジョンを指定
  • --latest-task-definition: タスク定義の最新リビジョンを使用
  • --skip-task-definition: 新しいタスク定義の登録をスキップ
  • --force-new-deployment: サービス定義とタスク定義が変更されていなくても、新しいデプロイメントを強制

Verify

サービス/タスク定義に関連するリソースを検証します。

例えば、以下を確認します：

  • ECSクラスターが存在するか。
  • サービス定義内のターゲットグループが、定義内のコンテナ名とポートと一致するか。
  • タスクロールとタスク実行ロールが存在し、ecs-tasks.amazonaws.comによって引き受けられるか。
  • タスク定義で定義されたURLにコンテナイメージが存在するか。（ECRまたはDockerHub公開イメージのみチェック。）
  • タスク定義内のシークレットが存在し、読み取り可能か。
  • 指定されたCloudWatchロググループストリームにログストリームを作成し、メッセージを送信できるか。

Run

タスク定義を使用して一時的なタスクを実行します。

$ ecspresso run --config ecspresso.yml --task-def=db-migrate.json

--task-defが設定されていない場合、ecspressoはサービスに含まれるタスク定義を使用します。

Scale

サービスの希望するカウント数を変更します。

$ ecspresso scale --tasks 10

scaleコマンドはdeploy --skip-task-definition --no-update-serviceと同等です。

Tasks

サービスによって実行されるタスク、またはタスク定義と同じファミリーを持つタスクを一覧表示します。

$ ECSPRESSO_FILTER_COMMAND=peco ecspresso tasks --find

Exec

タスク上でコマンドを実行します。

$ ecspresso exec --port-forward --port 80 --local-port 8080

各コマンドの詳細については、ecspresso command --helpを参照してください。
