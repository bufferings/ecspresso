設定ファイル概要

ecspressoの設定ファイルは、ecspresso.yml（またはecspresso.yaml、ecspresso.json、ecspresso.jsonnet）という名前のファイルで、ecspressoの動作を制御するための設定を記述します。この設定ファイルは、config.goファイルで定義されているConfig構造体にマッピングされます。

graph TD A[設定ファイル] --> B[YAML形式] A --> C[JSON形式] A --> D[Jsonnet形式] B --> E[ecspresso.yml] B --> F[ecspresso.yaml] C --> G[ecspresso.json] D --> H[ecspresso.jsonnet] E --> I[Config構造体] F --> I G --> I H --> I I --> J[タスク定義パス] I --> K[サービス定義パス] I --> L[クラスター名] I --> M[サービス名] I --> N[プラグイン設定]

設定ファイルは、以下のような目的で使用されます：

  • タスク定義ファイルとサービス定義ファイルのパスを指定する
  • ECSクラスターとサービスの名前を指定する
  • デプロイメントオプションを設定する
  • プラグインを設定する
  • 環境変数を設定する

設定ファイル構造

ecspressoの設定ファイルは、Config構造体にマッピングされます。この構造体は、config.goファイルで以下のように定義されています：

type Config struct {
    Region                string            `yaml:"region"`
    Cluster               string            `yaml:"cluster"`
    Service               string            `yaml:"service"`
    ServiceDefinitionPath string            `yaml:"service_definition"`
    TaskDefinitionPath    string            `yaml:"task_definition"`
    Timeout               time.Duration     `yaml:"timeout"`
    CodeDeploy            *CodeDeployConfig `yaml:"codedeploy"`
    AppSpec               *AppSpecConfig    `yaml:"appspec"`
    Plugins               map[string]map[string]interface{} `yaml:"plugins"`
    Envs                  []string          `yaml:"envs"`
    EnvFile               string            `yaml:"env_file"`
    Vars                  map[string]string `yaml:"vars"`
    ECS                   *ECSConfig        `yaml:"ecs"`
    Params                map[string]interface{} `yaml:"params"`
}

主要なフィールドの説明：

  • Region: AWS リージョン
  • Cluster: ECS クラスター名
  • Service: ECS サービス名
  • ServiceDefinitionPath: サービス定義ファイルのパス
  • TaskDefinitionPath: タスク定義ファイルのパス
  • Timeout: タイムアウト時間
  • CodeDeploy: CodeDeploy の設定
  • AppSpec: AppSpec の設定
  • Plugins: プラグインの設定
  • Envs: 環境変数のリスト
  • EnvFile: 環境変数ファイルのパス
  • Vars: 変数のマップ
  • ECS: ECS の設定
  • Params: その他のパラメータ

CodeDeployConfig構造体は、以下のように定義されています：

type CodeDeployConfig struct {
    ApplicationName     string `yaml:"application_name"`
    DeploymentGroupName string `yaml:"deployment_group_name"`
    TerminationWaitTime int64  `yaml:"termination_wait_time"`
}

AppSpecConfig構造体は、以下のように定義されています：

type AppSpecConfig struct {
    Create bool `yaml:"create"`
}

ECSConfig構造体は、以下のように定義されています：

type ECSConfig struct {
    EnableExecuteCommand bool `yaml:"enable_execute_command"`
}

サポートされる形式

ecspressoは、以下の形式の設定ファイルをサポートしています：

graph TD A[サポートされる形式] --> B[YAML] A --> C[JSON] A --> D[Jsonnet] B --> E[ecspresso.yml] B --> F[ecspresso.yaml] C --> G[ecspresso.json] D --> H[ecspresso.jsonnet]

YAML形式

region: ap-northeast-1
cluster: my-cluster
service: my-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
plugins:
  tfstate:
    path: terraform.tfstate
  ssm:
    path: /path/to/parameter
envs:
  - APP_NAME
  - APP_ENV
env_file: .env
vars:
  key1: value1
  key2: value2
ecs:
  enable_execute_command: true

JSON形式

{
  "region": "ap-northeast-1",
  "cluster": "my-cluster",
  "service": "my-service",
  "service_definition": "ecs-service-def.json",
  "task_definition": "ecs-task-def.json",
  "timeout": "10m",
  "plugins": {
    "tfstate": {
      "path": "terraform.tfstate"
    },
    "ssm": {
      "path": "/path/to/parameter"
    }
  },
  "envs": [
    "APP_NAME",
    "APP_ENV"
  ],
  "env_file": ".env",
  "vars": {
    "key1": "value1",
    "key2": "value2"
  },
  "ecs": {
    "enable_execute_command": true
  }
}

Jsonnet形式

  timeout: '10m',
  plugins: {
    tfstate: {
      path: 'terraform.tfstate'
    },
    ssm: {
      path: '/path/to/parameter'
    }
  },
  envs: [
    'APP_NAME',
    'APP_ENV'
  ],
  env_file: '.env',
  vars: {
    key1: 'value1',
    key2: 'value2'
  },
  ecs: {
    enable_execute_command: true
  }
}

テンプレート関数

ecspressoは、設定ファイル内でテンプレート関数を使用することをサポートしています。これらの関数は、config.goファイルのnewTemplateFuncs関数で定義されています。

graph TD A[テンプレート関数] --> B[環境変数関数] A --> C[プラグイン関数] A --> D[ユーティリティ関数] B --> E[env] B --> F[must_env] C --> G[tfstate] C --> H[tfstatef] C --> I[cfn_output] C --> J[cfn_export] C --> K[ssm] C --> L[secretsmanager_arn] C --> M[external] D --> N[json] D --> O[yaml] D --> P[base64] D --> Q[base64decode]

環境変数関数

  • env: 環境変数の値を取得します。環境変数が存在しない場合は空文字列を返します。
  • must_env: 環境変数の値を取得します。環境変数が存在しない場合はエラーを返します。

プラグイン関数

  • tfstate: Terraformの状態ファイルからリソース情報を取得します。
  • tfstatef: Terraformの状態ファイルからリソース情報を取得します（フォーマット指定可能）。
  • cfn_output: CloudFormationスタックの出力値を取得します。
  • cfn_export: CloudFormationスタックのエクスポート値を取得します。
  • ssm: SSM Parameter Storeからパラメータを取得します。
  • secretsmanager_arn: Secrets Managerのシークレット名からARNを取得します。
  • external: 外部コマンドを実行して結果を取得します。

ユーティリティ関数

  • json: オブジェクトをJSON形式に変換します。
  • yaml: オブジェクトをYAML形式に変換します。
  • base64: 文字列をBase64エンコードします。
  • base64decode: Base64エンコードされた文字列をデコードします。

設定例

以下は、ecspressoの設定ファイルの例です：

基本的な設定

region: ap-northeast-1
cluster: my-cluster
service: my-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m

環境変数を使用した設定

region: {{ env "AWS_REGION" }}
cluster: {{ env "ECS_CLUSTER" }}
service: {{ env "ECS_SERVICE" }}
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
envs:
  - APP_NAME
  - APP_ENV
env_file: .env

プラグインを使用した設定

region: ap-northeast-1
cluster: {{ tfstate "aws_ecs_cluster.main.name" }}
service: {{ tfstate "aws_ecs_service.main.name" }}
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
plugins:
  tfstate:
    path: terraform.tfstate

CodeDeployを使用した設定

region: ap-northeast-1
cluster: my-cluster
service: my-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m
codedeploy:
  application_name: my-app
  deployment_group_name: my-deployment-group
  termination_wait_time: 300
appspec:
  create: true

Jsonnetを使用した設定

local env = std.extVar('env');
local region = std.extVar('region');

{
  region: region,
  cluster: 'my-cluster',
  service: 'my-service-' + env,
  service_definition: 'ecs-service-def-' + env + '.json',
  task_definition: 'ecs-task-def-' + env + '.json',
  timeout: '10m',
  plugins: {
    tfstate: {
      path: 'terraform.tfstate'
    }
  },
  envs: [
    'APP_NAME',
    'APP_ENV'
  ],
  env_file: '.env.' + env
}

複数環境の設定

region: ap-northeast-1
cluster: my-cluster
service: my-service-{{ env "ENV" }}
service_definition: ecs-service-def-{{ env "ENV" }}.json
task_definition: ecs-task-def-{{ env "ENV" }}.json
timeout: 10m
envs:
  - APP_NAME
  - APP_ENV
env_file: .env.{{ env "ENV" }}
vars:
  env: {{ env "ENV" }}
