設定ファイル構造

ecspresso 用の設定ファイル（YAML、JSON、または Jsonnet 形式）はconfig.goで定義されており、Config構造体として実装されています：

region: ap-northeast-1 # または AWS_REGION 環境変数
cluster: default
service: myservice
task_definition: taskdef.json
service_definition: service.json # (オプション)
appspec: appspec.yaml # (オプション、CodeDeployを使用する場合)
timeout: 5m # デフォルトは 10m
plugins:
  - name: tfstate
    config:
      path: terraform.tfstate
  - name: ssm
  - name: cloudformation
    config:
      stack_names:
        - my-stack-1
  - name: secretsmanager
ignore:
  tags:
    - ecspresso:ignore # サービスとタスク定義のタグを無視

内部実装

Config構造体には以下の主要なフィールドがあります：

  • Region: AWS リージョン
  • Cluster: ECS クラスター名
  • Service: ECS サービス名
  • TaskDefinition: タスク定義ファイルのパス
  • ServiceDefinition: サービス定義ファイルのパス（オプション）
  • AppSpec: CodeDeploy AppSpec ファイルのパス（オプション）
  • Timeout: 操作のタイムアウト時間
  • Plugins: プラグイン設定のリスト
  • CodeDeploy: CodeDeploy 関連の設定
  • RequiredVersion: 必要な ecspresso のバージョン制約

設定読み込みプロセス

設定ファイルの読み込みはconfig.goのNewConfig関数で実装されています。内部的には以下のステップで処理されます：

flowchart TD A[設定ファイル] --> B[newLoader関数] B --> C{ファイル形式} C -->|YAML| D[YAMLローダー] C -->|JSON| E[JSONローダー] C -->|Jsonnet| F[Jsonnetローダー] D --> G[テンプレート変数置換] E --> G F --> G G --> H[Config構造体へのデシリアライズ] H --> I[プラグインの初期化] I --> J[設定の検証]
  1. newLoader関数でファイル形式（YAML、JSON、Jsonnet）に応じたローダーを作成
  2. ファイルを読み込み、テンプレート変数を置換
  3. Config構造体にデシリアライズ
  4. プラグインの初期化と設定
  5. 設定の検証（必須フィールドの確認など）

ecspresso deployは以下のように動作します：

  • task-definitionファイル（JSONまたはJsonnet）から新しいタスク定義を登録します。
    • JSONファイル内の{{ env `FOO` `bar` }}構文を環境変数 "FOO" の値で置き換えます。
      • "FOO" が定義されていない場合、"bar" で置き換えられます
    • JSONファイル内の{{ must_env `FOO` }}構文を環境変数 "FOO" の値で置き換えます。
      • "FOO" が定義されていない場合、即座に中止します。
  • service_definitionファイル（JSONまたはJsonnet）によってサービスタスクを更新します。
  • サービスが安定するまで待機します。

テンプレート構文

ecspressoはテンプレートファイルをレンダリングするためにGo の text/template 標準パッケージを使用し、YAMLまたはJSONとして解析します。

Jsonnetを使用する場合、ecspressoはまずJsonnetファイルをレンダリングし、その後text/templateとして解析します。テンプレート関数の構文{{}}はJsonnet構文と競合するため、テンプレート関数は "{{ ... }}" を使用した文字列値のみをレンダリングできます。非文字列値をレンダリングするには、代わりにJsonnet関数の使用を検討してください。

設定ファイルとタスク/サービス定義ファイルはgo-configによって読み込まれ、テンプレート関数env、must_env、json_escapeを提供します。

テンプレート関数

デフォルトでは、ecspressoは以下のテンプレート関数を提供します：

env

"{{ env `NAME` `default value` }}"

このプレースホルダーを環境変数 NAME の値で置き換えます。設定されていない場合は "default value" がデフォルト値になります。

must_env

"{{ must_env `NAME` }}"

このプレースホルダーを環境変数 NAME の値で置き換えます。設定されていない場合、ecspressoはパニックして強制的に停止します。

must_envで重要な値を定義することで、実行前にこれらの値が設定されていることを確認し、意図しないデプロイを防止するのに役立ちます。

json_escape

"{{ must_env `JSON_VALUE` | json_escape }}"

値をJSON文字列としてエスケープします。これは引用符などのエスケープが必要な文字列として値を埋め込む場合に便利です。

プラグインが提供するテンプレート関数

ecspressoはプラグインを通じていくつかのテンプレート関数も追加します。プラグインセクションを参照してください。

テンプレート関数の実装はconfig.goのnewLoader関数内で行われています。各関数はtext/templateパッケージのFuncMapとして定義され、テンプレート処理時に使用されます。
