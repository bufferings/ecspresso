設定ファイル

ecspresso 用の設定ファイル（YAML、JSON、または Jsonnet 形式）：

region: ap-northeast-1 # または AWS_REGION 環境変数
cluster: default
service: myservice
task_definition: taskdef.json
timeout: 5m # デフォルトは 10m
ignore:
  tags:
    - ecspresso:ignore # サービスとタスク定義のタグを無視

ecspresso deployは以下のように動作します：

  • task-definitionファイル（JSONまたはJsonnet）から新しいタスク定義を登録します。
    • JSONファイル内の{{ env `FOO` `bar` }}構文を環境変数 "FOO" の値で置き換えます。
      • "FOO" が定義されていない場合、"bar" で置き換えられます
    • JSONファイル内の{{ must_env `FOO` }}構文を環境変数 "FOO" の値で置き換えます。
      • "FOO" が定義されていない場合、即座に中止します。
  • service_definitionファイル（JSONまたはJsonnet）によってサービスタスクを更新します。
  • サービスが安定するまで待機します。

設定ファイルとタスク/サービス定義ファイルはgo-configによって読み込まれ、テンプレート関数env、must_env、json_escapeを提供します。

テンプレート構文

ecspressoはテンプレートファイルをレンダリングするためにGo の text/template 標準パッケージを使用し、YAMLまたはJSONとして解析します。

Jsonnetを使用する場合、ecspressoはまずJsonnetファイルをレンダリングし、その後text/templateとして解析します。テンプレート関数の構文{{}}はJsonnet構文と競合するため、テンプレート関数は "{{ ... }}" を使用した文字列値のみをレンダリングできます。非文字列値をレンダリングするには、代わりにJsonnet関数の使用を検討してください。

デフォルトでは、ecspressoは以下のテンプレート関数を提供します：

env

"{{ env `NAME` `default value` }}"

このプレースホルダーを環境変数 NAME の値で置き換えます。設定されていない場合は "default value" がデフォルト値になります。

must_env

"{{ must_env `NAME` }}"

このプレースホルダーを環境変数 NAME の値で置き換えます。設定されていない場合、ecspressoはパニックして強制的に停止します。

must_envで重要な値を定義することで、実行前にこれらの値が設定されていることを確認し、意図しないデプロイを防止するのに役立ちます。

json_escape

"{{ must_env `JSON_VALUE` | json_escape }}"

値をJSON文字列としてエスケープします。これは引用符などのエスケープが必要な文字列として値を埋め込む場合に便利です。

プラグインが提供するテンプレート関数

ecspressoはプラグインを通じていくつかのテンプレート関数も追加します。プラグインセクションを参照してください。
