コア機能概要

ecspressoのコア機能は、ecspresso.goファイルに実装されているApp構造体を中心に構築されています。この構造体は、AWS ECSサービスとタスク定義の管理、デプロイメント、検証、差分比較などの機能を提供します。

graph TD A[App構造体] --> B[設定管理] A --> C[デプロイメント管理] A --> D[タスク管理] A --> E[リソース検証] A --> F[リソース差分] B --> G[設定ファイル読込] B --> H[テンプレート処理] B --> I[プラグイン統合] C --> J[ECSデプロイ] C --> K[CodeDeployデプロイ] C --> L[ロールバック] D --> M[タスク実行] D --> N[タスク停止] D --> O[コマンド実行] E --> P[リソース存在確認] E --> Q[権限確認] F --> R[タスク定義比較] F --> S[サービス定義比較]

ecspressoのコア機能は以下の要素で構成されています：

  • 設定管理: 設定ファイルの読み込み、解析、検証を行います。
  • デプロイメント管理: ECSサービスのデプロイ、ロールバック、スケーリングを行います。
  • タスク管理: ECSタスクの実行、停止、コマンド実行を行います。
  • リソース検証: AWS上のリソースが正しく設定されているかを検証します。
  • リソース差分: ローカル設定とAWS上の設定の差分を表示します。

App構造体

App構造体は、ecspressoの中心的なコンポーネントで、以下のフィールドを持っています：

type App struct {
    ecs            ECS
    codedeploy     CodeDeploy
    autoScaling    AutoScaling
    cloudWatchLogs CloudWatchLogs
    iam            IAM
    ssm            SSM
    elb            ELB
    elbv2          ELBV2
    serviceLister  ServiceLister
    taskDefinition *TaskDefinition
    service        *Service
    config         *Config
    verifier       *verifier
    loader         *configLoader
    vpcLattice     VPCLattice
    ecsta          *ecsta.Ecsta
}

主要なフィールドの説明：

  • ecs: ECS APIクライアントのインターフェース
  • codedeploy: CodeDeploy APIクライアントのインターフェース
  • autoScaling: Application Auto Scaling APIクライアントのインターフェース
  • cloudWatchLogs: CloudWatch Logs APIクライアントのインターフェース
  • taskDefinition: タスク定義情報
  • service: サービス情報
  • config: 設定情報
  • verifier: リソース検証機能
  • loader: 設定ファイル読み込み機能

App構造体は、以下のような主要なメソッドを提供しています：

AWS統合

ecspressoは、AWS SDK for Go v2を使用して、以下のAWSサービスと統合しています：

graph TD A[ecspresso] --> B[AWS ECS] A --> C[AWS CodeDeploy] A --> D[AWS Application Auto Scaling] A --> E[AWS CloudWatch Logs] A --> F[AWS IAM] A --> G[AWS SSM] A --> H[AWS ELB] A --> I[AWS ELBV2] A --> J[AWS VPC Lattice] A --> K[AWS Secrets Manager]

各AWSサービスとの統合は、インターフェースを通じて行われています。これにより、テストの容易さと拡張性が向上しています。例えば、ECSサービスとの統合は以下のようなインターフェースで定義されています：

type ECS interface {
    DescribeServices(ctx context.Context, params *ecs.DescribeServicesInput, optFns ...func(*ecs.Options)) (*ecs.DescribeServicesOutput, error)
    DescribeTaskDefinition(ctx context.Context, params *ecs.DescribeTaskDefinitionInput, optFns ...func(*ecs.Options)) (*ecs.DescribeTaskDefinitionOutput, error)
    RegisterTaskDefinition(ctx context.Context, params *ecs.RegisterTaskDefinitionInput, optFns ...func(*ecs.Options)) (*ecs.RegisterTaskDefinitionOutput, error)
    DeregisterTaskDefinition(ctx context.Context, params *ecs.DeregisterTaskDefinitionInput, optFns ...func(*ecs.Options)) (*ecs.DeregisterTaskDefinitionOutput, error)
    UpdateService(ctx context.Context, params *ecs.UpdateServiceInput, optFns ...func(*ecs.Options)) (*ecs.UpdateServiceOutput, error)
    CreateService(ctx context.Context, params *ecs.CreateServiceInput, optFns ...func(*ecs.Options)) (*ecs.CreateServiceOutput, error)
    RunTask(ctx context.Context, params *ecs.RunTaskInput, optFns ...func(*ecs.Options)) (*ecs.RunTaskOutput, error)
    DescribeTasks(ctx context.Context, params *ecs.DescribeTasksInput, optFns ...func(*ecs.Options)) (*ecs.DescribeTasksOutput, error)
    ListTasks(ctx context.Context, params *ecs.ListTasksInput, optFns ...func(*ecs.Options)) (*ecs.ListTasksOutput, error)
    StopTask(ctx context.Context, params *ecs.StopTaskInput, optFns ...func(*ecs.Options)) (*ecs.StopTaskOutput, error)
    ListTaskDefinitions(ctx context.Context, params *ecs.ListTaskDefinitionsInput, optFns ...func(*ecs.Options)) (*ecs.ListTaskDefinitionsOutput, error)
}

コアワークフロー

ecspressoの一般的なワークフローは以下のようになります：

sequenceDiagram participant User as ユーザー participant CLI as CLIコマンド participant App as App構造体 participant Config as 設定管理 participant AWS as AWSサービス User->>CLI: コマンド実行 CLI->>App: コマンドディスパッチ App->>Config: 設定ファイル読込 Config-->>App: 設定情報 alt デプロイの場合 App->>AWS: タスク定義登録 AWS-->>App: タスク定義ARN App->>AWS: サービス更新 AWS-->>App: サービス更新結果 App->>AWS: サービス状態確認 AWS-->>App: サービス状態 else タスク実行の場合 App->>AWS: タスク実行 AWS-->>App: タスク情報 App->>AWS: タスク状態確認 AWS-->>App: タスク状態 else 検証の場合 App->>AWS: リソース情報取得 AWS-->>App: リソース情報 App->>App: リソース検証 else 差分確認の場合 App->>AWS: 現在の設定取得 AWS-->>App: 現在の設定 App->>App: ローカル設定と比較 end App-->>CLI: 結果 CLI-->>User: 結果表示

このワークフローは、App構造体のメソッドを通じて実装されています。各メソッドは、特定の機能を提供し、必要に応じてAWSサービスと通信します。

拡張ポイント

ecspressoは、以下の拡張ポイントを提供しています：

graph TD A[拡張ポイント] --> B[プラグインシステム] A --> C[テンプレート関数] A --> D[Jsonnet統合] A --> E[外部コマンド] B --> F[tfstateプラグイン] B --> G[CloudFormationプラグイン] B --> H[SSMプラグイン] B --> I[Secrets Managerプラグイン] C --> J[環境変数参照] C --> K[プラグイン関数] D --> L[Jsonnetファイル] D --> M[Jsonnetネイティブ関数] E --> N[外部コマンド実行] E --> O[外部プラグイン]

これらの拡張ポイントにより、ecspressoの機能を拡張し、特定のユースケースに合わせてカスタマイズすることができます。

プラグインシステム

プラグインシステムは、plugin.goファイルに実装されており、ConfigPluginインターフェースを通じて機能を拡張します。詳細については、4.3 プラグインシステムを参照してください。

テンプレート関数

テンプレート関数は、設定ファイル内で使用できる関数で、環境変数の参照やプラグインが提供する関数などがあります。詳細については、4.1 設定ファイル形式を参照してください。

Jsonnet統合

Jsonnet統合は、設定ファイルをJsonnetで記述できるようにする機能です。詳細については、4.4 Jsonnet統合を参照してください。

外部コマンド

外部コマンドは、ecspressoから外部コマンドを実行する機能です。詳細については、4.3 プラグインシステムの「外部コマンドプラグイン」セクションを参照してください。
