定義ファイル概要

ecspressoでは、タスク定義とサービス定義を別々のJSONまたはYAMLファイルで管理します。これらのファイルは、ecspresso.ymlのtask_definitionとservice_definitionパラメータで指定します。

graph TD A[ecspresso.yml] --> B[task_definition] A --> C[service_definition] B --> D[タスク定義ファイル] C --> E[サービス定義ファイル] D --> F[コンテナ定義] D --> G[タスクロール] D --> H[実行ロール] D --> I[ネットワークモード] E --> J[クラスター] E --> K[サービス名] E --> L[タスク定義] E --> M[デプロイ設定] E --> N[ネットワーク設定] E --> O[ロードバランサー設定]

定義ファイルは、以下のような目的で使用されます：

  • ECSタスク定義とサービス定義を宣言的に管理する
  • バージョン管理システムで変更履歴を追跡する
  • 環境間で設定を共有する
  • CI/CDパイプラインで自動デプロイを行う

タスク定義ファイル

タスク定義ファイルは、ECSタスクの実行方法を定義するJSONまたはYAMLファイルです。このファイルは、AWS ECSのタスク定義と互換性があり、以下のような主要なフィールドを持っています：

{
  "family": "string",
  "taskRoleArn": "string",
  "executionRoleArn": "string",
  "networkMode": "string",
  "containerDefinitions": [
    {
      "name": "string",
      "image": "string",
      "essential": boolean,
      "command": ["string", ...],
      "entryPoint": ["string", ...],
      "environment": [
        {
          "name": "string",
          "value": "string"
        },
        ...
      ],
      "secrets": [
        {
          "name": "string",
          "valueFrom": "string"
        },
        ...
      ],
      "logConfiguration": {
        "logDriver": "string",
        "options": {
          "string": "string",
          ...
        }
      },
      "portMappings": [
        {
          "containerPort": number,
          "hostPort": number,
          "protocol": "string"
        },
        ...
      ],
      "healthCheck": {
        "command": ["string", ...],
        "interval": number,
        "timeout": number,
        "retries": number,
        "startPeriod": number
      },
      "cpu": number,
      "memory": number,
      "memoryReservation": number
    },
    ...
  ],
  "volumes": [
    {
      "name": "string",
      "host": {
        "sourcePath": "string"
      },
      "dockerVolumeConfiguration": {
        "scope": "string",
        "autoprovision": boolean,
        "driver": "string",
        "driverOpts": {
          "string": "string",
          ...
        },
        "labels": {
          "string": "string",
          ...
        }
      },
      "efsVolumeConfiguration": {
        "fileSystemId": "string",
        "rootDirectory": "string",
        "transitEncryption": "string",
        "transitEncryptionPort": number,
        "authorizationConfig": {
          "accessPointId": "string",
          "iam": "string"
        }
      }
    },
    ...
  ],
  "placementConstraints": [
    {
      "type": "string",
      "expression": "string"
    },
    ...
  ],
  "requiresCompatibilities": ["string", ...],
  "cpu": "string",
  "memory": "string",
  "tags": [
    {
      "key": "string",
      "value": "string"
    },
    ...
  ]
}

主要なフィールドの説明：

  • family: タスク定義のファミリー名
  • taskRoleArn: タスクに割り当てるIAMロールのARN
  • executionRoleArn: タスク実行に使用するIAMロールのARN
  • networkMode: ネットワークモード（awsvpc, bridge, host, none）
  • containerDefinitions: コンテナの定義のリスト
  • volumes: ボリュームの定義のリスト
  • placementConstraints: タスクのプレースメント制約のリスト
  • requiresCompatibilities: タスクの互換性要件（EC2, FARGATE）
  • cpu: タスクに割り当てるCPUユニット
  • memory: タスクに割り当てるメモリ（MB）
  • tags: タスク定義に付けるタグのリスト

コンテナ定義の主要なフィールドの説明：

  • name: コンテナ名
  • image: コンテナイメージ
  • essential: 必須コンテナかどうか
  • command: コンテナで実行するコマンド
  • entryPoint: コンテナのエントリーポイント
  • environment: 環境変数のリスト
  • secrets: シークレットのリスト
  • logConfiguration: ログ設定
  • portMappings: ポートマッピングのリスト
  • healthCheck: ヘルスチェック設定
  • cpu: コンテナに割り当てるCPUユニット
  • memory: コンテナに割り当てるメモリ（MB）
  • memoryReservation: コンテナに予約するメモリ（MB）

サービス定義ファイル

サービス定義ファイルは、ECSサービスの実行方法を定義するJSONまたはYAMLファイルです。このファイルは、AWS ECSのサービス定義と互換性があり、以下のような主要なフィールドを持っています：

{
  "cluster": "string",
  "serviceName": "string",
  "taskDefinition": "string",
  "loadBalancers": [
    {
      "targetGroupArn": "string",
      "containerName": "string",
      "containerPort": number
    },
    ...
  ],
  "serviceRegistries": [
    {
      "registryArn": "string",
      "port": number,
      "containerName": "string",
      "containerPort": number
    },
    ...
  ],
  "desiredCount": number,
  "launchType": "string",
  "platformVersion": "string",
  "deploymentConfiguration": {
    "deploymentCircuitBreaker": {
      "enable": boolean,
      "rollback": boolean
    },
    "maximumPercent": number,
    "minimumHealthyPercent": number
  },
  "placementConstraints": [
    {
      "type": "string",
      "expression": "string"
    },
    ...
  ],
  "placementStrategy": [
    {
      "type": "string",
      "field": "string"
    },
    ...
  ],
  "networkConfiguration": {
    "awsvpcConfiguration": {
      "subnets": ["string", ...],
      "securityGroups": ["string", ...],
      "assignPublicIp": "string"
    }
  },
  "healthCheckGracePeriodSeconds": number,
  "schedulingStrategy": "string",
  "deploymentController": {
    "type": "string"
  },
  "tags": [
    {
      "key": "string",
      "value": "string"
    },
    ...
  ],
  "enableECSManagedTags": boolean,
  "propagateTags": "string",
  "enableExecuteCommand": boolean
}

主要なフィールドの説明：

  • cluster: ECSクラスター名
  • serviceName: サービス名
  • taskDefinition: タスク定義のARNまたはファミリー名:リビジョン
  • loadBalancers: ロードバランサーの設定のリスト
  • serviceRegistries: サービスレジストリの設定のリスト
  • desiredCount: 希望するタスク数
  • launchType: 起動タイプ（EC2, FARGATE）
  • platformVersion: プラットフォームバージョン
  • deploymentConfiguration: デプロイ設定
  • placementConstraints: タスクのプレースメント制約のリスト
  • placementStrategy: タスクのプレースメント戦略のリスト
  • networkConfiguration: ネットワーク設定
  • healthCheckGracePeriodSeconds: ヘルスチェックの猶予期間（秒）
  • schedulingStrategy: スケジューリング戦略（REPLICA, DAEMON）
  • deploymentController: デプロイコントローラーの設定
  • tags: サービスに付けるタグのリスト
  • enableECSManagedTags: ECS管理タグを有効にするかどうか
  • propagateTags: タグの伝播方法（TASK_DEFINITION, SERVICE, NONE）
  • enableExecuteCommand: ECS Execを有効にするかどうか

テンプレート関数の使用

ecspressoでは、タスク定義ファイルとサービス定義ファイル内でテンプレート関数を使用することができます。これにより、環境変数やプラグインが提供する値を参照できます。

graph TD A[テンプレート関数] --> B[環境変数関数] A --> C[プラグイン関数] A --> D[ユーティリティ関数] B --> E[env] B --> F[must_env] C --> G[tfstate] C --> H[cfn_output] C --> I[ssm] C --> J[secretsmanager_arn] D --> K[json] D --> L[yaml] D --> M[base64]

環境変数関数の使用例

{
  "family": "{{ env "APP_NAME" }}",
  "taskRoleArn": "{{ env "TASK_ROLE_ARN" }}",
  "executionRoleArn": "{{ env "EXECUTION_ROLE_ARN" }}",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "web",
      "image": "{{ env "ECR_REPOSITORY" }}:{{ env "IMAGE_TAG" }}",
      "essential": true,
      "environment": [
        {
          "name": "NODE_ENV",
          "value": "{{ env "NODE_ENV" }}"
        }
      ]
    }
  ]
}

プラグイン関数の使用例

{
  "family": "my-app",
  "taskRoleArn": "{{ tfstate "aws_iam_role.ecs_task_role.arn" }}",
  "executionRoleArn": "{{ tfstate "aws_iam_role.ecs_task_execution_role.arn" }}",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "web",
      "image": "{{ env "ECR_REPOSITORY" }}:{{ env "IMAGE_TAG" }}",
      "essential": true,
      "secrets": [
        {
          "name": "DATABASE_URL",
          "valueFrom": "{{ secretsmanager_arn "my-app/database-url" }}"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "{{ cfn_output "MyStack" "LogGroupName" }}",
          "awslogs-region": "{{ env "AWS_REGION" }}",
          "awslogs-stream-prefix": "web"
        }
      }
    }
  ]
}

定義例

以下は、ecspressoで使用できるタスク定義とサービス定義の例です：

基本的なタスク定義

{
  "family": "my-app",
  "executionRoleArn": "arn:aws:iam::123456789012:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::123456789012:role/ecsTaskRole",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "web",
      "image": "123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:latest",
      "essential": true,
      "portMappings": [
        {
          "containerPort": 80,
          "hostPort": 80,
          "protocol": "tcp"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/my-app",
          "awslogs-region": "ap-northeast-1",
          "awslogs-stream-prefix": "web"
        }
      },
      "environment": [
        {
          "name": "NODE_ENV",
          "value": "production"
        }
      ],
      "cpu": 256,
      "memory": 512
    }
  ],
  "requiresCompatibilities": [
    "FARGATE"
  ],
  "cpu": "256",
  "memory": "512"
}

  },

    "awsvpcConfiguration": {
      "subnets": [
        "subnet-12345678",

      ],
      "securityGroups": [
        "sg-12345678"

  },
  "loadBalancers": [

  ],
  "healthCheckGracePeriodSeconds": 60,
