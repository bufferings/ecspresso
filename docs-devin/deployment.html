デプロイの実装

ecspressoのデプロイ機能はdeploy.goファイルに実装されています。中心となるのはApp構造体のDeployメソッドで、以下のような処理を行います：

func (d *App) Deploy(ctx context.Context, opt DeployOption) error {
	ctx, cancel := d.Start(ctx)
	defer cancel()
	
	// 設定の読み込みと検証
	// タスク定義の登録（オプションによる）
	// デプロイ戦略の選択（ECSまたはCodeDeploy）
	// デプロイの実行
	// サービスの安定性待機（オプションによる）
	
	return nil
}

デプロイ処理は、選択されたデプロイ戦略に基づいて実行されます。ECSローリングデプロイとCodeDeployブルー/グリーンデプロイの2つの戦略がサポートされています。

デプロイ戦略

ecspressoは以下の2つのデプロイ戦略をサポートしています：

flowchart LR Deploy[デプロイ] --> Strategy{戦略} Strategy -->|ECS| ECS[ECSローリングデプロイ] Strategy -->|CodeDeploy| CD[CodeDeployブルー/グリーン] ECS --> Rolling[ローリングアップデート] CD --> BlueGreen[ブルー/グリーンデプロイ]

ECSローリングデプロイ

ECSローリングデプロイは、タスクを徐々に置き換えていく戦略です。この戦略はdeployECS関数で実装されています：

func (d *App) deployECS(ctx context.Context, opt DeployOption) error {
	// サービス定義の取得
	// タスク定義の更新
	// サービスの更新
	// サービスの安定性待機
	return nil
}

CodeDeployブルー/グリーンデプロイ

CodeDeployブルー/グリーンデプロイは、新しい環境（グリーン）を作成し、トラフィックを徐々に移行する戦略です。この戦略はdeployCodeDeploy関数で実装されています：

func (d *App) deployCodeDeploy(ctx context.Context, opt DeployOption) error {
	// AppSpecの生成
	// デプロイメントの作成
	// デプロイメントの待機
	return nil
}

デプロイオプション

デプロイ操作はDeployOption構造体で設定できます：

type DeployOption struct {
	DryRun                bool   `help:"dry run" default:"false"`
	SkipTaskDefinition    bool   `help:"skip register task definition" default:"false" negatable:""`
	Revision              string `help:"revision of task definition"`
	Timeout               string `help:"timeout" default:""`
	ForceNewDeployment    bool   `help:"force new deployment for service" default:"false" negatable:""`
	NoWait                bool   `help:"don't wait for service stable" default:"false" negatable:""`
	SuspendAutoScaling    bool   `help:"suspend application auto scaling while deploy" default:"false" negatable:""`
	RollbackEvents        string `help:"rollback events on deploy" default:"" enum:"" enum:"DEPLOYMENT_FAILURE" enum:"DEPLOYMENT_STOP_ON_ALARM" enum:"DEPLOYMENT_STOP_ON_REQUEST" enum:"DEPLOYMENT_FAILURE,DEPLOYMENT_STOP_ON_ALARM" enum:"DEPLOYMENT_FAILURE,DEPLOYMENT_STOP_ON_REQUEST" enum:"DEPLOYMENT_STOP_ON_ALARM,DEPLOYMENT_STOP_ON_REQUEST" enum:"DEPLOYMENT_FAILURE,DEPLOYMENT_STOP_ON_ALARM,DEPLOYMENT_STOP_ON_REQUEST"`
	UpdateService         bool   `help:"update service definition" default:"true" negatable:""`
	LatestTaskDefinition  bool   `help:"deploy with the latest task definition without registering" default:"false" negatable:""`
	WaitUntil             string `help:"wait until the service reaches a certain state" default:"STABLE" enum:"RUNNING" enum:"STABLE" enum:"COMPLETED"`
	PutLogs               bool   `help:"put logs to CloudWatch Logs on verify" default:"true" negatable:""`
}

主要なオプションには以下があります：

  • DryRun：実際の変更を行わずに、何が行われるかをシミュレーションします。
  • SkipTaskDefinition：タスク定義の登録をスキップします。
  • ForceNewDeployment：サービスの強制的な新しいデプロイメントを行います。
  • NoWait：サービスの安定性を待たずに終了します。
  • WaitUntil：サービスが特定の状態に達するまで待機します（RUNNING、STABLE、COMPLETED）。

デプロイプロセス

ecspressoのデプロイプロセスは以下のステップで構成されています：

sequenceDiagram participant App participant Config as 設定管理 participant Verify as リソース検証 participant TaskDef as タスク定義 participant Service as サービス participant AWS as AWS API App->>Config: 設定読み込み App->>Verify: リソース検証 Verify-->>App: 検証結果 alt タスク定義の登録が必要 App->>TaskDef: タスク定義の準備 App->>AWS: RegisterTaskDefinition AWS-->>App: タスク定義ARN end alt デプロイ戦略 = ECS App->>Service: サービス定義の準備 App->>AWS: UpdateService AWS-->>App: サービス更新結果 opt 安定性待機が必要 App->>AWS: DescribeServices（繰り返し） AWS-->>App: サービス状態 end else デプロイ戦略 = CodeDeploy App->>Service: AppSpecの準備 App->>AWS: CreateDeployment AWS-->>App: デプロイメントID opt デプロイメント待機が必要 App->>AWS: GetDeployment（繰り返し） AWS-->>App: デプロイメント状態 end end

デプロイプロセスの各ステップは、deploy.goファイル内の対応する関数で実装されています。

ロールバック機能

ecspressoは、デプロイメントの問題が発生した場合にサービスをロールバックする機能を提供しています。この機能はrollback.goファイルに実装されています。

func (d *App) Rollback(ctx context.Context, opt RollbackOption) error {
	ctx, cancel := d.Start(ctx)
	defer cancel()
	
	// 以前のタスク定義の取得
	// サービスの更新
	// サービスの安定性待機
	
	return nil
}

ロールバックオプションはRollbackOption構造体で設定できます：

type RollbackOption struct {
	DryRun                  bool   `help:"dry run" default:"false"`
	DeregisterTaskDefinition bool   `help:"deregister rolled back task definition" default:"false" negatable:""`
	NoWait                  bool   `help:"don't wait for service stable" default:"false" negatable:""`
	Revision                int    `help:"revision of task definition to rollback" default:"0"`
}

また、デプロイ時にRollbackEventsオプションを設定することで、特定のイベント（デプロイメント失敗、アラームによる停止、リクエストによる停止）が発生した場合に自動的にロールバックすることもできます。
