開発概要

ecspressoは、Go言語で開発されたオープンソースプロジェクトです。このセクションでは、ecspressoの開発に貢献するための情報を提供します。

graph TD A[開発プロセス] --> B[環境設定] A --> C[コード変更] A --> D[テスト] A --> E[プルリクエスト] A --> F[レビュー] A --> G[マージ] B --> B1[Goのインストール] B --> B2[依存関係のインストール] B --> B3[リポジトリのクローン] C --> C1[機能追加] C --> C2[バグ修正] C --> C3[リファクタリング] D --> D1[ユニットテスト] D --> D2[統合テスト] D --> D3[リントチェック] E --> E1[ブランチ作成] E --> E2[コミット] E --> E3[プッシュ] E --> E4[PRの作成] F --> F1[コードレビュー] F --> F2[CI/CDチェック] F --> F3[フィードバック対応] G --> G1[マージ] G --> G2[リリース]

ecspressoの開発に貢献するには、以下の手順に従います：

  1. GitHubリポジトリをフォークする
  2. ローカル環境を設定する
  3. 機能追加やバグ修正を行う
  4. テストを実行する
  5. プルリクエストを作成する
  6. レビューとフィードバックに対応する

開発環境

ecspressoの開発環境を設定するには、以下の手順に従います：

前提条件

  • Go 1.16以上
  • Git
  • AWS CLI（テスト用）
  • Docker（テスト用）

リポジトリのクローン

git clone https://github.com/kayac/ecspresso.git
cd ecspresso

依存関係のインストール

go mod download

ビルド

make build

テスト

make test

リントチェック

make lint

開発用バイナリの実行

./bin/ecspresso --help

ブランチ作成

新しい機能やバグ修正を行う場合は、新しいブランチを作成します：

git checkout -b feature/new-feature

コード変更

コードを変更し、テストを追加または更新します。変更は小さく、焦点を絞ったものにすることをお勧めします。

テスト実行

変更がテストに合格することを確認します：

make test

コミット

変更をコミットします。コミットメッセージは明確で説明的なものにしてください：

git add .
git commit -m "Add new feature: description of the feature"

プッシュ

変更をフォークリポジトリにプッシュします：

git push origin feature/new-feature

プルリクエスト作成

GitHubでプルリクエストを作成します。プルリクエストの説明には、以下の情報を含めてください：

  • 変更の目的
  • 変更の概要
  • テスト方法
  • 関連する問題（ある場合）

レビューとフィードバック

プルリクエストがレビューされ、フィードバックが提供されます。フィードバックに基づいて変更を行い、同じブランチにプッシュします。

マージ

プルリクエストが承認されると、メンテナによってマージされます。

コーディングガイドライン

ecspressoのコードベースに貢献する際は、以下のガイドラインに従ってください：

コードスタイル

  • Go標準のコードスタイルに従う
  • gofmtを使用してコードをフォーマットする
  • golintとgo vetのエラーを修正する
  • エラー処理を適切に行う
  • コメントを追加して、複雑なロジックを説明する

テスト

  • 新しい機能には単体テストを追加する
  • バグ修正には、バグを再現するテストを追加する
  • テストカバレッジを維持または向上させる
  • モックを使用して外部依存関係をテストする

ドキュメント

  • 新しい機能にはドキュメントを追加する
  • 既存のドキュメントを更新して、変更を反映する
  • コードにはGoDocコメントを追加する

コミットメッセージ

  • 明確で説明的なコミットメッセージを書く
  • コミットメッセージの最初の行は50文字以内にする
  • 必要に応じて、詳細な説明を追加する
  • 関連する問題番号を含める（例：「Fix #123」）

ドキュメント

ecspressoのドキュメントは、以下の場所にあります：

  • README.md: プロジェクトの概要と基本的な使用方法
  • docs-devin/: 詳細なドキュメント
  • GoDoc: コードのドキュメント

ドキュメントを更新する場合は、以下のガイドラインに従ってください：

  • 明確で簡潔な言葉を使用する
  • 例を提供して、概念を説明する
  • スクリーンショットや図を使用して、視覚的に説明する
  • ドキュメントの構造を維持する
  • リンク切れや古い情報を修正する

ドキュメントの変更も、コード変更と同じプロセスでプルリクエストを作成してください。
