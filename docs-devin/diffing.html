リソース差分概要

ecspressoのリソース差分機能は、ローカル設定とAWS上の設定の差分を表示するために使用されます。この機能はdiff.goファイルに実装されており、タスク定義とサービス定義の差分を表示します。

graph TD A[リソース差分] --> B[タスク定義差分] A --> C[サービス定義差分] B --> D[コンテナ定義差分] B --> E[タスクロール差分] B --> F[実行ロール差分] B --> G[ネットワークモード差分] B --> H[ボリューム差分] C --> I[クラスター差分] C --> J[サービスロール差分] C --> K[ロードバランサー差分] C --> L[デプロイ設定差分] C --> M[ネットワーク設定差分]

リソース差分は、以下のような目的で使用されます：

  • デプロイ前に変更内容を確認する
  • 現在のAWS上の設定とローカル設定の違いを確認する
  • 設定ドリフトを検出する
  • 変更履歴を追跡する

差分の実装

ecspressoのリソース差分機能は、diff.goファイルに実装されています。中心となるのはApp構造体のDiffメソッドで、以下のような処理を行います：

func (d *App) Diff(ctx context.Context, opt DiffOption) error {
    var result string
    if !opt.OnlyService {
        td, err := d.taskDefinitionForDiff(ctx)
        if err != nil {
            return err
        }
        if td != nil {
            tdDiff, err := d.diffTaskDefinition(ctx, td, opt)
            if err != nil {
                return err
            }
            result += tdDiff
        }
    }
    if !opt.OnlyTaskDefinition {
        sv, err := d.serviceForDiff(ctx)
        if err != nil {
            return err
        }
        if sv != nil {
            svDiff, err := d.diffService(ctx, sv, opt)
            if err != nil {
                return err
            }
            result += svDiff
        }
    }
    if result == "" {
        d.Log("No diff")
        return nil
    }
    fmt.Println(result)
    return nil
}

タスク定義の差分は、diffTaskDefinitionメソッドで計算されます：

func (d *App) diffTaskDefinition(ctx context.Context, td *TaskDefinition, opt DiffOption) (string, error) {
    newTd, err := d.LoadTaskDefinition(d.config.TaskDefinitionPath)
    if err != nil {
        return "", err
    }
    if !opt.WithSecrets {
        newTd = newTd.RemoveSecrets()
        td = td.RemoveSecrets()
    }
    return d.diff(td, newTd, opt)
}

サービス定義の差分は、diffServiceメソッドで計算されます：

func (d *App) diffService(ctx context.Context, sv *Service, opt DiffOption) (string, error) {
    newSv, err := d.LoadService(d.config.ServiceDefinitionPath)
    if err != nil {
        return "", err
    }
    if !opt.WithSecrets {
        newSv = newSv.RemoveSecrets()
        sv = sv.RemoveSecrets()
    }
    return d.diff(sv, newSv, opt)
}

差分の表示には、diffメソッドが使用されます：

func (d *App) diff(from, to interface{}, opt DiffOption) (string, error) {
    fromJSON, err := json.MarshalIndent(from, "", "  ")
    if err != nil {
        return "", err
    }
    toJSON, err := json.MarshalIndent(to, "", "  ")
    if err != nil {
        return "", err
    }
    if opt.Unified {
        diff := difflib.UnifiedDiff{
            A:        difflib.SplitLines(string(fromJSON)),
            B:        difflib.SplitLines(string(toJSON)),
            FromFile: "Current",
            ToFile:   "Local",
            Context:  3,
        }
        return difflib.GetUnifiedDiffString(diff)
    }
    dmp := diffmatchpatch.New()
    diffs := dmp.DiffMain(string(fromJSON), string(toJSON), false)
    var buf bytes.Buffer
    for _, diff := range diffs {
        switch diff.Type {
        case diffmatchpatch.DiffInsert:
            buf.WriteString("\x1b[32m" + diff.Text + "\x1b[0m")
        case diffmatchpatch.DiffDelete:
            buf.WriteString("\x1b[31m" + diff.Text + "\x1b[0m")
        case diffmatchpatch.DiffEqual:
            buf.WriteString(diff.Text)
        }
    }
    return buf.String(), nil
}

差分プロセス

ecspressoのリソース差分プロセスは、以下のステップで構成されています：

sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWSサービス User->>App: ecspresso diff App->>App: 設定ファイル読込 alt タスク定義の差分を表示する場合 App->>App: ローカルタスク定義読込 App->>AWS: DescribeTaskDefinition AWS-->>App: 現在のタスク定義 App->>App: タスク定義の差分計算 end alt サービス定義の差分を表示する場合 App->>App: ローカルサービス定義読込 App->>AWS: DescribeService AWS-->>App: 現在のサービス定義 App->>App: サービス定義の差分計算 end App-->>User: 差分表示

差分計算の主要なステップは以下の通りです：

タスク定義の差分計算

  1. ローカルのタスク定義ファイルを読み込む
  2. AWS上の現在のタスク定義を取得する
  3. 必要に応じてシークレットを削除する
  4. 両者をJSON形式に変換する
  5. JSONの差分を計算する
  6. 差分を表示する

サービス定義の差分計算

  1. ローカルのサービス定義ファイルを読み込む
  2. AWS上の現在のサービス定義を取得する
  3. 必要に応じてシークレットを削除する
  4. 両者をJSON形式に変換する
  5. JSONの差分を計算する
  6. 差分を表示する

差分オプション

ecspressoのリソース差分は、DiffOption構造体で設定できます：

type DiffOption struct {
    Unified           bool
    OnlyTaskDefinition bool
    OnlyService       bool
    WithSecrets       bool
}

主要なオプションの説明：

  • Unified: 統一形式で差分を表示する
  • OnlyTaskDefinition: タスク定義のみの差分を表示する
  • OnlyService: サービス定義のみの差分を表示する
  • WithSecrets: シークレットを含めて差分を表示する

これらのオプションは、diffコマンドのフラグとして指定できます：

ecspresso diff [オプション]
オプション 説明 デフォルト値
--unified 統一形式で差分を表示する false
--only-task-definition タスク定義のみの差分を表示する false
--only-service サービス定義のみの差分を表示する false
--with-secrets シークレットを含めて差分を表示する false

差分例

以下は、ecspressoのリソース差分の使用例です：

基本的な差分表示

ecspresso diff

この例では、タスク定義とサービス定義の両方の差分を表示します。シークレットは表示されません。

統一形式での差分表示

ecspresso diff --unified

この例では、統一形式（unified diff形式）で差分を表示します。これは、gitのdiffコマンドと同様の形式です。

タスク定義のみの差分表示

ecspresso diff --only-task-definition

この例では、タスク定義のみの差分を表示します。サービス定義の差分は表示されません。

サービス定義のみの差分表示

ecspresso diff --only-service

この例では、サービス定義のみの差分を表示します。タスク定義の差分は表示されません。

シークレットを含めた差分表示

ecspresso diff --with-secrets

この例では、シークレットを含めて差分を表示します。通常、シークレットは表示されませんが、このオプションを指定すると表示されます。

CI/CDパイプラインでの差分表示

ecspresso diff --unified > diff.txt

この例は、CI/CDパイプラインでの使用に適しています。差分をファイルに保存することで、後で確認できます。
