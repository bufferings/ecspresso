概要

ecspresso は、Go言語で実装された Amazon ECS (Elastic Container Service) 用のデプロイメントツールです。ソースコード分析に基づき、以下の主要コンポーネントで構成されています：

主な機能

  • 宣言的な設定ファイルを使用した ECS サービスのデプロイ、スケーリング、ロールバック
  • ローカル定義と実際にデプロイされたリソースの差分検出と比較
  • ECS コンテナ内での一時的なタスクの実行とコマンド実行
  • デプロイ前のリソース構成の詳細な検証
  • AWS CodeDeploy を通じたブルー/グリーンデプロイメント戦略のサポート

内部アーキテクチャ

ecspresso のコードベースは以下の主要コンポーネントで構成されています：

  • App構造体: ecspresso.go で定義され、すべての操作の中心的なオーケストレーターとして機能します。AWS SDKクライアントと設定への参照を含み、すべての主要な操作メソッドを実装しています。
  • CLIインターフェース: cli.go でコマンドライン引数を解析し、適切なハンドラーにディスパッチします。
  • 設定管理: config.go で ecspresso.ymlから読み込まれた設定値を保持します。YAML、JSON、Jsonnet形式をサポートしています。
  • デプロイ操作: deploy.go でECSサービスのデプロイロジックを実装しています。ECSローリングデプロイとCodeDeployの両方をサポートします。
  • 検証機能: verify.go でリソースが正しく構成されていることを確認します。
  • 差分比較: diff.go でローカル定義と実際のリソースの違いを表示します。
  • タスク管理: tasks.go および run.go でタスクの実行と管理を担当します。
  • プラグインシステム: plugin.go で機能拡張のための柔軟なプラグインシステムを提供します。

最新アップデート (v2.5.0)

ソースコード分析に基づく最新バージョンの主な機能と改善点：

  • deploy --wait-until オプションの追加による細かなサービスデプロイ制御
  • 外部プラグインシステムの追加によるカスタム拡張の容易化
  • CodeDeployの設定に deployment_config_name オプションの追加
  • Goの標準ロガー log/slog への移行による改善されたログ機能
  • ログと出力のJSON形式サポートの追加によるプログラム的処理の強化
  • コンテキスト化された状態による検証モジュールの改善
  • CodeDeployのAppSpecでのJSON形式サポートの追加
