インストール方法

ecspressoは複数の方法でインストールできます。以下の方法から選択してください：

Homebrew (macOS と Linux)

$ brew install kayac/tap/ecspresso

asdf (macOS と Linux)

$ asdf plugin add ecspresso
# または
$ asdf plugin add ecspresso https://github.com/kayac/asdf-ecspresso.git

$ asdf install ecspresso 2.5.0
$ asdf global ecspresso 2.5.0

aqua (macOS と Linux)

aquaはCLIバージョンマネージャーです。

$ aqua g -i kayac/ecspresso

バイナリパッケージ

Releasesページからダウンロードできます。ecspressoは単一のバイナリファイルとして提供され、追加の依存関係なしで実行できます。

Dockerイメージ

Docker環境がある場合は、Dockerイメージを使用することもできます：

$ docker pull ghcr.io/kayac/ecspresso:latest
$ docker run --rm -it -v $HOME/.aws:/root/.aws -v $(pwd):/work ghcr.io/kayac/ecspresso:latest [コマンド]

ソースからのビルド

ecspressoはGo言語で実装されており、ソースコードからビルドすることも可能です。ビルドプロセスはMakefileで定義されています：

$ git clone https://github.com/kayac/ecspresso.git
$ cd ecspresso
$ make build

内部的には、cmd/ecspresso/main.goがエントリーポイントとなり、ecspresso.CLI関数を呼び出してコマンドライン引数を処理します。ビルドには以下の主要な依存関係が使用されています：

  • AWS SDK for Go v2: AWS APIとの通信
  • github.com/alecthomas/kingpin/v2: コマンドライン引数の解析
  • github.com/kayac/go-config: 設定ファイルの読み込みと変数置換
  • github.com/google/go-jsonnet: Jsonnetファイルの処理
  • github.com/fatih/color: カラー出力のサポート

CI/CD環境でのインストール

CircleCI Orbs

https://circleci.com/orbs/registry/orb/fujiwara/ecspresso

version: 2.1
orbs:
  ecspresso: fujiwara/ecspresso@2.0.4
jobs:
  install:
    steps:
      - checkout
      - ecspresso/install:
          version: v2.5.0 # または latest
          # version-file: .ecspresso-version
          os: linux # または windows または darwin
          arch: amd64 # または arm64
      - run:
          command: |
            ecspresso version

Orb バージョンによって、version: latestでインストールされるecspressoのバージョンが異なります：

  • fujiwara/ecspresso@0.0.15 - 最新のリリースバージョン (v2以降)
  • fujiwara/ecspresso@1.0.0 - v1.xの最新バージョン
  • fujiwara/ecspresso@2.0.3 - v2.xの最新バージョン

注意: 新しいバージョンのecspressoがリリースされると予期しない動作を引き起こす可能性があるため、version: latestの使用は推奨されません。

GitHub Actions

Action kayac/ecspresso@v2 は Linux(x86_64) 用の ecspresso バイナリを /usr/local/bin にインストールします。このアクションはインストールのみを実行します。

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: kayac/ecspresso@v2
        with:
          version: v2.5.0 # または latest
          # version-file: .ecspresso-version
      - run: |
          ecspresso deploy --config ecspresso.yml

ecspressoの最新バージョンを使用するには、パラメータ "latest" を渡します。

      - uses: kayac/ecspresso@v2
        with:
          version: latest

Action バージョンによって、version: latestでインストールされるecspressoのバージョンが異なります：

  • kayac/ecspresso@v1 - v1.xの最新バージョン
  • kayac/ecspresso@v2 - v2.xの最新バージョン

注意: 新しいバージョンのecspressoがリリースされると予期しない動作を引き起こす可能性があるため、version: latestの使用は推奨されません。

必要条件

ecspressoを使用するには、以下の条件を満たす必要があります：

AWS認証情報

ecspressoはAWS SDKを使用してAWSリソースにアクセスするため、適切なAWS認証情報が必要です。以下のいずれかの方法で認証情報を設定できます：

  • 環境変数（AWS_ACCESS_KEY_ID、AWS_SECRET_ACCESS_KEY）
  • 共有認証情報ファイル（~/.aws/credentials）
  • IAMロール（EC2インスタンスやECSタスクで実行する場合）

必要なIAMアクセス権限

ecspressoを使用するには、以下のAWSサービスへのアクセス権限が必要です：

  • Amazon ECS（サービスとタスク定義の管理）
  • Amazon ECR（コンテナイメージの参照）
  • AWS IAM（ロールとポリシーの検証）
  • Amazon CloudWatch Logs（ログの設定）
  • AWS CodeDeploy（ブルー/グリーンデプロイメントを使用する場合）

最小限の権限ポリシーの例：

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ecs:DescribeServices",
        "ecs:DescribeTaskDefinition",
        "ecs:RegisterTaskDefinition",
        "ecs:UpdateService",
        "ecs:RunTask",
        "ecs:DescribeTasks",
        "ecs:ListTasks",
        "ecs:StopTask",
        "iam:PassRole"
      ],
      "Resource": "*"
    }
  ]
}

初期設定

ecspressoを使用するには、プロジェクトディレクトリに設定ファイルを作成する必要があります。

設定ファイルの作成

以下のコマンドを実行して、新しいecspresso設定ファイルを作成します：

$ mkdir -p my-ecs-app
$ cd my-ecs-app
$ ecspresso init --region ap-northeast-1 --cluster my-cluster --service my-service

このコマンドは、ecspresso.ymlという設定ファイルを作成します。既存のECSサービスがある場合は、そのサービスの設定が取得されます。新しいサービスを作成する場合は、基本的な設定が生成されます。

設定ファイルの例

基本的なecspresso.ymlファイルの例：

region: ap-northeast-1
cluster: my-cluster
service: my-service
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
timeout: 10m0s
plugins:
  - name: ssm
  - name: secretsmanager

この設定ファイルでは、以下の項目を指定しています：

  • region: AWSリージョン
  • cluster: ECSクラスター名
  • service: ECSサービス名
  • service_definition: サービス定義ファイルのパス
  • task_definition: タスク定義ファイルのパス
  • timeout: 操作のタイムアウト時間
  • plugins: 使用するプラグインのリスト

最初のステップ

ecspressoの基本的な使い方を学びましょう。

ヘルプの表示

利用可能なコマンドとオプションを確認するには、以下のコマンドを実行します：

$ ecspresso --help
$ ecspresso [コマンド] --help

サービス情報の取得

既存のECSサービスの情報を取得するには、以下のコマンドを実行します：

$ ecspresso status

設定の検証

設定ファイルとタスク定義が正しいかどうかを検証するには、以下のコマンドを実行します：

$ ecspresso verify

差分の確認

ローカルの設定とAWS上の設定の差分を確認するには、以下のコマンドを実行します：

$ ecspresso diff

サービスのデプロイ

ECSサービスをデプロイするには、以下のコマンドを実行します：

$ ecspresso deploy

これらの基本的なコマンドを使用して、ecspressoの機能を試してみてください。詳細なコマンドリファレンスについては、CLIコマンドリファレンスを参照してください。
