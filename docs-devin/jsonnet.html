Jsonnet概要

Jsonnetは、JSONのスーパーセットであるデータテンプレート言語です。変数、条件分岐、ループ、関数などのプログラミング機能を提供し、複雑な設定ファイルを簡潔に記述できます。

graph TD A[Jsonnet] --> B[変数] A --> C[条件分岐] A --> D[ループ] A --> E[関数] A --> F[インポート] A --> G[継承] B --> H[ローカル変数] B --> I[外部変数] C --> J[if-then-else] D --> K[配列内包表記] D --> L[オブジェクト内包表記] E --> M[ローカル関数] E --> N[標準ライブラリ関数] F --> O[ファイルインポート] F --> P[URLインポート] G --> Q[オブジェクト継承]

Jsonnetの主な特徴：

  • JSONの上位互換性
  • 変数と計算式
  • 条件分岐とループ
  • 関数定義と呼び出し
  • ファイルのインポート
  • オブジェクトの継承と合成
  • テンプレート化と再利用

ecspressoでのJsonnet統合

ecspressoは、設定ファイル、タスク定義ファイル、サービス定義ファイルにJsonnetを使用することをサポートしています。この機能はconfig.goファイルに実装されています。

graph TD A[ecspresso] --> B[設定ファイル] A --> C[タスク定義ファイル] A --> D[サービス定義ファイル] B --> E[ecspresso.jsonnet] C --> F[task-definition.jsonnet] D --> G[service-definition.jsonnet] E --> H[Jsonnet評価] F --> H G --> H H --> I[JSON変換] I --> J[設定読込]

ecspressoでJsonnetを使用するには、以下の方法があります：

  • ecspresso.jsonnetファイルを作成する
  • タスク定義ファイルとサービス定義ファイルを.jsonnet拡張子で作成する
  • --configオプションでJsonnetファイルを指定する
  • --task-definitionオプションでJsonnetファイルを指定する
  • --service-definitionオプションでJsonnetファイルを指定する

Jsonnetファイルは、ecspressoによって評価され、JSONに変換されてから読み込まれます。この処理は、config.goファイルのloadJSONNet関数で行われます：

func loadJSONNet(path string, extVars map[string]string) ([]byte, error) {
    vm := jsonnet.MakeVM()
    for k, v := range extVars {
        vm.ExtVar(k, v)
    }
    for name, fn := range jsonnetNativeFuncs {
        vm.NativeFunction(jsonnetNativeFunc(name, fn))
    }
    return vm.EvaluateFile(path)
}

Jsonnetの使用方法

ecspressoでJsonnetを使用するには、以下の手順に従います：

設定ファイルでJsonnetを使用する

  1. ecspresso.jsonnetファイルを作成する
  2. Jsonnet構文を使用して設定を記述する
  3. ecspressoコマンドを実行する

タスク定義ファイルでJsonnetを使用する

  1. task-definition.jsonnetファイルを作成する
  2. Jsonnet構文を使用してタスク定義を記述する
  3. ecspresso.ymlファイルでtask_definitionパラメータにtask-definition.jsonnetを指定する
  4. ecspressoコマンドを実行する

サービス定義ファイルでJsonnetを使用する

  1. service-definition.jsonnetファイルを作成する
  2. Jsonnet構文を使用してサービス定義を記述する
  3. ecspresso.ymlファイルでservice_definitionパラメータにservice-definition.jsonnetを指定する
  4. ecspressoコマンドを実行する

外部変数の使用

Jsonnetファイルでは、std.extVar関数を使用して外部変数を参照できます。外部変数は、以下の方法で指定できます：

  • --ext-strオプションで指定する
  • --ext-codeオプションで指定する
  • 環境変数で指定する（envsパラメータで指定した環境変数）
sequenceDiagram participant User as ユーザー participant CLI as ecspresso participant Jsonnet as Jsonnet VM participant Config as 設定読込 User->>CLI: ecspresso deploy --ext-str env=prod CLI->>Jsonnet: 外部変数設定（env=prod） CLI->>Jsonnet: ecspresso.jsonnet評価 Jsonnet-->>CLI: JSON CLI->>Config: JSON読込 Config-->>CLI: 設定情報 CLI->>User: デプロイ実行

Jsonnetネイティブ関数

ecspressoは、Jsonnetファイル内で使用できるネイティブ関数を提供しています。これらの関数は、config.goファイルのjsonnetNativeFuncsマップで定義されています。

graph TD A[Jsonnetネイティブ関数] --> B[プラグイン関数] A --> C[ユーティリティ関数] B --> D[tfstate] B --> E[cfn_output] B --> F[ssm] B --> G[secretsmanager_arn] C --> H[env] C --> I[must_env]

プラグイン関数

  • tfstate: Terraformの状態ファイルからリソース情報を取得します。
  • cfn_output: CloudFormationスタックの出力値を取得します。
  • ssm: SSM Parameter Storeからパラメータを取得します。
  • secretsmanager_arn: Secrets Managerのシークレット名からARNを取得します。

ユーティリティ関数

  • env: 環境変数の値を取得します。環境変数が存在しない場合は空文字列を返します。
  • must_env: 環境変数の値を取得します。環境変数が存在しない場合はエラーを返します。

これらの関数は、Jsonnetファイル内で以下のように使用できます：

local tfstate = std.native('tfstate');
local env = std.native('env');

{
  cluster: tfstate('aws_ecs_cluster.main.name'),
  service: 'my-service-' + env('ENV'),
  // ...
}

Jsonnet例

以下は、ecspressoでJsonnetを使用する例です：

設定ファイルの例

// ecspresso.jsonnet
local env = std.extVar('env');
local region = std.extVar('region');

{
  region: region,
  cluster: 'my-cluster',
  service: 'my-service-' + env,
  service_definition: 'service-definition-' + env + '.json',
  task_definition: 'task-definition-' + env + '.json',
  timeout: '10m',
  plugins: {
    tfstate: {
      path: 'terraform.tfstate'
    }
  },
  envs: [
    'APP_NAME',
    'APP_ENV'
  ],
  env_file: '.env.' + env
}

タスク定義ファイルの例

// task-definition.jsonnet
local env = std.extVar('env');
local tfstate = std.native('tfstate');
local env_func = std.native('env');

{
  family: 'my-app-' + env,
  executionRoleArn: tfstate('aws_iam_role.ecs_task_execution_role.arn'),
  taskRoleArn: tfstate('aws_iam_role.ecs_task_role.arn'),
  networkMode: 'awsvpc',
  containerDefinitions: [
    {
      name: 'web',
      image: env_func('ECR_REPOSITORY') + ':' + env_func('IMAGE_TAG'),
      essential: true,
      portMappings: [
        {
          containerPort: 80,
          hostPort: 80,
          protocol: 'tcp'
        }
      ],
      logConfiguration: {
        logDriver: 'awslogs',
        options: {
          'awslogs-group': '/ecs/my-app-' + env,
          'awslogs-region': env_func('AWS_REGION'),
          'awslogs-stream-prefix': 'web'
        }
      },
      environment: [
        {
          name: 'NODE_ENV',
          value: env
        }
      ],
      cpu: 256,
      memory: 512
    }
  ],
  requiresCompatibilities: [
    'FARGATE'
  ],
  cpu: '256',
  memory: '512'
}

サービス定義ファイルの例

// service-definition.jsonnet
local env = std.extVar('env');
local tfstate = std.native('tfstate');
local env_func = std.native('env');

{
  cluster: 'my-cluster',
  serviceName: 'my-service-' + env,
  taskDefinition: 'my-app-' + env,
  desiredCount: if env == 'prod' then 2 else 1,
  launchType: 'FARGATE',
  platformVersion: 'LATEST',
  deploymentConfiguration: {
    deploymentCircuitBreaker: {
      enable: true,
      rollback: true
    },
    maximumPercent: 200,
    minimumHealthyPercent: 100
  },
  networkConfiguration: {
    awsvpcConfiguration: {
      subnets: [
        tfstate('aws_subnet.private_a.id'),
        tfstate('aws_subnet.private_c.id')
      ],
      securityGroups: [
        tfstate('aws_security_group.app.id')
      ],
      assignPublicIp: 'DISABLED'
    }
  },
  loadBalancers: [
    {
      targetGroupArn: tfstate('aws_lb_target_group.app_' + env + '.arn'),
      containerName: 'web',
      containerPort: 80
    }
  ],
  healthCheckGracePeriodSeconds: 60,
  schedulingStrategy: 'REPLICA',
  deploymentController: {
    type: 'ECS'
  },
  enableECSManagedTags: true,
  propagateTags: 'SERVICE',
  enableExecuteCommand: true
}

複数環境の設定例

// ecspresso.jsonnet
local env = std.extVar('env');
local region = std.extVar('region');

local environments = {
  dev: {
    desiredCount: 1,
    cpu: '256',
    memory: '512',
    enableExecuteCommand: true
  },
  stg: {
    desiredCount: 1,
    cpu: '512',
    memory: '1024',
    enableExecuteCommand: true
  },
  prod: {
    desiredCount: 2,
    cpu: '1024',
    memory: '2048',
    enableExecuteCommand: false
  }
};

local envConfig = environments[env];

{
  region: region,
  cluster: 'my-cluster',
  service: 'my-service-' + env,
  service_definition: 'service-definition.jsonnet',
  task_definition: 'task-definition.jsonnet',
  timeout: '10m',
  plugins: {
    tfstate: {
      path: 'terraform.tfstate'
    }
  },
  envs: [
    'APP_NAME',
    'APP_ENV',
    'ECR_REPOSITORY',
    'IMAGE_TAG'
  ],
  env_file: '.env.' + env,
  vars: {
    env: env,
    desired_count: std.toString(envConfig.desiredCount),
    cpu: envConfig.cpu,
    memory: envConfig.memory,
    enable_execute_command: std.toString(envConfig.enableExecuteCommand)
  }
}

コマンド実行例

# 開発環境にデプロイ
ecspresso deploy --config ecspresso.jsonnet --ext-str env=dev --ext-str region=ap-northeast-1

# ステージング環境にデプロイ
ecspresso deploy --config ecspresso.jsonnet --ext-str env=stg --ext-str region=ap-northeast-1

# 本番環境にデプロイ
ecspresso deploy --config ecspresso.jsonnet --ext-str env=prod --ext-str region=ap-northeast-1
