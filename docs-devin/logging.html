ログ概要

ecspressoは、操作の進行状況、エラー、デバッグ情報を報告するためのログ機能を提供しています。このセクションでは、ecspressoのログとエラー処理について説明します。

graph TD A[ログレベル] --> B[INFO] A --> C[WARN] A --> D[ERROR] A --> E[DEBUG] B --> B1[標準情報] C --> C1[警告情報] D --> D1[エラー情報] E --> E1[デバッグ情報] F[ログ出力先] --> G[標準出力] F --> H[標準エラー出力] I[ログ形式] --> J[テキスト] I --> K[カラー]

ecspressoのログは、以下のレベルで出力されます：

  • INFO: 標準的な情報メッセージ
  • WARN: 警告メッセージ
  • ERROR: エラーメッセージ
  • DEBUG: デバッグ情報（デバッグモードが有効な場合のみ）

ログ実装

ecspressoのログ機能は、ecspresso.goファイルに実装されています。以下は、主要なログ関数の説明です：

// LogInfo は INFO レベルのログを出力します
func LogInfo(format string, v ...interface{}) {
    fmt.Fprintf(os.Stdout, format+"\n", v...)
}

// LogWarn は WARN レベルのログを出力します
func LogWarn(format string, v ...interface{}) {
    fmt.Fprintf(os.Stderr, "WARN: "+format+"\n", v...)
}

// LogError は ERROR レベルのログを出力します
func LogError(format string, v ...interface{}) {
    fmt.Fprintf(os.Stderr, "ERROR: "+format+"\n", v...)
}

// LogDebug は DEBUG レベルのログを出力します（デバッグモードが有効な場合のみ）
func LogDebug(format string, v ...interface{}) {
    if debug {
        fmt.Fprintf(os.Stderr, "DEBUG: "+format+"\n", v...)
    }
}

これらの関数は、fmt.Fprintfを使用して、標準出力または標準エラー出力にログメッセージを出力します。

カラーログ

ecspressoは、カラーログをサポートしています。カラーログは、--colorオプションで有効にできます：

// LogColorInfo はカラー INFO レベルのログを出力します
func LogColorInfo(format string, v ...interface{}) {
    fmt.Fprintf(os.Stdout, colorstring.Color("[green]"+format+"\n"), v...)
}

// LogColorWarn はカラー WARN レベルのログを出力します
func LogColorWarn(format string, v ...interface{}) {
    fmt.Fprintf(os.Stderr, colorstring.Color("[yellow]WARN: "+format+"\n"), v...)
}

// LogColorError はカラー ERROR レベルのログを出力します
func LogColorError(format string, v ...interface{}) {
    fmt.Fprintf(os.Stderr, colorstring.Color("[red]ERROR: "+format+"\n"), v...)
}

// LogColorDebug はカラー DEBUG レベルのログを出力します（デバッグモードが有効な場合のみ）
func LogColorDebug(format string, v ...interface{}) {
    if debug {
        fmt.Fprintf(os.Stderr, colorstring.Color("[blue]DEBUG: "+format+"\n"), v...)
    }
}

カラーログは、github.com/mitchellh/colorstringパッケージを使用して実装されています。

エラー処理

ecspressoは、Goの標準的なエラー処理パターンを使用しています。エラーは、関数の戻り値として返され、呼び出し元で処理されます。

エラー処理パターン

ecspressoの一般的なエラー処理パターンは以下の通りです：

func someFunction() error {
    result, err := someOperation()
    if err != nil {
        return fmt.Errorf("operation failed: %w", err)
    }
    // 処理を続行
    return nil
}

このパターンでは、エラーが発生した場合、fmt.Errorfを使用してエラーメッセージを追加し、%wフォーマット指定子を使用して元のエラーをラップします。これにより、エラーの連鎖が作成され、デバッグが容易になります。

エラーのラップ

Go 1.13以降、errors.Isとerrors.As関数を使用して、ラップされたエラーを検査できます：

if errors.Is(err, context.Canceled) {
    // コンテキストがキャンセルされた場合の処理
} else if errors.Is(err, os.ErrNotExist) {
    // ファイルが存在しない場合の処理
} else {
    // その他のエラーの処理
}

ecspressoは、この機能を使用して、特定のエラータイプに基づいて異なる処理を行います。

エラーの種類

ecspressoでは、以下のようなエラーが発生する可能性があります：

  • 設定エラー: 設定ファイルの読み込みや解析に関するエラー
  • AWS APIエラー: AWS APIの呼び出しに関するエラー
  • タイムアウトエラー: 操作がタイムアウトした場合のエラー
  • 検証エラー: リソースの検証に関するエラー
  • 入力エラー: ユーザー入力に関するエラー

デバッグモード

ecspressoは、デバッグモードをサポートしています。デバッグモードは、--debugオプションで有効にできます：

ecspresso deploy --debug

デバッグモードが有効な場合、以下の情報が出力されます：

  • 詳細なログメッセージ
  • AWS APIリクエストとレスポンス
  • 内部状態情報
  • タイミング情報

デバッグモードは、問題のトラブルシューティングに役立ちます。

ログのベストプラクティス

ecspressoを使用する際のログとエラー処理のベストプラクティスを紹介します：

適切なログレベルの使用

ログレベルを適切に使用して、重要な情報を見やすくします：

  • INFO: 通常の操作の進行状況
  • WARN: 潜在的な問題（ただし操作は続行可能）
  • ERROR: 操作が失敗した場合
  • DEBUG: トラブルシューティングに役立つ詳細情報

デバッグモードの活用

問題が発生した場合は、--debugオプションを使用して詳細な情報を取得します：

ecspresso deploy --debug > deploy.log 2>&1

このコマンドは、標準出力と標準エラー出力の両方をdeploy.logファイルにリダイレクトします。

エラーメッセージの解釈

エラーメッセージには、問題の原因と解決策のヒントが含まれています。以下は、一般的なエラーメッセージとその解釈です：

エラーメッセージ 解釈 解決策
ERROR: Failed to load config: open ecspresso.yml: no such file or directory 設定ファイルが見つかりません 正しいディレクトリにいることを確認するか、--configオプションで設定ファイルのパスを指定します
ERROR: Failed to describe service: ServiceNotFoundException: Service not found. 指定されたサービスが存在しません サービス名とクラスター名が正しいことを確認するか、--create-serviceオプションを使用して新しいサービスを作成します
ERROR: Failed to register task definition: AccessDeniedException: User is not authorized to perform ecs:RegisterTaskDefinition AWS認証情報に必要な権限がありません AWS認証情報に適切な権限があることを確認します
ERROR: Failed to update service: InvalidParameterException: The specified target group does not exist. 指定されたターゲットグループが存在しません サービス定義のロードバランサー設定を確認します
ERROR: Failed to wait for service stable: context deadline exceeded サービスが安定状態になるのを待っている間にタイムアウトしました --timeoutオプションでタイムアウト時間を増やすか、サービスの問題を調査します

CI/CDパイプラインでのログ

CI/CDパイプラインでecspressoを使用する場合は、以下のベストプラクティスに従います：

  • 常に--debugオプションを使用して、問題のトラブルシューティングを容易にします
  • ログをファイルに保存して、後で分析できるようにします
  • エラーが発生した場合は、パイプラインを失敗させて、問題を早期に検出します
  • デプロイの前にecspresso verifyとecspresso diffを実行して、潜在的な問題を検出します
# CI/CDパイプラインの例
set -e  # エラーが発生した場合にスクリプトを終了

# 設定を検証
ecspresso verify --debug || {
  echo "Verification failed"
  exit 1
}

# 差分を確認
ecspresso diff --debug || {
  echo "Diff check failed"
  exit 1
}

# デプロイを実行
ecspresso deploy --debug || {
  echo "Deployment failed"
  exit 1
}

echo "Deployment succeeded"
