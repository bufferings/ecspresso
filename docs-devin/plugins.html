プラグインシステム概要

ecspressoはテンプレート関数とJsonnetネイティブ関数を拡張するプラグインをサポートしています。プラグインシステムはplugin.goで実装されており、ConfigPluginインターフェースを通じて機能を拡張します。

内部的には、App.LoadPluginsメソッドが設定ファイルからプラグイン定義を読み込み、各プラグインを初期化します。プラグインはConfigPluginインターフェースを実装し、テンプレート関数を提供します。v2.5.0からは外部プラグインもサポートされ、findExternalPlugin関数によってecspresso-plugin-*という名前のバイナリが検索されます。

内部実装

プラグインシステムの中心はConfigPluginインターフェースで、以下のメソッドを定義しています：

type ConfigPlugin interface {
    FuncMap() template.FuncMap
    NativeFuncs() map[string]interface{}
}

各プラグインはこのインターフェースを実装し、テンプレート関数（FuncMap）とJsonnetネイティブ関数（NativeFuncs）を提供します。

sequenceDiagram participant App participant Config participant PluginLoader participant Plugin participant AWS App->>Config: LoadPlugins() Config->>PluginLoader: プラグイン設定を読み込み PluginLoader->>Plugin: プラグインを初期化 Plugin->>AWS: AWSクライアントを初期化 Plugin-->>PluginLoader: プラグインインスタンス PluginLoader-->>Config: プラグインリスト Config-->>App: プラグイン関数を登録

tfstateプラグイン

tfstateプラグインはtfstateとtfstatefテンプレート関数を導入します。このプラグインはplugin/tfstate/tfstate.goで実装されており、Terraformの状態ファイルからリソース情報を取得します。

# ecspresso.yml
region: ap-northeast-1
cluster: default
service: test
service_definition: ecs-service-def.json
task_definition: ecs-task-def.json
plugins:
  - name: tfstate
    config:
      url: s3://my-bucket/terraform.tfstate
      # または path: terraform.tfstate    # ローカルファイルへのパス
# ecs-service-def.json
{
  "networkConfiguration": {
    "awsvpcConfiguration": {
      "subnets": [
        "{{ tfstatef `aws_subnet.private['%s'].id` `az-a` }}"
      ],
      "securityGroups": [
        "{{ tfstate `data.aws_security_group.default.id` }}"
      ]
    }
  }
}

{{ tfstate "resource_type.resource_name.attr" }}はtfstate内のリソースの属性値に展開されます。

{{ tfstatef "resource_type.resource_name['%s'].attr" "index" }}は{{ tfstatef "resource_type.resource_name['index'].attr" }}と同様です。この関数は環境変数を使用してリソースアドレスを構築するのに役立ちます。

tfstate Jsonnet関数

local tfstate = std.native('tfstate');
{
  networkConfiguration: {
    awsvpcConfiguration: {
      subnets: [
        tfstate('aws_subnet.private["%s"].id' % 'az-z'),
        tfstate(std.format('aws_subnet.private["%s"].id', 'az-b')),
      ],
      securityGroups: [
        tfstate('data.aws_security_group.default.id'),
      ]
    }
  }
}

CloudFormationプラグイン

cloudformationプラグインはcfn_outputとcfn_exportテンプレート関数を導入します。このプラグインはplugin/cloudformation/cloudformation.goで実装されており、AWS CloudFormationスタックからリソース情報を取得します。

# ecspresso.yml
# ...
plugins:
  - name: cloudformation

cfn_output StackName OutputKeyはStackName内のOutputKeyのOutputValueを検索します。内部的にはDescribeStacks APIを使用して、スタックの出力値を取得します。

cfn_export ExportNameは名前でエクスポートされた値を検索します。内部的にはListExports APIを使用して、エクスポートされた値を取得します。

SSMプラグイン

ssmテンプレート関数はAWS Systems Manager（SSM）Parameter Storeからパラメータを読み取ります。このプラグインはplugin/ssm/ssm.goで実装されており、AWS SSM Parameter Storeからパラメータを取得します。

{
  "string": "{{ ssm `/path/to/string` }}",
  "stringlist": "{{ ssm `/path/to/stringlist` 1 }}",
  "securestring": "{{ ssm `/path/to/securestring` }}"
}

内部的にはGetParameter APIを使用して、パラメータを取得します。StringListタイプのパラメータの場合、オプションのインデックスを指定して特定の値を取得できます。SecureStringタイプのパラメータの場合、自動的に復号化されます。

Secrets Managerプラグイン

secretsmanager_arnテンプレート関数はシークレット名からSecrets Managerのシークレット ARNを解決します。このプラグインはplugin/secretsmanager/secretsmanager.goで実装されており、AWS Secrets Managerからシークレットを取得します。

  "secrets": [
    {
      "name": "FOO",
      "valueFrom": "{{ secretsmanager_arn `foo` }}"
    }
  ]

内部的にはListSecrets APIを使用して、シークレット名からARNを解決します。また、GetSecretValue APIを使用して、シークレットの値を取得します。

外部コマンドプラグイン

externalプラグインは任意の外部コマンドを実行する関数を導入します。このプラグインはplugin/external/external.goで実装されており、外部コマンドを実行して結果を取得します。

# ecspresso.yml
plugins:
  - name: external
    config:
      name: jq
      command: ["jq", "-n"]
      num_args: 1
      timeout: 5

configセクションは以下のパラメータを定義します：

  • name: テンプレート関数名
  • command: 外部コマンドと引数（配列）
    • コマンドは標準出力にJSON文字列または任意の文字列を返す必要があります。
  • num_args: 引数の数（オプション、デフォルト0）
  • parser: パーサータイプ "json"または "string"（オプション、デフォルト "json"）
  • timeout: コマンド実行タイムアウト秒（オプション、デフォルトはタイムアウトなし）

内部的には、external.New関数がプラグインを初期化し、external.Funcメソッドがテンプレート関数を提供します。コマンド実行はexec.Commandを使用して行われ、タイムアウトはcontext.WithTimeoutで制御されます。

外部プラグイン（v2.5.0以降）

v2.5.0からは、ecspresso-plugin-*という名前のバイナリを使用して外部プラグインを実装できるようになりました。この機能はplugin.goのfindExternalPlugin関数で実装されています。

sequenceDiagram participant App participant PluginLoader participant ExternalPlugin App->>PluginLoader: LoadPlugins() PluginLoader->>PluginLoader: findExternalPlugin() PluginLoader->>ExternalPlugin: プラグイン設定をJSON形式で送信 ExternalPlugin-->>PluginLoader: 関数マップをJSON形式で返す PluginLoader-->>App: プラグイン関数を登録

外部プラグインは以下のインターフェースを実装する必要があります：

type ConfigPlugin interface {
    FuncMap() template.FuncMap
    NativeFuncs() map[string]interface{}
}

外部プラグインは標準入力からJSON形式の設定を受け取り、標準出力にJSON形式の関数マップを返します。これにより、ecspressoの機能を拡張するカスタムプラグインを開発できます。

外部プラグインの検索はPATH環境変数に基づいて行われます。プラグイン名はecspresso-plugin-{name}の形式である必要があります。例えば、ssmプラグインの場合はecspresso-plugin-ssmという名前のバイナリが検索されます。
