リリース概要

ecspressoのリリースプロセスは、GitHubリリースとGoReleaserを使用して自動化されています。このセクションでは、ecspressoのリリースプロセスについて説明します。

graph TD A[リリースプロセス] --> B[バージョン決定] A --> C[変更履歴更新] A --> D[タグ作成] A --> E[リリースビルド] A --> F[リリース公開] B --> B1[セマンティックバージョニング] C --> C1[CHANGELOG.md更新] D --> D1[Gitタグ作成] D --> D2[タグプッシュ] E --> E1[GoReleaser実行] E --> E2[クロスプラットフォームビルド] F --> F1[GitHubリリース] F --> F2[Homebrewタップ更新]

ecspressoのリリースには、以下のツールが使用されます：

  • Git
  • GitHub
  • GoReleaser
  • GitHub Actions

リリースプロセス

ecspressoのリリースプロセスは、以下のステップで構成されています：

バージョン決定

リリースの最初のステップは、新しいバージョン番号の決定です。ecspressoは、セマンティックバージョニング（SemVer）を使用しています：

  • メジャーバージョン: 後方互換性のない変更
  • マイナーバージョン: 後方互換性のある機能追加
  • パッチバージョン: 後方互換性のあるバグ修正

変更履歴更新

次に、CHANGELOG.mdファイルを更新して、新しいバージョンの変更内容を記録します：

# 変更履歴の例
## v1.2.0 (2023-01-15)

### 機能追加
- 新しいコマンド `ecspresso exec` を追加
- プラグインシステムに外部プラグインのサポートを追加

### バグ修正
- デプロイ中のタイムアウト処理を修正
- タスク定義のレンダリングエラーを修正

### その他
- 依存関係を更新
- ドキュメントを改善

タグ作成

変更履歴を更新した後、新しいバージョンのGitタグを作成します：

git tag v1.2.0
git push origin v1.2.0

リリースビルド

タグがプッシュされると、GitHub Actionsのリリースワークフローが自動的に実行されます。このワークフローは、GoReleaserを使用して以下の処理を行います：

  • 複数のプラットフォーム向けにバイナリをビルド
  • リリースアーカイブを作成
  • チェックサムを生成
  • GitHubリリースを作成
  • Homebrewタップを更新

リリース公開

リリースビルドが完了すると、GitHubリリースページに新しいリリースが公開されます。ユーザーは、このページからバイナリをダウンロードできます。

バージョニング

ecspressoは、セマンティックバージョニング（SemVer）を使用しています。バージョン番号は、MAJOR.MINOR.PATCHの形式で表されます：

  • MAJOR: 後方互換性のない変更を行った場合にインクリメント
  • MINOR: 後方互換性のある機能追加を行った場合にインクリメント
  • PATCH: 後方互換性のあるバグ修正を行った場合にインクリメント

プレリリースバージョンとビルドメタデータは、MAJOR.MINOR.PATCH-pre.release+build.metadataの形式で表されます。

バージョン番号の例：

  • v1.0.0: 最初の安定リリース
  • v1.1.0: 新機能を追加したリリース
  • v1.1.1: バグ修正を行ったリリース
  • v2.0.0: 後方互換性のない変更を行ったリリース
  • v1.2.0-beta.1: ベータリリース

ecspressoのバージョン情報は、cmd/ecspresso/main.goファイルで定義されています：

var Version string

func main() {
    ecspresso.Version = Version
    // ...
}

このバージョン情報は、ビルド時に-ldflagsオプションで設定されます：

go build -ldflags "-s -w -X main.Version=$(VERSION)" -o bin/ecspresso cmd/ecspresso/main.go

リリース自動化

ecspressoのリリースプロセスは、GitHub ActionsとGoReleaserを使用して自動化されています。

GitHub Actions

GitHub Actionsは、タグがプッシュされたときに自動的にリリースワークフローを実行します。このワークフローは、.github/workflows/release.ymlファイルで定義されています：

name: Release
on:
  push:
    tags:
      - 'v*'
jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-go@v2
        with:
          go-version: 1.16
      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v2
        with:
          version: latest
          args: release --rm-dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

GoReleaser

GoReleaserは、Goプロジェクトのリリースを自動化するツールです。ecspressoでは、.goreleaser.ymlファイルでGoReleaserの設定を行っています：

builds:
  - env:
      - CGO_ENABLED=0
    goos:
      - linux
      - darwin
      - windows
    goarch:
      - amd64
      - arm64
    ldflags:
      - -s -w -X main.Version={{.Version}}
archives:
  - replacements:
      darwin: Darwin
      linux: Linux
      windows: Windows
      amd64: x86_64
checksum:
  name_template: 'checksums.txt'
snapshot:
  name_template: "{{ .Tag }}-next"
changelog:
  sort: asc
  filters:
    exclude:
      - '^docs:'
      - '^test:'
brews:
  - tap:
      owner: kayac
      name: homebrew-tap
    folder: Formula
    homepage: https://github.com/kayac/ecspresso
    description: ecspresso is a deployment tool for Amazon ECS
    test: |
      system "#{bin}/ecspresso --version"

この設定により、GoReleaserは以下の処理を行います：

  • Linux、macOS、Windows向けにバイナリをビルド
  • amd64とarm64アーキテクチャをサポート
  • バージョン情報をバイナリに埋め込む
  • リリースアーカイブを作成
  • チェックサムを生成
  • 変更履歴を生成
  • Homebrewタップを更新
