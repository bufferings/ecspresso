runコマンド

runコマンドは、ECSタスクを実行するために使用します。このコマンドはrun.goファイルに実装されています。

基本的な使い方

ecspresso run [オプション]

主なオプション

オプション 説明 デフォルト値
--dry-run 実際にタスクを実行せずに、実行されるアクションを表示する false
--count 実行するタスクの数 1
--revision 使用するタスク定義のリビジョン -
--latest-task-definition 最新のタスク定義を使用する false
--launch-type 起動タイプ（EC2 または FARGATE） -
--network-configuration ネットワーク設定のJSONファイル -
--overrides タスク定義オーバーライドのJSONファイル -
--skip-task-definition タスク定義の登録をスキップする false
--wait タスクが完了するまで待機する false
--watch-container ログを監視するコンテナ名 -

タスク実行プロセス

runコマンドの内部処理フロー：

sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWS ECS User->>App: ecspresso run App->>App: 設定ファイル読込 App->>App: タスク定義準備 alt タスク定義の登録をスキップしない場合 App->>AWS: RegisterTaskDefinition AWS-->>App: タスク定義ARN end App->>AWS: RunTask AWS-->>App: タスク情報 alt タスクの完了を待機する場合 App->>AWS: DescribeTasks（繰り返し） AWS-->>App: タスク状態 opt コンテナログを監視する場合 App->>AWS: GetLogEvents AWS-->>App: ログイベント end end App-->>User: 完了メッセージ

使用例

# 基本的なタスク実行
ecspresso run

# ドライランモード（実際にタスクを実行しない）
ecspresso run --dry-run

# 複数のタスクを実行
ecspresso run --count 3

# 特定のリビジョンのタスク定義を使用
ecspresso run --revision 10

# 最新のタスク定義を使用
ecspresso run --latest-task-definition

# Fargateで実行
ecspresso run --launch-type FARGATE --network-configuration network.json

# タスク定義をオーバーライド
ecspresso run --overrides overrides.json

# タスクが完了するまで待機
ecspresso run --wait

# コンテナログを監視
ecspresso run --wait --watch-container web

execコマンド

execコマンドは、実行中のECSタスクでコマンドを実行するために使用します。このコマンドはexec.goファイルに実装されています。

基本的な使い方

ecspresso exec [オプション] -- [コマンド]

主なオプション

オプション 説明 デフォルト値
--container コマンドを実行するコンテナ名 -
--interactive 対話モードで実行する false
--port-forward ポートフォワーディングを有効にする false
--task-id コマンドを実行するタスクID -
--timeout タイムアウト時間（秒） 600

コマンド実行プロセス

execコマンドの内部処理フロー：

flowchart TD A[exec開始] --> B{タスクID指定?} B -->|Yes| C[指定タスクを使用] B -->|No| D[実行中のタスクを検索] D --> E{タスクが見つかった?} E -->|Yes| F[タスクを選択] E -->|No| G[エラー終了] C --> H[SSMセッション開始] F --> H H --> I[コマンド実行] I --> J[結果表示] J --> K[終了]

使用例

# 実行中のタスクでコマンドを実行
ecspresso exec -- ls -la

# 特定のコンテナでコマンドを実行
ecspresso exec --container web -- ps aux

# 対話モードで実行
ecspresso exec --interactive -- /bin/bash

# 特定のタスクでコマンドを実行
ecspresso exec --task-id 12345678-1234-1234-1234-123456789012 -- env

# ポートフォワーディングを有効にして実行
ecspresso exec --port-forward -- curl localhost:8080

tasksコマンド

tasksコマンドは、ECSタスクを一覧表示または停止するために使用します。このコマンドはtasks.goファイルに実装されています。

基本的な使い方

ecspresso tasks [オプション]

主なオプション

オプション 説明 デフォルト値
--delete タスクを停止する false
--id 特定のタスクIDを指定する -
--output 出力形式（table または json） table
--status フィルタリングするタスクのステータス RUNNING

タスク管理プロセス

tasksコマンドの内部処理フロー：

sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWS ECS User->>App: ecspresso tasks App->>App: 設定ファイル読込 App->>AWS: ListTasks AWS-->>App: タスク一覧 App->>AWS: DescribeTasks AWS-->>App: タスク詳細 alt タスクを停止する場合 App->>AWS: StopTask AWS-->>App: 停止結果 end App-->>User: タスク情報表示

使用例

# 実行中のタスクを一覧表示
ecspresso tasks

# すべてのステータスのタスクを一覧表示
ecspresso tasks --status ALL

# JSON形式で出力
ecspresso tasks --output json

# 特定のタスクの詳細を表示
ecspresso tasks --id 12345678-1234-1234-1234-123456789012

# タスクを停止
ecspresso tasks --delete --id 12345678-1234-1234-1234-123456789012

registerコマンド

registerコマンドは、タスク定義を登録するために使用します。このコマンドはecspresso.goファイルに実装されています。

基本的な使い方

ecspresso register [オプション]

主なオプション

オプション 説明 デフォルト値
--dry-run 実際に登録せずに、実行されるアクションを表示する false
--output 出力形式（table または json） table

タスク定義登録プロセス

registerコマンドの内部処理フロー：

flowchart TD A[register開始] --> B[設定ファイル読込] B --> C[タスク定義ファイル読込] C --> D[タスク定義レンダリング] D --> E{ドライラン?} E -->|Yes| F[タスク定義表示] E -->|No| G[RegisterTaskDefinition API呼び出し] G --> H[登録結果表示] F --> I[終了] H --> I

使用例

# タスク定義を登録
ecspresso register

# ドライランモード（実際に登録しない）
ecspresso register --dry-run

# JSON形式で出力
ecspresso register --output json

タスクオーバーライド

runコマンドでは、--overridesオプションを使用してタスク定義をオーバーライドできます。オーバーライドは以下のような形式のJSONファイルで指定します：

{
  "containerOverrides": [
    {
      "name": "web",
      "command": ["npm", "run", "migrate"],
      "environment": [
        {
          "name": "NODE_ENV",
          "value": "production"
        }
      ],
      "cpu": 512,
      "memory": 1024
    }
  ],
  "executionRoleArn": "arn:aws:iam::123456789012:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::123456789012:role/ecsTaskRole"
}

主なオーバーライド項目：

  • containerOverrides: コンテナごとのオーバーライド設定
  • command: コンテナで実行するコマンド
  • environment: 環境変数
  • cpu: CPUユニット
  • memory: メモリ（MB）
  • executionRoleArn: タスク実行ロールのARN
  • taskRoleArn: タスクロールのARN

ベストプラクティス

タスク管理コマンドを使用する際のベストプラクティス：

  • 一時的なタスクの活用: データベースマイグレーションやバッチ処理などの一時的な作業にはecspresso runを使用します。
  • デバッグの効率化: 問題が発生した場合はecspresso execを使用して、実行中のコンテナに接続してデバッグします。
  • タスクのクリーンアップ: 不要なタスクはecspresso tasks --deleteを使用して停止し、リソースを解放します。
  • オーバーライドの活用: 同じタスク定義を使用して異なる設定でタスクを実行する場合は、オーバーライドを活用します。
  • ログの監視: --waitと--watch-containerオプションを組み合わせて、タスクの実行状況をリアルタイムで監視します。
  • CI/CDパイプラインの統合: データベースマイグレーションなどの作業をCI/CDパイプラインに統合して自動化します。
  • タスク定義の管理: ecspresso registerを使用して、タスク定義を管理します。
  • ポートフォワーディングの活用: ecspresso exec --port-forwardを使用して、ローカルマシンからコンテナ内のサービスにアクセスします。
