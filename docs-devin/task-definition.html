タスク定義概要

タスク定義は、ECSタスクの実行方法を定義するJSONまたはYAMLファイルです。ecspressoでは、タスク定義ファイルをecspresso.ymlのtask_definitionパラメータで指定します。

graph TD A[タスク定義] --> B[コンテナ定義] A --> C[タスクロール] A --> D[実行ロール] A --> E[ネットワークモード] A --> F[ボリューム] A --> G[プレースメント制約] B --> H[イメージ] B --> I[ポートマッピング] B --> J[環境変数] B --> K[ログ設定] B --> L[ヘルスチェック] B --> M[リソース制限]

ecspressoは、run.goファイルとecspresso.goファイルでタスク定義の管理機能を提供しています。タスク定義は、TaskDefinition構造体で表現され、以下の操作が可能です：

  • タスク定義の登録
  • タスク定義の更新
  • タスク定義の登録解除
  • タスク定義の実行
  • タスク定義のレンダリング

タスク定義構造

ecspressoでは、タスク定義はTaskDefinition構造体で表現されます。この構造体は、AWS ECSのタスク定義と互換性があり、以下のような主要なフィールドを持っています：

type TaskDefinition struct {
    ContainerDefinitions    []ContainerDefinition
    Family                  string
    TaskRoleArn            string
    ExecutionRoleArn       string
    NetworkMode            string
    Volumes                []Volume
    PlacementConstraints   []PlacementConstraint
    RequiresCompatibilities []string
    Cpu                    string
    Memory                 string
    Tags                   []Tag
}

主要なフィールドの説明：

  • ContainerDefinitions: コンテナの定義のリスト
  • Family: タスク定義のファミリー名
  • TaskRoleArn: タスクに割り当てるIAMロールのARN
  • ExecutionRoleArn: タスク実行に使用するIAMロールのARN
  • NetworkMode: ネットワークモード（awsvpc, bridge, host, none）
  • Volumes: ボリュームの定義のリスト
  • PlacementConstraints: タスクのプレースメント制約のリスト
  • RequiresCompatibilities: タスクの互換性要件（EC2, FARGATE）
  • Cpu: タスクに割り当てるCPUユニット
  • Memory: タスクに割り当てるメモリ（MB）
  • Tags: タスク定義に付けるタグのリスト

コンテナ定義は、以下のような主要なフィールドを持っています：

type ContainerDefinition struct {
    Name                  string
    Image                 string
    Essential             bool
    Command               []string
    EntryPoint            []string
    Environment           []KeyValuePair
    EnvironmentFiles      []EnvironmentFile
    Secrets               []Secret
    LogConfiguration      *LogConfiguration
    PortMappings          []PortMapping
    HealthCheck           *HealthCheck
    Cpu                   int
    Memory                int
    MemoryReservation     int
    MountPoints           []MountPoint
    VolumesFrom           []VolumeFrom
    DependsOn             []ContainerDependency
    LinuxParameters       *LinuxParameters
}

タスク定義管理

ecspressoでは、以下のコマンドを使用してタスク定義を管理できます：

flowchart TD A[タスク定義管理] --> B[register] A --> C[deploy] A --> D[run] A --> E[render] B --> F[タスク定義登録] C --> G[タスク定義登録とサービス更新] D --> H[タスク定義登録とタスク実行] E --> I[タスク定義レンダリング]

タスク定義の登録

registerコマンドは、タスク定義を登録するために使用します：

ecspresso register [オプション]

内部的には、App構造体のRegisterメソッドが呼び出され、以下の処理が行われます：

  1. タスク定義ファイルを読み込む
  2. テンプレート関数を処理する
  3. AWS ECSのRegisterTaskDefinition APIを呼び出す
  4. 登録されたタスク定義のARNを返す

タスク定義の実行

runコマンドは、タスク定義を実行するために使用します：

ecspresso run [オプション]

内部的には、App構造体のRunメソッドが呼び出され、以下の処理が行われます：

  1. タスク定義ファイルを読み込む
  2. 必要に応じてタスク定義を登録する
  3. AWS ECSのRunTask APIを呼び出す
  4. 必要に応じてタスクの完了を待機する

タスク定義のレンダリング

renderコマンドは、タスク定義をレンダリングするために使用します：

ecspresso render --task-definition [オプション]

内部的には、App構造体のRenderメソッドが呼び出され、以下の処理が行われます：

  1. タスク定義ファイルを読み込む
  2. テンプレート関数を処理する
  3. レンダリングされたタスク定義を表示または保存する

タスク定義オーバーライド

runコマンドでは、--overridesオプションを使用してタスク定義をオーバーライドできます。オーバーライドは以下のような形式のJSONファイルで指定します：

{
  "containerOverrides": [
    {
      "name": "web",
      "command": ["npm", "run", "migrate"],
      "environment": [
        {
          "name": "NODE_ENV",
          "value": "production"
        }
      ],
      "cpu": 512,
      "memory": 1024
    }
  ],
  "executionRoleArn": "arn:aws:iam::123456789012:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::123456789012:role/ecsTaskRole"
}

オーバーライドは、run.goファイルのRunメソッドで処理されます。オーバーライドファイルが指定されている場合、以下の処理が行われます：

  1. オーバーライドファイルを読み込む
  2. テンプレート関数を処理する
  3. タスク定義にオーバーライドを適用する
  4. オーバーライドされたタスク定義でタスクを実行する
sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWS ECS User->>App: ecspresso run --overrides overrides.json App->>App: タスク定義読込 App->>App: オーバーライド読込 App->>App: オーバーライド適用 App->>AWS: RunTask AWS-->>App: タスク情報 App-->>User: 完了メッセージ

タスク定義例

以下は、ecspressoで使用できるタスク定義の例です：

基本的なタスク定義

{
  "family": "my-app",
  "executionRoleArn": "arn:aws:iam::123456789012:role/ecsTaskExecutionRole",
  "taskRoleArn": "arn:aws:iam::123456789012:role/ecsTaskRole",
  "networkMode": "awsvpc",
  "containerDefinitions": [
    {
      "name": "web",
      "image": "123456789012.dkr.ecr.ap-northeast-1.amazonaws.com/my-app:latest",
      "essential": true,
      "portMappings": [
        {
          "containerPort": 80,
          "hostPort": 80,
          "protocol": "tcp"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/my-app",
          "awslogs-region": "ap-northeast-1",
          "awslogs-stream-prefix": "web"
        }
      },
      "environment": [
        {
          "name": "NODE_ENV",
          "value": "production"
        }
      ],
      "cpu": 256,
      "memory": 512
    }
  ],
  "requiresCompatibilities": [
    "FARGATE"
  ],
  "cpu": "256",
  "memory": "512"
}

          "awslogs-group": "/ecs/{{ env `APP_NAME` }}",

      },

        {
          "name": "NODE_ENV",
