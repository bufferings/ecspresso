よくある問題と解決策

ecspressoを使用する際によく発生する問題とその解決策を紹介します。

認証エラー

問題

Error: NoCredentialProviders: no valid providers in chain

AWS認証情報が設定されていないか、無効になっています。

解決策

  1. AWS認証情報が正しく設定されているか確認します。
  2. 環境変数（AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY）が設定されているか確認します。
  3. ~/.aws/credentialsファイルが正しく設定されているか確認します。
  4. IAMロールが正しく設定されているか確認します。

リージョンエラー

問題

Error: MissingRegion: could not find region configuration

AWSリージョンが設定されていません。

解決策

  1. 環境変数AWS_REGIONを設定します。
  2. ~/.aws/configファイルでリージョンを設定します。
  3. ecspresso.ymlファイルでリージョンを指定します。
  4. コマンドラインで--regionオプションを使用します。

設定ファイルエラー

問題

Error: failed to load config: open ecspresso.yml: no such file or directory

設定ファイルが見つかりません。

解決策

  1. カレントディレクトリに設定ファイルが存在するか確認します。
  2. --configオプションで正しいパスを指定します。
  3. ecspresso initコマンドで設定ファイルを生成します。

デプロイ関連の問題

デプロイ時によく発生する問題とその解決策を紹介します。

タスク定義の登録エラー

問題

Error: failed to register task definition: InvalidParameterException: The container definition's memory reservation is invalid

タスク定義のメモリ設定が無効です。

解決策

  1. タスク定義のメモリ設定を確認します。
  2. Fargateの場合、サポートされているCPUとメモリの組み合わせを使用します。
  3. コンテナのメモリ予約がタスクのメモリ制限を超えていないか確認します。

サービス更新エラー

問題

Error: failed to update service: InvalidParameterException: The specified target group does not exist

指定されたターゲットグループが存在しません。

解決策

  1. サービス定義のロードバランサー設定を確認します。
  2. ターゲットグループが存在するか確認します。
  3. ターゲットグループのARNが正しいか確認します。
  4. verifyコマンドを使用して、リソースを事前に検証します。

デプロイタイムアウト

問題

Error: context deadline exceeded

デプロイがタイムアウトしました。

解決策

  1. タイムアウト設定を長くします（ecspresso.ymlのtimeoutパラメータ）。
  2. --no-waitオプションを使用して、サービスの安定を待たずにデプロイを完了します。
  3. ヘルスチェックの設定を確認し、アプリケーションが正常に起動しているか確認します。
  4. CloudWatchログを確認して、アプリケーションのエラーを特定します。

タスク関連の問題

タスク実行時によく発生する問題とその解決策を紹介します。

タスク実行エラー

問題

Error: failed to run task: ResourceNotFoundException: The specified task definition does not exist

指定されたタスク定義が存在しません。

解決策

  1. タスク定義が正しく登録されているか確認します。
  2. タスク定義のARNまたはファミリー名が正しいか確認します。
  3. --task-defオプションで正しいタスク定義ファイルを指定します。

execコマンドエラー

問題

Error: failed to execute command: InvalidParameterException: The execute command agent is not running

ECS Execエージェントが実行されていません。

解決策

  1. タスク定義でECS Execが有効になっているか確認します。
  2. タスク実行ロールにssmmessages:CreateControlChannelなどの必要な権限があるか確認します。
  3. コンテナがSSMエージェントをサポートしているか確認します。

タスク停止エラー

問題

Error: failed to stop task: TaskNotFound: The specified task was not found

指定されたタスクが見つかりません。

解決策

  1. タスクIDが正しいか確認します。
  2. tasksコマンドを使用して、実行中のタスクを確認します。
  3. タスクが既に停止している可能性があります。

設定関連の問題

設定ファイルに関連する問題とその解決策を紹介します。

YAML/JSON構文エラー

問題

Error: failed to load config: yaml: line 10: did not find expected key

YAML構文エラーが発生しています。

解決策

  1. YAMLまたはJSONの構文を確認します。
  2. インデントが正しいか確認します。
  3. 引用符が正しく閉じられているか確認します。
  4. オンラインのYAML/JSON検証ツールを使用して構文を検証します。

テンプレート関数エラー

問題

Error: failed to render template: template: :1:3: executing "" at <env "AWS_REGION">: map has no entry for key "AWS_REGION"

テンプレート関数で参照している環境変数が設定されていません。

解決策

  1. 必要な環境変数が設定されているか確認します。
  2. envではなくmust_env関数を使用している場合、環境変数は必須です。
  3. --env-fileオプションで環境変数ファイルを指定します。
  4. デフォルト値を設定します（例：{{ env "AWS_REGION" "ap-northeast-1" }}）。

Jsonnetエラー

問題

Error: failed to load config: jsonnet: RUNTIME ERROR: Undefined external variable: ENV

Jsonnetで参照している外部変数が設定されていません。

解決策

  1. --envオプションで外部変数を設定します（例：--env ENV=prod）。
  2. Jsonnetファイルで外部変数のデフォルト値を設定します。
  3. 外部変数の名前が正しいか確認します。

AWS関連の問題

AWS APIとの通信に関連する問題とその解決策を紹介します。

APIレート制限

問題

Error: failed to describe service: RequestLimitExceeded: Request limit exceeded

AWS APIのレート制限に達しました。

解決策

  1. リクエストの頻度を下げます。
  2. バックオフ戦略を実装します（一定時間待ってから再試行）。
  3. AWS Support Centerでレート制限の引き上げをリクエストします。

権限エラー

問題

Error: failed to describe service: AccessDeniedException: User is not authorized to perform ecs:DescribeServices

必要な権限がありません。

解決策

  1. IAMユーザーまたはロールに必要な権限を追加します。
  2. ecspressoの操作に必要な最小限の権限を確認します。
  3. AWS CLIの認証情報が正しく設定されているか確認します。

リソース制限エラー

問題

Error: failed to run task: ResourceLimitExceeded: The account has reached its limit of tasks

AWSアカウントのリソース制限に達しました。

解決策

  1. 不要なリソース（タスク、サービス）を削除します。
  2. AWS Service Quotasでリソース制限の引き上げをリクエストします。
  3. 別のリージョンでリソースを作成することを検討します。

デバッグ方法

ecspressoの問題をデバッグするための方法を紹介します。

デバッグログの有効化

詳細なログを表示するには、--debugオプションを使用します：

$ ecspresso deploy --debug

これにより、AWS APIリクエストやレスポンスなどの詳細な情報が表示されます。

ドライランの実行

実際の変更を行わずに実行内容を確認するには、--dry-runオプションを使用します：

$ ecspresso deploy --dry-run

これにより、実行される操作の詳細が表示されますが、実際の変更は行われません。

差分の確認

ローカルの設定とAWS上の設定の差分を確認するには、diffコマンドを使用します：

$ ecspresso diff

これにより、タスク定義やサービス定義の変更点が表示されます。

CloudWatchログの確認

タスクのログを確認するには、AWS Management ConsoleのCloudWatch Logsを使用します。ログは通常、以下のパスに保存されています：

/ecs/<タスク定義名>

または、AWS CLIを使用してログを取得することもできます：

$ aws logs get-log-events --log-group-name /ecs/my-service --log-stream-name ecs/app/<タスクID>

サービスイベントの確認

サービスのイベントを確認するには、statusコマンドを使用します：

$ ecspresso status --events 20

これにより、直近20件のサービスイベントが表示されます。デプロイの失敗理由などの情報が含まれています。

一般的な解決策

設定の検証

デプロイ前に設定を検証することで、多くの問題を事前に防ぐことができます：

$ ecspresso verify

これにより、タスク定義やサービス定義で参照されているリソースが実際に存在するかが確認されます。

最新バージョンの使用

ecspressoの最新バージョンを使用することで、既知のバグが修正されている可能性があります：

$ go install github.com/kayac/ecspresso/v2@latest

または、リリースページから最新のバイナリをダウンロードします：

https://github.com/kayac/ecspresso/releases

AWS CLIでの確認

AWS CLIを使用して、AWSリソースの状態を直接確認することができます：

$ aws ecs describe-services --cluster my-cluster --services my-service
$ aws ecs describe-task-definition --task-definition my-task-definition
$ aws ecs list-tasks --cluster my-cluster --service-name my-service

これにより、ecspressoの問題がAWS APIの問題なのか、設定の問題なのかを切り分けることができます。
