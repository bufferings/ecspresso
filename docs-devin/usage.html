基本的な使い方

ecspressoのCLIインターフェースはcli.goで実装されており、kingpinライブラリを使用してコマンドライン引数を解析しています。内部的にはCLI関数がエントリーポイントとなり、コマンドとフラグを処理します。

flowchart TD A[ユーザー] -->|コマンド実行| B[CLI関数] B -->|引数解析| C[kingpinライブラリ] C -->|コマンド特定| D[Appメソッド呼び出し] D -->|実行| E[AWS API操作] 
Usage: ecspresso <command>

Flags:
  -h, --help                      ヘルプを表示します
      --envfile=ENVFILE,...       環境変数ファイル ($ECSPRESSO_ENVFILE)
      --debug                     デバッグログを有効にします ($ECSPRESSO_DEBUG)
      --ext-str=KEY=VALUE;...     Jsonnet 用の外部文字列値 ($ECSPRESSO_EXT_STR)
      --ext-code=KEY=VALUE;...    Jsonnet 用の外部コード値 ($ECSPRESSO_EXT_CODE)
      --config="ecspresso.yml"    設定ファイル ($ECSPRESSO_CONFIG)
      --assume-role-arn=""        引き受ける IAM ロールの ARN ($ECSPRESSO_ASSUME_ROLE_ARN)
      --timeout=TIMEOUT           タイムアウト。設定ファイルで上書き可能 ($ECSPRESSO_TIMEOUT)
      --filter-command=STRING     フィルターコマンド ($ECSPRESSO_FILTER_COMMAND)
      --[no-]color                カラー出力を有効/無効にします ($ECSPRESSO_COLOR)

Commands:
  appspec
    CodeDeploy 用の AppSpec YAML を標準出力に出力します

  delete
    サービスを削除します

  deploy
    サービスをデプロイします

  deregister
    タスク定義を登録解除します

  diff
    タスク定義、サービス定義と現在実行中のサービスおよびタスク定義との差分を表示します

  exec
    タスク上でコマンドを実行します

  init --service=SERVICE
    既存の ECS サービスから設定ファイルを作成します

  refresh
    サービスを更新します。deploy --skip-task-definition --force-new-deployment --no-update-service と同等です

  register
    タスク定義を登録します

  render <targets>
    設定、サービス定義またはタスク定義ファイルを標準出力にレンダリングします

  revisions
    タスク定義のリビジョンを表示します

  rollback
    サービスをロールバックします

  run
    タスクを実行します

  scale
    サービスをスケールします。deploy --skip-task-definition --no-update-service と同等です

  status
    サービスのステータスを表示します

  tasks
    サービス内またはタスク定義と同じファミリーを持つタスクを一覧表示します

  verify
    設定内のリソースを検証します

  wait
    サービスが安定するまで待機します

  version
    バージョンを表示します

サブコマンドのより詳細なオプションについては、ecspresso sub-command --helpを参照してください。

内部コマンド構造

ecspressoのコマンド構造はcli.goで定義されており、各コマンドは対応するGoファイルで実装されています：

classDiagram class App { +Deploy(ctx, opt) error +Diff(ctx, opt) error +Verify(ctx, opt) error +Run(ctx, opt) error +Exec(ctx, opt) error +Rollback(ctx, opt) error } class CLI { +ParseCLIv2(ctx, args) (int, error) } CLI --> App : 呼び出し
  • deploy: deploy.goのApp.Deployメソッドで実装。タスク定義の登録とサービスの更新を行います。
  • diff: diff.goのApp.Diffメソッドで実装。ローカル定義と実際のリソースを比較します。
  • verify: verify.goのApp.Verifyメソッドで実装。リソースの存在と権限を検証します。
  • run: run.goのApp.Runメソッドで実装。一時的なタスクを実行します。
  • exec: exec.goのApp.Execメソッドで実装。実行中のタスクでコマンドを実行します。
  • rollback: rollback.goのApp.Rollbackメソッドで実装。サービスを以前のバージョンに戻します。

各コマンドはApp構造体のメソッドとして実装され、CLI関数がコマンドライン引数を解析して適切なメソッドを呼び出します。

クイックスタート

ecspressoを使用すると、既存/実行中のECSサービスをコードで簡単に管理できます。

ECSサービスに対して--region、--cluster、--serviceオプションを指定してecspresso initを試してみてください：

$ ecspresso init --region ap-northeast-1 --cluster default --service myservice --config ecspresso.yml
2019/10/12 01:31:48 myservice/default save service definition to ecs-service-def.json
2019/10/12 01:31:48 myservice/default save task definition to ecs-task-def.json
2019/10/12 01:31:48 myservice/default save config to ecspresso.yml

生成されたファイル（ecspresso.yml、ecs-service-def.json、ecs-task-def.json）を確認してください。

sequenceDiagram participant User as ユーザー participant Ecspresso as ecspresso participant AWS as AWS API User->>Ecspresso: ecspresso init Ecspresso->>AWS: サービス情報取得 AWS-->>Ecspresso: サービス定義 Ecspresso->>AWS: タスク定義取得 AWS-->>Ecspresso: タスク定義 Ecspresso->>User: 設定ファイル生成

これでecspressoを使用してサービスをデプロイできます！

$ ecspresso deploy --config ecspresso.yml

次のステップ

ecspressoはサービスとタスク定義ファイルをテンプレートとして読み込むことができます。典型的な使用例はタスク定義ファイル内のイメージタグを置き換えることです。

ecs-task-def.json を以下のように変更します：

-  "image": "nginx:latest",
+  "image": "nginx:{{ must_env `IMAGE_TAG` }}",

次に、環境変数IMAGE_TAGを指定してサービスをデプロイします：

$ IMAGE_TAG=stable ecspresso deploy --config ecspresso.yml

テンプレート機能を使用することで、CI/CDパイプラインでの自動デプロイが容易になります。環境変数を使用してイメージタグやその他の設定を動的に変更できます。
