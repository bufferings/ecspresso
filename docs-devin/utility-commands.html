verifyコマンド

verifyコマンドは、設定を検証するために使用します。このコマンドはverify.goファイルに実装されています。

基本的な使い方

ecspresso verify [オプション]

主なオプション

オプション 説明 デフォルト値
--cache 検証結果をキャッシュする false
--only-task-definition タスク定義のみを検証する false
--save-stopped 停止したタスク定義を保存する false
--secrets シークレットを検証する false

検証プロセス

verifyコマンドの内部処理フロー：

flowchart TD A[verify開始] --> B[設定ファイル読込] B --> C[タスク定義ファイル読込] C --> D[サービス定義ファイル読込] D --> E[タスク定義検証] E --> F{タスク定義のみ?} F -->|Yes| G[完了] F -->|No| H[サービス定義検証] H --> I[IAMロール検証] I --> J[ロググループ検証] J --> K[ターゲットグループ検証] K --> L[シークレット検証] L --> M[完了]

使用例

# 基本的な検証
ecspresso verify

# タスク定義のみを検証
ecspresso verify --only-task-definition

# シークレットを検証
ecspresso verify --secrets

# 検証結果をキャッシュ
ecspresso verify --cache

diffコマンド

diffコマンドは、ローカル設定とAWS上の設定の差分を表示するために使用します。このコマンドはdiff.goファイルに実装されています。

基本的な使い方

ecspresso diff [オプション]

主なオプション

オプション 説明 デフォルト値
--unified 統一形式で差分を表示する false
--only-task-definition タスク定義のみの差分を表示する false
--only-service サービス定義のみの差分を表示する false
--with-secrets シークレットを含めて差分を表示する false

差分表示プロセス

diffコマンドの内部処理フロー：

sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWS ECS User->>App: ecspresso diff App->>App: 設定ファイル読込 alt タスク定義の差分を表示する場合 App->>App: ローカルタスク定義読込 App->>AWS: DescribeTaskDefinition AWS-->>App: 現在のタスク定義 App->>App: タスク定義の差分計算 end alt サービス定義の差分を表示する場合 App->>App: ローカルサービス定義読込 App->>AWS: DescribeService AWS-->>App: 現在のサービス定義 App->>App: サービス定義の差分計算 end App-->>User: 差分表示

使用例

# 基本的な差分表示
ecspresso diff

# 統一形式で差分を表示
ecspresso diff --unified

# タスク定義のみの差分を表示
ecspresso diff --only-task-definition

# サービス定義のみの差分を表示
ecspresso diff --only-service

# シークレットを含めて差分を表示
ecspresso diff --with-secrets

statusコマンド

statusコマンドは、ECSサービスの状態を表示するために使用します。このコマンドはecspresso.goファイルに実装されています。

基本的な使い方

ecspresso status [オプション]

主なオプション

オプション 説明 デフォルト値
--events サービスイベントを表示する false
--output 出力形式（table または json） table

状態表示プロセス

statusコマンドの内部処理フロー：

flowchart TD A[status開始] --> B[設定ファイル読込] B --> C[DescribeService API呼び出し] C --> D[サービス情報取得] D --> E{イベント表示?} E -->|Yes| F[サービスイベント取得] E -->|No| G[サービス情報のみ表示] F --> H[イベント情報表示] G --> I[完了] H --> I

使用例

# 基本的な状態表示
ecspresso status

# サービスイベントを表示
ecspresso status --events

# JSON形式で出力
ecspresso status --output json

initコマンド

initコマンドは、設定ファイルを初期化するために使用します。このコマンドはecspresso.goファイルに実装されています。

基本的な使い方

ecspresso init [オプション]

主なオプション

オプション 説明 デフォルト値
--cluster ECSクラスター名 -
--service ECSサービス名 -
--task-definition タスク定義名 -
--region AWSリージョン -
--dry-run 実際に初期化せずに、実行されるアクションを表示する false

初期化プロセス

initコマンドの内部処理フロー：

sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWS ECS User->>App: ecspresso init alt サービス名が指定されている場合 App->>AWS: DescribeService AWS-->>App: サービス情報 App->>AWS: DescribeTaskDefinition AWS-->>App: タスク定義情報 end App->>App: 設定ファイル作成 App->>App: タスク定義ファイル作成 App->>App: サービス定義ファイル作成 App-->>User: 完了メッセージ

使用例

# 基本的な初期化
ecspresso init --cluster my-cluster --service my-service

# ドライランモード（実際に初期化しない）
ecspresso init --cluster my-cluster --service my-service --dry-run

# タスク定義名を指定して初期化
ecspresso init --cluster my-cluster --service my-service --task-definition my-task-definition

# リージョンを指定して初期化
ecspresso init --cluster my-cluster --service my-service --region ap-northeast-1

renderコマンド

renderコマンドは、タスク定義とサービス定義をレンダリングするために使用します。このコマンドはecspresso.goファイルに実装されています。

基本的な使い方

ecspresso render [オプション]

主なオプション

オプション 説明 デフォルト値
--output 出力ファイル名 -
--task-definition タスク定義のみをレンダリングする false
--service-definition サービス定義のみをレンダリングする false

レンダリングプロセス

renderコマンドの内部処理フロー：

flowchart TD A[render開始] --> B[設定ファイル読込] B --> C{レンダリング対象?} C -->|タスク定義| D[タスク定義ファイル読込] C -->|サービス定義| E[サービス定義ファイル読込] C -->|両方| F[両方のファイル読込] D --> G[タスク定義レンダリング] E --> H[サービス定義レンダリング] F --> G F --> H G --> I{出力先?} H --> I I -->|標準出力| J[標準出力に表示] I -->|ファイル| K[ファイルに保存] J --> L[完了] K --> L

使用例

# 両方の定義をレンダリングして標準出力に表示
ecspresso render

# タスク定義のみをレンダリング
ecspresso render --task-definition

# サービス定義のみをレンダリング
ecspresso render --service-definition

# レンダリング結果をファイルに保存
ecspresso render --output rendered.json

appspecコマンド

appspecコマンドは、CodeDeployのAppSpecを生成するために使用します。このコマンドはappspec.goファイルに実装されています。

基本的な使い方

ecspresso appspec [オプション]

主なオプション

オプション 説明 デフォルト値
--output 出力ファイル名 -
--task-definition タスク定義ARN -

AppSpec生成プロセス

appspecコマンドの内部処理フロー：

flowchart TD A[appspec開始] --> B[設定ファイル読込] B --> C[タスク定義ファイル読込] C --> D[サービス定義ファイル読込] D --> E[AppSpec設定読込] E --> F[AppSpec生成] F --> G{出力先?} G -->|標準出力| H[標準出力に表示] G -->|ファイル| I[ファイルに保存] H --> J[完了] I --> J

使用例

# AppSpecを生成して標準出力に表示
ecspresso appspec

# AppSpecをファイルに保存
ecspresso appspec --output appspec.yaml

# タスク定義ARNを指定してAppSpecを生成
ecspresso appspec --task-definition arn:aws:ecs:ap-northeast-1:123456789012:task-definition/my-task:10

ベストプラクティス

ユーティリティコマンドを使用する際のベストプラクティス：

  • デプロイ前の検証: デプロイ前にecspresso verifyとecspresso diffを実行して、設定が正しいことを確認します。
  • CI/CDパイプラインの統合: ecspresso verifyをCI/CDパイプラインに統合して、設定の問題を早期に発見します。
  • 差分の確認: 変更を適用する前にecspresso diffを使用して、変更内容を確認します。
  • 状態の監視: ecspresso statusを使用して、サービスの状態を定期的に確認します。
  • 設定の初期化: 新しいプロジェクトではecspresso initを使用して、設定ファイルを初期化します。
  • テンプレートの活用: ecspresso renderを使用して、テンプレート関数の結果を確認します。
  • CodeDeployの統合: ブルー/グリーンデプロイを使用する場合はecspresso appspecを使用して、AppSpecを生成します。
