リソース検証概要

ecspressoのリソース検証機能は、デプロイ前にAWS上のリソースが正しく設定されているかを確認するために使用されます。この機能はverify.goファイルに実装されており、verifier構造体とverifyState構造体を中心に構築されています。

graph TD A[リソース検証] --> B[タスク定義検証] A --> C[サービス定義検証] A --> D[IAMロール検証] A --> E[ロググループ検証] A --> F[ターゲットグループ検証] A --> G[シークレット検証] B --> H[コンテナ定義] B --> I[タスクロール] B --> J[実行ロール] C --> K[クラスター] C --> L[サービスロール] C --> M[ロードバランサー] D --> N[ロールの存在確認] D --> O[ポリシーの確認] E --> P[ロググループの存在確認] F --> Q[ターゲットグループの存在確認] F --> R[ターゲットグループの設定確認] G --> S[シークレットの存在確認] G --> T[シークレットの値確認]

リソース検証は、以下のような目的で使用されます：

  • デプロイ前にリソースが正しく設定されているかを確認する
  • 設定ミスによるデプロイ失敗を防ぐ
  • 必要なリソースが存在することを確認する
  • 権限の問題を事前に検出する

検証の実装

ecspressoのリソース検証機能は、verify.goファイルに実装されています。中心となるのはverifier構造体で、以下のようなフィールドを持っています：

type verifier struct {
    ecs            ECS
    iam            IAM
    cloudWatchLogs CloudWatchLogs
    elbv2          ELBV2
    ssm            SSM
    secretsManager SecretsManager
    state          *verifyState
    config         *Config
    taskDefinition *TaskDefinition
    service        *Service
}

また、検証状態を管理するためのverifyState構造体も定義されています：

type verifyState struct {
    taskRoleExists        bool
    executionRoleExists   bool
    serviceRoleExists     bool
    logGroupExists        map[string]bool
    targetGroupExists     map[string]bool
    secretExists          map[string]bool
    secretValueExists     map[string]bool
    ssmParameterExists    map[string]bool
    ssmParameterValueExists map[string]bool
}

App構造体のVerifyメソッドは、以下のような処理を行います：

func (d *App) Verify(ctx context.Context, opt VerifyOption) error {
    v := &verifier{
        ecs:            d.ecs,
        iam:            d.iam,
        cloudWatchLogs: d.cloudWatchLogs,
        elbv2:          d.elbv2,
        ssm:            d.ssm,
        secretsManager: d.secretsManager,
        config:         d.config,
        taskDefinition: d.taskDefinition,
        service:        d.service,
        state:          newVerifyState(),
    }
    if opt.Cache {
        v.state = d.verifier.state
    } else {
        d.verifier = v
    }
    return v.verify(ctx, opt)
}

検証プロセス

ecspressoのリソース検証プロセスは、以下のステップで構成されています：

sequenceDiagram participant User as ユーザー participant App as ecspresso participant AWS as AWSサービス User->>App: ecspresso verify App->>App: 設定ファイル読込 App->>App: タスク定義ファイル読込 App->>App: サービス定義ファイル読込 alt タスク定義のみの検証 App->>App: タスク定義検証 else 全体の検証 App->>App: タスク定義検証 App->>App: サービス定義検証 par IAMロール検証 App->>AWS: GetRole（タスクロール） AWS-->>App: ロール情報 App->>AWS: GetRole（実行ロール） AWS-->>App: ロール情報 App->>AWS: GetRole（サービスロール） AWS-->>App: ロール情報 and ロググループ検証 App->>AWS: DescribeLogGroups AWS-->>App: ロググループ情報 and ターゲットグループ検証 App->>AWS: DescribeTargetGroups AWS-->>App: ターゲットグループ情報 and シークレット検証 App->>AWS: DescribeSecret AWS-->>App: シークレット情報 opt シークレット値の検証 App->>AWS: GetSecretValue AWS-->>App: シークレット値 end end end App-->>User: 検証結果

各リソースの検証は、以下のような処理を行います：

タスク定義検証

  • タスク定義のフォーマットが正しいことを確認
  • 必須フィールドが設定されていることを確認
  • コンテナ定義が正しいことを確認

IAMロール検証

  • タスクロールが存在することを確認
  • 実行ロールが存在することを確認
  • サービスロールが存在することを確認

ロググループ検証

  • ログ設定で指定されたロググループが存在することを確認

ターゲットグループ検証

  • ロードバランサー設定で指定されたターゲットグループが存在することを確認

シークレット検証

  • シークレット参照で指定されたシークレットが存在することを確認
  • オプションでシークレット値が存在することを確認

検証オプション

ecspressoのリソース検証は、VerifyOption構造体で設定できます：

type VerifyOption struct {
    OnlyTaskDefinition bool
    Secrets            bool
    Cache              bool
    SaveStopped        bool
}

主要なオプションの説明：

  • OnlyTaskDefinition: タスク定義のみを検証する
  • Secrets: シークレット値を検証する
  • Cache: 検証結果をキャッシュする
  • SaveStopped: 停止したタスク定義を保存する

これらのオプションは、verifyコマンドのフラグとして指定できます：

ecspresso verify [オプション]
オプション 説明 デフォルト値
--only-task-definition タスク定義のみを検証する false
--secrets シークレット値を検証する false
--cache 検証結果をキャッシュする false
--save-stopped 停止したタスク定義を保存する false

検証例

以下は、ecspressoのリソース検証の使用例です：

基本的な検証

ecspresso verify

この例では、タスク定義とサービス定義の両方を検証します。IAMロール、ロググループ、ターゲットグループの存在も確認されます。

タスク定義のみの検証

ecspresso verify --only-task-definition

この例では、タスク定義のみを検証します。サービス定義やその他のリソースは検証されません。

シークレット値の検証

ecspresso verify --secrets

この例では、シークレット参照で指定されたシークレットが存在することを確認し、さらにシークレット値も存在することを確認します。

検証結果のキャッシュ

ecspresso verify --cache

この例では、検証結果をキャッシュします。これにより、後続の検証が高速化されます。

CI/CDパイプラインでの検証

ecspresso verify --only-task-definition

この例は、CI/CDパイプラインでの使用に適しています。タスク定義のみを検証することで、検証時間を短縮できます。
